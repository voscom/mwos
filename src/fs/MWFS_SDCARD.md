# MWFS_SDCARD

**Имя модуля:** `fs` (реализация SD CARD)

**MWFS_SDCARD** — класс реализации файловой системы FAT/exFAT на SD-карте для микроконтроллеров. Наследует базовый класс `MWFS` и предоставляет конкретную реализацию всех методов для работы с внешними SD-картами через интерфейс SPI.

## Описание

Класс предназначен для:
*   **Работа с SD-картой:** Реализация всех методов файловой системы для работы с внешними SD-картами.
*   **Интеграция с MWOSFS:** Используется модулем `MWOSFS` для управления файлами и папками на SD-карте.
*   **Поддержка FAT:** Работа с файловыми системами FAT12, FAT16, FAT32 и exFAT.
*   **SPI интерфейс:** Подключение SD-карты через интерфейс SPI с настраиваемым CS пином.

**Наследование:** `MWFS_SDCARD` → `MWFS` → `MWOSUnitName`

**Тип модуля:** Не является модулем (класс реализации файловой системы)

**Зависимости:**
*   Библиотека `SPI.h` (Arduino Core)
*   Библиотека `SdFat.h` (https://github.com/adafruit/SdFat.git)
*   Библиотека `MWFS.h` (базовый класс файловой системы)
*   Платформа: **Любая Arduino-совместимая** с поддержкой SPI и SD-карт

## Основные возможности

*   **Поддержка FAT/exFAT:** Автоматическое определение типа файловой системы (FAT12, FAT16, FAT32, exFAT).
*   **SPI интерфейс:** Подключение через SPI с настраиваемым CS пином.
*   **Мониторинг памяти:** Получение информации о размере, использованном и свободном месте.
*   **Файловые операции:** Создание, чтение, запись, удаление и переименование файлов.
*   **Работа с директориями:** Создание, удаление и навигация по папкам.
*   **Список файлов:** Поочерёдное получение списка файлов в директории через `openNext()`.
*   **Форматирование:** Метод `format()` для полного очищения SD-карты.

## Параметры конфигурации

Класс не использует макросы `MWOS_PARAM()` так как является классом реализации файловой системы, а не модулем MWOS. Конфигурация осуществляется через конструктор и использование с модулем `MWOSFS`.

| Параметр | Тип | Описание |
|:---|:---|:---|
| `pin` | `MWOS_PIN_INT` | **CS пин SD-карты.** Пин для выбора устройства (по умолчанию SS) |
| `name` | `String` | **Имя файловой системы.** Устанавливается автоматически по типу FAT (FAT12/FAT16/FAT32/exFAT) |
| `currentDir` | `File` | **Текущая директория.** Для итерации файлов |
| `nextFile` | `File` | **Следующий файл.** Для работы с openNext() |

**Примечание:**
*   **Использование:** Класс используется только через модуль `MWOSFS`.
*   **Библиотека:** Требуется библиотека SdFat от Adafruit (https://github.com/adafruit/SdFat.git).
*   **Инициализация:** При создании объекта происходит попытка инициализации SD-карты.

## Конфигурация модуля

```cpp
; класс MWFS_SDCARD не имеет параметров конфигурации через MWOS_PARAM
; Настройка осуществляется через модуль MWOSFS:

; модуль fs - работа с файловой системой
; ID - Описание параметра
; 0 - Команды файловой системы (текстовые команды)
fs:cmd = ""
; 1 - Общий размер файловой системы (байты)
fs:size = 0
; 2 - Использованное место (байты)
fs:used = 0
; 3 - Свободное место (байты)
fs:free = 0
```

**Примечание:**
*   **Параметры:** Класс не использует систему параметров MWOS напрямую.
*   **Настройка:** Все настройки задаются через модуль `MWOSFS`.
*   **CS пин:** Задается в конструкторе при создании объекта (по умолчанию SS).

## Конструкторы

Класс не является шаблонным.

1.  **С указанием CS пина**
    ```cpp
    MWFS_SDCARD(MWOS_PIN_INT csPin=SS);
    ```
    Создаёт экземпляр файловой системы SD-карты с указанием CS пина. По умолчанию используется `SS`.
    *   **Параметры:** `csPin` — пин выбора устройства (Chip Select).

## Основные методы

### Инициализация

*   `virtual void init()` *(переопределён)*
    *   Инициализирует файловую систему и определяет тип FAT.
    *   Устанавливает имя файловой системы (FAT12/FAT16/FAT32/exFAT).

*   `virtual bool format()` *(переопределён)*
    *   Форматирует SD-карту (удаляет все файлы).
    *   **Возвращает:** `bool` — успех операции.

### Информация о памяти

*   `virtual uint32_t page()` *(наследован)*
    *   Возвращает размер страницы файловой системы.
    *   **Возвращает:** `uint32_t` — размер страницы (по умолчанию 2048 байт).

*   `virtual uint64_t size()` *(переопределён)*
    *   Возвращает общий размер SD-карты.
    *   Вычисляется как `clusterCount × sectorsPerCluster × 512`.
    *   **Возвращает:** `uint64_t` — размер в байтах.

*   `virtual uint64_t available()` *(переопределён)*
    *   Возвращает доступное свободное место.
    *   **Возвращает:** `uint64_t` — свободное место в байтах.

*   `uint64_t used()` *(наследован)*
    *   Вычисляет использованное место (size - available).
    *   **Возвращает:** `uint64_t` — использованное место.

### Файловые операции

*   `virtual bool exists(const char* path)` *(переопределён)*
    *   Проверяет существование файла или папки.
    *   **Возвращает:** `bool` — `true` если существует.

*   `virtual File open(const char* path, const uint8_t mode)` *(переопределён)*
    *   Открывает файл для чтения или записи.
    *   Поддерживает режимы: `MWFILE_READ`, `MWFILE_WRITE`, `MWFILE_APPEND`, `MWFILE_WRITE_READ`.
    *   **Возвращает:** `File` — объект файла (SdFile).

*   `virtual bool remove(const char* path)` *(переопределён)*
    *   Удаляет файл.
    *   **Возвращает:** `bool` — успех операции.

*   `virtual bool rename(const char* pathFrom, const char* pathTo)` *(переопределён)*
    *   Переименовывает файл или папку.
    *   **Возвращает:** `bool` — успех операции.

### Операции с директориями

*   `virtual bool mkdir(const char* path, bool createParentPath)` *(переопределён)*
    *   Создаёт новую папку.
    *   **Возвращает:** `bool` — успех операции.

*   `virtual bool rmdir(const char* path)` *(переопределён)*
    *   Удаляет папку.
    *   **Возвращает:** `bool` — успех операции.

*   `virtual bool chdir(const char *path)` *(переопределён)*
    *   Переходит в указанную папку.
    *   Сбрасывает итератор `openNext()` к первому файлу.
    *   **Возвращает:** `bool` — успех операции.

*   `virtual File * openNext(const uint8_t mode)` *(переопределён)*
    *   Открывает следующий файл в текущей папке.
    *   **Возвращает:** `File*` — указатель на файл или NULL если файлов больше нет.

*   `virtual String getFileName(File * file)` *(переопределён)*
    *   Возвращает имя файла из объекта File.
    *   **Возвращает:** `String` — имя файла (до 12 символов + расширение).

## Пример использования

### Через модуль MWOSFS

```cpp
#include "MWOS3.h"
#include "MWOSFS.h"
#include "MWFS_SDCARD.h"

// Создаём объект файловой системы SD-карты
// CS пин по умолчанию = SS
MWFS_SDCARD sdCard(SS);

// Создаём модуль MWOSFS и привязываем файловую систему
MWOSFS fsModule(&sdCard);

void setup() {
    mwos.start();
    
    // Управление через команды модуля fs
    // {fs:cmd=ls;/}
    // {fs:cmd=del;file.txt}
}
```

### С указанием CS пина

```cpp
#include "MWOS3.h"
#include "MWOSFS.h"
#include "MWFS_SDCARD.h"

// Создаём объект файловой системы SD-карты
// CS пин = 5
MWFS_SDCARD sdCard(5);

// Создаём модуль MWOSFS и привязываем файловую систему
MWOSFS fsModule(&sdCard);

void setup() {
    mwos.start();
}
```

### Команды файловой системы

```cpp
// Создание папки
// {fs:cmd=md;myfolder}

// Удаление папки
// {fs:cmd=rd;myfolder}

// Удаление файла
// {fs:cmd=del;file.txt}

// Переименование файла
// {fs:cmd=rn;old.txt;new.txt}

// Листинг файлов
// {fs:cmd=ls;/}
// Ответ: ({fs:cmd=ls;size1=file1.txt;size2=file2.txt})

// Перезагрузка контроллера
// {fs:cmd=reboot}

// Сохранение конфигурации
// {fs:cmd=cfg;save}

// Загрузка конфигурации
// {fs:cmd=cfg;load}

// Форматирование SD-карты
// {fs:cmd=format;fs}
```

### Чтение файла

```cpp
// Читать файл с позиции 0, максимум 1000 символов
// {fs:cmd=type;file.txt,0,1000}
// Ответ: ({fs:cmd=type;содержимое файла})

// Читать следующую часть файла
// {fs:cmd=type;file.txt,1000,1000}
```

### Запись в файл

```cpp
// Добавить текст в файл
// {fs:cmd=echo;file.txt;текст для записи}
// Ответ: ({fs:cmd=echo;Echo file: размер})
```

### Настройка platformio.ini

```ini
[env:esp32dev]
platform = espressif32
board = esp32dev
framework = arduino

; Библиотека SdFat
lib_deps = 
    https://github.com/adafruit/SdFat.git
```

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `pin` | `RAM` | Не сохраняется, устанавливается в конструкторе |
| `name` | `PROGMEM` | Имя файловой системы (FAT12/FAT16/FAT32/exFAT) |
| `currentDir` | `RAM` | Текущая директория, не сохраняется |
| `nextFile` | `RAM` | Следующий файл, не сохраняется |

**Примечание:**
*   **MWOSFS:** Параметры модуля `MWOSFS` (cmd, size, used, free) не сохраняются в энергонезависимой памяти.
*   **Файлы:** Файлы конфигурации сохраняются на SD-карте.

## Важные замечания

1.  **Библиотека SdFat:** Требуется библиотека SdFat от Adafruit (https://github.com/adafruit/SdFat.git). Стандартная библиотека SD может не работать.
2.  **Типы FAT:** Автоматически определяется тип файловой системы (FAT12, FAT16, FAT32, exFAT) при инициализации.
3.  **SPI интерфейс:** SD-карта подключается через SPI. Необходимы пины MOSI, MISO, SCK и CS.
4.  **CS пин:** Пин выбора устройства (Chip Select) задаётся в конструкторе. По умолчанию используется `SS`.
5.  **Форматирование:** Метод `format()` удаляет **все файлы и папки** на SD-карте. Использовать с осторожностью.
6.  **Интеграция с MWOSFS:** Класс предназначен для использования только через модуль `MWOSFS`. Не рекомендуется использовать напрямую.
7.  **Режимы файлов:**
    *   `MWFILE_READ` (0) — открытие для чтения
    *   `MWFILE_WRITE` (1) — открытие для записи (перезаписывает файл)
    *   `MWFILE_APPEND` (2) — открытие для добавления в конец
    *   `MWFILE_WRITE_READ` (3) — открытие для чтения и записи
8.  **Имя ФС:** Имя устанавливается автоматически по типу FAT (FAT12/FAT16/FAT32/exFAT).
9.  **Существует/Не существует:** Метод `exists()` работает как для файлов, так и для папок.
10. **Переименование:** Метод `rename()` может работать как для файлов, так и для папок.
11. **Создание папок:** Метод `mkdir()` поддерживает параметр `createParentPath` для создания родительских папок.
12. **Конфигурационные файлы:** Модуль `MWOSFS` загружает конфигурацию из файлов `/defaults.cfg`, `/t{тип}.cfg`, `/{CID}.cfg`, `/u{CID}.cfg` на SD-карте.
13. **Питание:** SD-карта требует стабильного питания 3.3V или 5V в зависимости от модели.
14. **Логирование:** Включается через флаг `MW_LOG_MODULE` для отладки операций с файлами.
15. **Скорость SPI:** По умолчанию используется `SPI_HALF_SPEED` для надёжности. Можно изменить для увеличения скорости.
