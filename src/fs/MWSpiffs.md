# MWSpiffs

**Имя модуля:** `fs` (реализация SPIFFS)

**MWSpiffs** — класс реализации файловой системы SPIFFS для микроконтроллеров ESP32 и ESP8266. Наследует базовый класс `MWFS` и предоставляет конкретную реализацию всех методов для работы с внутренней flash-памятью через файловую систему SPIFFS.

## Описание

Класс предназначен для:
*   **Работа с SPIFFS:** Реализация всех методов файловой системы для ESP32/ESP8266.
*   **Интеграция с MWOSFS:** Используется модулем `MWOSFS` для управления файлами и папками.
*   **Абстракция платформы:** Унифицированный интерфейс для работы с файловой системой на разных платформах ESP.
*   **Итерация файлов:** Поддержка последовательного получения списка файлов в директории.

**Наследование:** `MWSpiffs` → `MWFS` → `MWOSUnitName`

**Тип модуля:** Не является модулем (класс реализации файловой системы)

**Зависимости:**
*   Библиотека `SPIFFS.h` (ESP32/ESP8266 Core)
*   Библиотека `MWFS.h` (базовый класс файловой системы)
*   Платформа: **ESP32, ESP8266** (требуется поддержка SPIFFS)

## Основные возможности

*   **Инициализация SPIFFS:** Автоматическая инициализация файловой системы при старте.
*   **Форматирование:** Метод `format()` для полного очищения файловой системы.
*   **Мониторинг памяти:** Получение информации о размере, использованном и свободном месте.
*   **Файловые операции:** Создание, чтение, запись, удаление и переименование файлов.
*   **Работа с директориями:** Создание, удаление и навигация по папкам.
*   **Список файлов:** Поочерёдное получение списка файлов в директории через `openNext()`.
*   **Разные платформы:** Поддержка ESP32 и ESP8266 с учётом различий в API.

## Параметры конфигурации

Класс не использует макросы `MWOS_PARAM()` так как является классом реализации файловой системы, а не модулем MWOS. Конфигурация осуществляется через наследование и использование с модулем `MWOSFS`.

| Параметр | Тип | Описание |
|:---|:---|:---|
| `name` | `String` | **Имя файловой системы.** Устанавливается как "SPIFFS" |
| `currentDir` | `Dir/File` | **Текущая директория.** Для итерации файлов (ESP8266/ESP32) |
| `currentFile` | `File` | **Текущий файл.** Для работы с открытым файлом |

**Примечание:**
*   **Использование:** Класс используется только через модуль `MWOSFS`.
*   **Платформа:** На ESP8266 используется тип `Dir` для директорий, на ESP32 — тип `File`.
*   **Инициализация:** Использует `SPIFFS.begin(true)` для авто-форматирования при ошибке.

## Конфигурация модуля

```cpp
; класс MWSpiffs не имеет параметров конфигурации через MWOS_PARAM
; Настройка осуществляется через модуль MWOSFS:

; модуль fs - работа с файловой системой
; ID - Описание параметра
; 0 - Команды файловой системы (текстовые команды)
fs:cmd = ""
; 1 - Общий размер файловой системы (байты)
fs:size = 0
; 2 - Использованное место (байты)
fs:used = 0
; 3 - Свободное место (байты)
fs:free = 0
```

**Примечание:**
*   **Параметры:** Класс не использует систему параметров MWOS напрямую.
*   **Настройка:** Все настройки задаются через модуль `MWOSFS`.
*   **Разметка разделов:** В `platformio.ini` необходимо указать `board_build.filesystem = spiffs`.

## Конструкторы

Класс не является шаблонным.

1.  **По умолчанию**
    ```cpp
    MWSpiffs();
    ```
    Создаёт экземпляр файловой системы SPIFFS. Имя устанавливается как "SPIFFS".

## Основные методы

### Инициализация

*   `virtual void init()` *(переопределён)*
    *   Инициализирует файловую систему SPIFFS.
    *   Использует `SPIFFS.begin(true)` для авто-форматирования.

*   `virtual bool format()` *(переопределён)*
    *   Форматирует файловую систему (удаляет все файлы).
    *   **Возвращает:** `bool` — успех операции.

### Информация о памяти

*   `virtual uint32_t page()` *(наследован)*
    *   Возвращает размер страницы файловой системы.
    *   **Возвращает:** `uint32_t` — размер страницы (по умолчанию 2048 байт).

*   `virtual uint64_t size()` *(переопределён)*
    *   Возвращает общий размер файловой системы.
    *   На ESP8266 использует `FSInfo`, на ESP32 — `SPIFFS.totalBytes()`.
    *   **Возвращает:** `uint64_t` — размер в байтах.

*   `virtual uint64_t available()` *(переопределён)*
    *   Возвращает доступное свободное место.
    *   **Возвращает:** `uint64_t` — свободное место в байтах.

*   `uint64_t used()` *(наследован)*
    *   Вычисляет использованное место (size - available).
    *   **Возвращает:** `uint64_t` — использованное место.

### Файловые операции

*   `virtual bool exists(const char* path)` *(переопределён)*
    *   Проверяет существование файла или папки.
    *   **Возвращает:** `bool` — `true` если существует.

*   `virtual File open(const char* path, const uint8_t mode)` *(переопределён)*
    *   Открывает файл для чтения или записи.
    *   Поддерживает режимы: `MWFILE_READ`, `MWFILE_WRITE`, `MWFILE_APPEND`, `MWFILE_WRITE_READ`.
    *   **Возвращает:** `File` — объект файла.

*   `virtual bool remove(const char* path)` *(переопределён)*
    *   Удаляет файл.
    *   **Возвращает:** `bool` — успех операции.

*   `virtual bool rename(const char* pathFrom, const char* pathTo)` *(переопределён)*
    *   Переименовывает файл или папку.
    *   **Возвращает:** `bool` — успех операции.

### Операции с директориями

*   `virtual bool mkdir(const char* path, bool createParentPath)` *(переопределён)*
    *   Создаёт новую папку.
    *   **Возвращает:** `bool` — успех операции.

*   `virtual bool rmdir(const char* path)` *(переопределён)*
    *   Удаляет папку.
    *   **Возвращает:** `bool` — успех операции.

*   `virtual bool chdir(const char *path)` *(переопределён)*
    *   Переходит в указанную папку.
    *   Сбрасывает итератор `openNext()` к первому файлу.
    *   **Возвращает:** `bool` — успех операции.

*   `virtual File * openNext(const uint8_t mode)` *(переопределён)*
    *   Открывает следующий файл в текущей папке.
    *   На ESP8266 использует `currentDir.next()`, на ESP32 — `currentDir.openNextFile()`.
    *   **Возвращает:** `File*` — указатель на файл или NULL если файлов больше нет.

*   `virtual String getFileName(File * file)` *(переопределён)*
    *   Возвращает имя файла из объекта File.
    *   **Возвращает:** `String` — имя файла.

## Пример использования

### Через модуль MWOSFS

```cpp
#include "MWOS3.h"
#include "MWOSFS.h"
#include "MWSpiffs.h"

// Создаём объект файловой системы SPIFFS
MWSpiffs spiffs;

// Создаём модуль MWOSFS и привязываем файловую систему
MWOSFS fsModule(&spiffs);

void setup() {
    mwos.start();
    
    // Управление через команды модуля fs
    // {fs:cmd=ls;/}
    // {fs:cmd=del;file.txt}
}
```

### Команды файловой системы

```cpp
// Создание папки
// {fs:cmd=md;myfolder}

// Удаление папки
// {fs:cmd=rd;myfolder}

// Удаление файла
// {fs:cmd=del;file.txt}

// Переименование файла
// {fs:cmd=rn;old.txt;new.txt}

// Листинг файлов
// {fs:cmd=ls;/}
// Ответ: ({fs:cmd=ls;size1=file1.txt;size2=file2.txt})

// Перезагрузка контроллера
// {fs:cmd=reboot}

// Сохранение конфигурации
// {fs:cmd=cfg;save}

// Загрузка конфигурации
// {fs:cmd=cfg;load}

// Форматирование файловой системы
// {fs:cmd=format;fs}
```

### Чтение файла

```cpp
// Читать файл с позиции 0, максимум 1000 символов
// {fs:cmd=type;file.txt,0,1000}
// Ответ: ({fs:cmd=type;содержимое файла})

// Читать следующую часть файла
// {fs:cmd=type;file.txt,1000,1000}
```

### Запись в файл

```cpp
// Добавить текст в файл
// {fs:cmd=echo;file.txt;текст для записи}
// Ответ: ({fs:cmd=echo;Echo file: размер})
```

### Настройка platformio.ini

```ini
[env:esp32dev]
platform = espressif32
board = esp32dev
framework = arduino

; Настройка файловой системы SPIFFS
board_build.filesystem = spiffs

; Библиотеки
lib_deps = 
    SPIFFS
```

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `name` | `PROGMEM` | Имя "SPIFFS" хранится в flash-памяти |
| `currentDir` | `RAM` | Текущая директория, не сохраняется |
| `currentFile` | `RAM` | Текущий файл, не сохраняется |

**Примечание:**
*   **MWOSFS:** Параметры модуля `MWOSFS` (cmd, size, used, free) не сохраняются в энергонезависимой памяти.
*   **Файлы:** Файлы конфигурации сохраняются в файловой системе SPIFFS.

## Важные замечания

1.  **Платформа:** Класс работает только на ESP32 и ESP8266 с поддержкой SPIFFS.
2.  **Инициализация:** Использует `SPIFFS.begin(true)` для авто-форматирования при ошибке.
3.  **Разметка разделов:** В `platformio.ini` необходимо указать `board_build.filesystem = spiffs`.
4.  **Итерация файлов:** На ESP8266 используется тип `Dir`, на ESP32 — тип `File` для итерации директорий.
5.  **Форматирование:** Метод `format()` удаляет **все файлы и папки** на файловой системе. Использовать с осторожностью.
6.  **Интеграция с MWOSFS:** Класс предназначен для использования только через модуль `MWOSFS`. Не рекомендуется использовать напрямую.
7.  **Режимы файлов:**
    *   `MWFILE_READ` (0) — открытие для чтения
    *   `MWFILE_WRITE` (1) — открытие для записи (перезаписывает файл)
    *   `MWFILE_APPEND` (2) — открытие для добавления в конец
    *   `MWFILE_WRITE_READ` (3) — открытие для чтения и записи
8.  **Имя ФС:** Имя устанавливается как "SPIFFS" через константу `MWOS_FSNAME_SPIFFS` в PROGMEM.
9.  **Существует/Не существует:** Метод `exists()` работает как для файлов, так и для папок.
10. **Переименование:** Метод `rename()` может работать как для файлов, так и для папок.
11. **Создание папок:** Метод `mkdir()` поддерживает параметр `createParentPath` для создания родительских папок.
12. **Конфигурационные файлы:** Модуль `MWOSFS` загружает конфигурацию из файлов `/defaults.cfg`, `/t{тип}.cfg`, `/{CID}.cfg`, `/u{CID}.cfg`.
13. **Устаревание:** SPIFFS считается устаревшей файловой системой. Для новых проектов рекомендуется использовать LittleFS.
14. **Логирование:** Включается через флаг `MW_LOG_MODULE` для отладки операций с файлами.

## См. также

*   **MWOSFS**: Модуль работы с файловой системой в системе MWOS (основной интерфейс).
*   **MWFS**: Базовый класс для всех файловых систем.
*   **MWLittleFS**: Реализация файловой системы LittleFS (рекомендуется для новых проектов).
*   **MWOSModule**: Базовый класс для всех модулей системы.
*   **SPIFFS**: Файловая система для ESP8266/ESP32 (документация Espressif).
*   **MWOSNetModule**: Модуль сети (использует CID для имён файлов конфигурации).
