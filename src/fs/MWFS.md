# MWFS

**Имя модуля:** `fs` (базовый класс)

**MWFS** — базовый класс для организации работы с файловыми системами в системе MWOS. Предоставляет абстрактный интерфейс для управления файлами и папками, который используется модулем `MWOSFS` для работы с конкретными файловыми системами (LittleFS, SPIFFS, SD и др.).

## Описание

Класс предназначен для:
*   **Абстракция файловой системы:** Унифицированный интерфейс для работы с различными файловыми системами.
*   **Интеграция с MWOSFS:** Базовый класс для модуля `MWOSFS`, не предназначен для отдельного использования.
*   **Файловые операции:** Создание, чтение, запись, удаление и переименование файлов.
*   **Работа с директориями:** Создание, удаление и навигация по папкам.
*   **Мониторинг памяти:** Получение информации о размере, использованном и свободном месте.

**Наследование:** `MWFS` → `MWOSUnitName`

**Тип модуля:** Не является модулем (базовый класс для файловых систем)

**Зависимости:**
*   Библиотека `FS.h` (Arduino Core)
*   Платформа: **ESP8266, ESP32** (требуется поддержка файловой системы)

## Основные возможности

*   **Универсальный интерфейс:** Единые методы для всех типов файловых систем.
*   **Режимы доступа:** Чтение, запись, добавление, чтение/запись.
*   **Проверка существования:** Метод `exists()` для проверки наличия файлов и папок.
*   **Навигация:** Поддержка текущей директории через `chdir()`.
*   **Итерация файлов:** Метод `openNext()` для последовательного получения списка файлов.
*   **Форматирование:** Метод `format()` для полного очищения файловой системы.
*   **Информация о памяти:** Методы `size()`, `used()`, `available()` для мониторинга.

## Параметры конфигурации

Класс не использует макросы `MWOS_PARAM()` так как является базовым классом для `MWOSFS`, а не самостоятельным модулем.

| Параметр | Тип | Описание |
|:---|:---|:---|
| `pin` | `MWOS_PIN_INT` | **Пин SD-карты.** Пин для работы с SD-картой (по умолчанию -99) |
| `name` | `String` | **Имя файловой системы.** Устанавливается в конструкторе |

**Примечание:**
*   **Наследование:** Конкретные реализации (LittleFS, SPIFFS, SD) наследуют этот класс и реализуют виртуальные методы.
*   **Использование:** Класс используется только через модуль `MWOSFS`.
*   **Режимы файлов:**
    *   `MWFILE_READ` (0) — только чтение
    *   `MWFILE_WRITE` (1) — только запись
    *   `MWFILE_APPEND` (2) — добавление в конец
    *   `MWFILE_WRITE_READ` (3) — чтение и запись

## Конфигурация модуля

```cpp
; класс MWFS не имеет параметров конфигурации через MWOS_PARAM
; Настройка осуществляется через модуль MWOSFS:

; модуль fs - работа с файловой системой
; ID - Описание параметра
; 0 - Команды файловой системы (текстовые команды)
fs:cmd = ""
; 1 - Общий размер файловой системы (байты)
fs:size = 0
; 2 - Использованное место (байты)
fs:used = 0
; 3 - Свободное место (байты)
fs:free = 0
```

**Примечание:**
*   **Параметры:** Класс не использует систему параметров MWOS напрямую.
*   **Настройка:** Все настройки задаются через модуль `MWOSFS`.
*   **Режимы файлов:** Используются константы `MWFILE_READ`, `MWFILE_WRITE` и др.

## Конструкторы

Класс не является шаблонным.

1.  **По умолчанию**
    ```cpp
    MWFS();
    ```
    Создаёт экземпляр базового класса файловой системы. Имя устанавливается как "FS error!".

## Основные методы

### Инициализация

*   `virtual void init()` *(виртуальный)*
    *   Инициализирует файловую систему.
    *   Переопределяется в потомках для конкретной реализации.

*   `virtual bool format()` *(виртуальный)*
    *   Форматирует файловую систему (удаляет все файлы).
    *   **Возвращает:** `bool` — успех операции.

### Информация о памяти

*   `virtual uint32_t page()` *(виртуальный)*
    *   Возвращает размер страницы файловой системы.
    *   **Возвращает:** `uint32_t` — размер страницы (по умолчанию 2048 байт).

*   `virtual uint64_t size()` *(виртуальный)*
    *   Возвращает общий размер файловой системы.
    *   **Возвращает:** `uint64_t` — размер в байтах.

*   `virtual uint64_t available()` *(виртуальный)*
    *   Возвращает доступное свободное место.
    *   **Возвращает:** `uint64_t` — свободное место в байтах.

*   `uint64_t used()`
    *   Вычисляет использованное место.
    *   **Возвращает:** `uint64_t` — использованное место (size - available).

### Файловые операции

*   `virtual bool exists(const char* path)` *(виртуальный)*
    *   Проверяет существование файла или папки.
    *   **Возвращает:** `bool` — `true` если существует.

*   `virtual File open(const char* path, const uint8_t mode)` *(виртуальный)*
    *   Открывает файл для чтения или записи.
    *   **Параметры:** `path` — путь к файлу, `mode` — режим доступа.
    *   **Возвращает:** `File` — объект файла.

*   `virtual bool remove(const char* path)` *(виртуальный)*
    *   Удаляет файл.
    *   **Возвращает:** `bool` — успех операции.

*   `virtual bool rename(const char* pathFrom, const char* pathTo)` *(виртуальный)*
    *   Переименовывает файл или папку.
    *   **Возвращает:** `bool` — успех операции.

### Операции с директориями

*   `virtual bool mkdir(const char* path, bool createParentPath)` *(виртуальный)*
    *   Создаёт новую папку.
    *   **Возвращает:** `bool` — успех операции.

*   `virtual bool rmdir(const char* path)` *(виртуальный)*
    *   Удаляет папку.
    *   **Возвращает:** `bool` — успех операции.

*   `virtual bool chdir(const char* path)` *(виртуальный)*
    *   Переходит в указанную папку.
    *   Сбрасывает итератор `openNext()` к первому файлу.
    *   **Возвращает:** `bool` — успех операции.

*   `virtual File* openNext(const uint8_t mode)` *(виртуальный)*
    *   Открывает следующий файл в текущей папке.
    *   **Возвращает:** `File*` — указатель на файл или NULL если файлов больше нет.

*   `virtual String getFileName(File* file)` *(виртуальный)*
    *   Возвращает имя файла из объекта File.
    *   **Возвращает:** `String` — имя файла.

## Пример использования

### Через модуль MWOSFS

```cpp
#include "MWOS3.h"
#include "MWOSFS.h"
#include "LittleFS.h"

// Создаём объект файловой системы LittleFS
class MWFS_LittleFS : public MWFS {
public:
    virtual void init() {
        LittleFS.begin();
    }
    virtual uint64_t size() {
        return LittleFS.totalBytes();
    }
    virtual uint64_t available() {
        return LittleFS.freeBytes();
    }
    // ... другие методы
};

MWFS_LittleFS littleFS;

// Создаём модуль MWOSFS и привязываем файловую систему
MWOSFS fsModule(&littleFS);

void setup() {
    mwos.start();
    
    // Управление через команды модуля fs
    // {fs:cmd=ls;/}
    // {fs:cmd=del;file.txt}
}
```

### Команды файловой системы

```cpp
// Создание папки
// {fs:cmd=md;myfolder}

// Удаление папки
// {fs:cmd=rd;myfolder}

// Удаление файла
// {fs:cmd=del;file.txt}

// Переименование файла
// {fs:cmd=rn;old.txt;new.txt}

// Листинг файлов
// {fs:cmd=ls;/}
// Ответ: ({fs:cmd=ls;size1=file1.txt;size2=file2.txt})

// Перезагрузка контроллера
// {fs:cmd=reboot}

// Сохранение конфигурации
// {fs:cmd=cfg;save}

// Загрузка конфигурации
// {fs:cmd=cfg;load}

// Форматирование файловой системы
// {fs:cmd=format;fs}
```

### Чтение файла

```cpp
// Читать файл с позиции 0, максимум 1000 символов
// {fs:cmd=type;file.txt,0,1000}
// Ответ: ({fs:cmd=type;содержимое файла})

// Читать следующую часть файла
// {fs:cmd=type;file.txt,1000,1000}
```

### Запись в файл

```cpp
// Добавить текст в файл
// {fs:cmd=echo;file.txt;текст для записи}
// Ответ: ({fs:cmd=echo;Echo file: размер})
```

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `pin` | `RAM` | Не сохраняется, устанавливается программно |
| `name` | `RAM` | Не сохраняется, устанавливается в конструкторе |

**Примечание:**
*   **MWOSFS:** Параметры модуля `MWOSFS` (cmd, size, used, free) не сохраняются в энергонезависимой памяти.
*   **Файлы:** Файлы конфигурации сохраняются в файловой системе (EEPROM/Flash).

## Важные замечания

1.  **Базовый класс:** MWFS является абстрактным базовым классом. Для работы необходимо создать потомка с реализацией всех виртуальных методов.
2.  **Интеграция с MWOSFS:** Класс предназначен для использования только через модуль `MWOSFS`. Не рекомендуется использовать напрямую.
3.  **Режимы файлов:**
    *   `MWFILE_READ` (0) — открытие для чтения
    *   `MWFILE_WRITE` (1) — открытие для записи (перезаписывает файл)
    *   `MWFILE_APPEND` (2) — открытие для добавления в конец
    *   `MWFILE_WRITE_READ` (3) — открытие для чтения и записи
4.  **Итерация файлов:** Метод `openNext()` возвращает файлы по одному. Вызов `chdir()` сбрасывает итератор к первому файлу.
5.  **Форматирование:** Метод `format()` удаляет **все файлы и папки** на файловой системе. Использовать с осторожностью.
6.  **Платформа:** На ESP8266/ESP32 используются реализации LittleFS или SPIFFS. На других платформах могут быть другие реализации.
7.  **SD-карты:** Для работы с SD-картами необходимо задать пин через параметр `pin`.
8.  **Размер страницы:** Метод `page()` возвращает размер страницы файловой системы (по умолчанию 2048 байт).
9.  **Имя ФС:** По умолчанию имя устанавливается как "FS error!". Переопределяется в потомках.
10. **Существует/Не существует:** Метод `exists()` работает как для файлов, так и для папок.
11. **Переименование:** Метод `rename()` может работать как для файлов, так и для папок.
12. **Создание папок:** Параметр `createParentPath` в `mkdir()` позволяет создавать родительские папки автоматически.
13. **MWOSFS модуль:** Все команды выполняются через параметр `cmd` модуля `MWOSFS`.
14. **Конфигурационные файлы:** Модуль `MWOSFS` загружает конфигурацию из файлов `/defaults.cfg`, `/t{тип}.cfg`, `/{CID}.cfg`, `/u{CID}.cfg`.
