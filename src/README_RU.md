# MWOS3.5 — Модульная ОС для IoT (v.3.5)

![Version](https://img.shields.io/badge/version-3.0-blue.svg)
[![License](https://img.shields.io/badge/license-MIT-green.svg)](LICENSE_RU)

![Language](https://img.shields.io/badge/language-C++11%20%7C%20MWScript-yellow.svg)

**MWOS (Modular Web Operating System)** — операционная система для микроконтроллеров, предназначенная для удалённого мониторинга и управления оборудованием.

> 💡 **Ключевое преимущество:** Архитектура MWOS заметно упрощает новую разработку. Вы пишете только логику работы с устройством, а ОС берёт на себя сеть, веб-интерфейс, хранение настроек и протоколы связи.

## 🚀 Возможности

MWOS обеспечивает беспрецедентную платформенную независимость, поддерживая более сотни микроконтроллеров, совместимых с платформой Arduino — от классических AVR и недорогих ESP8266 до мощных ESP32 и STM32. Конечный пользователь может сразу начать работать со своими устройствами через централизованный веб-интерфейс (iot.voscom.online или самостоятельно развёрнутый сервер) и кроссплатформенное приложение mwosApp: все настройки, мониторинг и управление осуществляются из единой точки, а данные в реальном времени накапливаются в центральной базе данных SQL, что открывает возможности для гибкой аналитики, построения отчётов и интеграции с внешними системами.

Практически любой функционал контроллера реализуется путём комбинирования готовых модулей MWOS и настройки их логического взаимодействия через систему параметров и событий — без необходимости писать низкоуровневый код. Разработчикам же достаточно создать новый модуль, унаследовавшись от базового класса, чтобы добавить в систему принципиально новые возможности, которые сразу становятся доступны для удалённой настройки и использования. Благодаря такой архитектуре любая задача, связанная с микроконтроллерами — от простого датчика до распределённой системы умного дома или промышленной автоматизации — решается быстрее, надёжнее и с минимальными затратами на поддержку.

| Возможность | Описание                                                                                                     |
|-------------|--------------------------------------------------------------------------------------------------------------|
| 🔌 Модульность | Каждое устройство управляется отдельным программным модулем                                                  |
| 🎛 Параметры | Гибкая настройка модулей: конфигурация, управление, мониторинг                                               |
| 🖥 Виджеты | Параметры автоматически отображаются как виджеты в веб-интерфейсе                                            |
| 🌐 Универсальный протокол | Бинарный протокол работает поверх WiFi, LAN, USB, RS232/485, CAN, радиоканала                                |
| ⚙️ MWScript | Собственный скриптовый язык для сложных сценариев на сервере и контроллере                                   |
| 🔄 Кроссплатформенность | Серверная часть на .NET работает на Windows и Linux. Программа mwosApp работает на Windows, Linux и Android. |

## 🎯 Начало работы

### Подключение существующего контроллера

1.  Перейдите в [Личный кабинет MWOS](https://iot.voscom.online/)
2.  Включите контроллер с прошивкой MWOS
3.  Контроллер появится в списке автоматически *(если он уже зарегистрирован на вас)*

### Если контроллер не появился:

📱 Вариант А: QR-код

- Выберите "Добавить контроллер по QR-коду"
- Отсканируйте код с экрана контроллера

⌨️ Вариант Б: Ручной ввод
- Введите код контроллера вручную

🤝 Вариант В: Запрос прав
- Если контроллер у другого администратора — укажите его Email или код
- Администратор получит заявку на предоставление вам прав доступа

> ✅ После появления контроллера в личном кабинете вы можете сразу управлять им и подключёнными устройствами.

## 🛠 Создание прошивки

### 1. Подготовка оборудования

- Выберите микроконтроллер из [списка поддерживаемых](#-поддержка-новых-платформ)
- Убедитесь, что есть интерфейс для подключения к сети: `WiFi`, `LAN`, `USB`, `RS232`, `RS485`, `CAN` и др.

### 2. Генерация прошивки в личном кабинете

В разделе **«Создать прошивку контроллера»** заполните форму:

- [ ] Выберите модель микроконтроллера
- [ ] Укажите тип сетевого интерфейса
- [ ] Отметьте программные модули устройств для включения в прошивку
- [ ] Нажмите **«Скачать прошивку BIN»** *(готовая к загрузке)*  
  или **«Скачать исходники»** *(для самостоятельной компиляции)*

### 3. Загрузка в устройство

Запишите полученный `.bin`-файл в микроконтроллер через Arduino IDE, esptool, PlatformIO или другую сервисную утилиту.

> 🎉 После первого включения контроллер автоматически появится в вашем личном кабинете, а вы станете его администратором.  
> Администратор может делегировать права управления другим пользователям.

## 💻 Разработка своих модулей

Мы постоянно пополняем библиотеку верифицированных модулей, но вы легко можете создать свой.

### Почему это просто?

✅ Не нужно писать код для:
- Подключения к Wi-Fi / Ethernet
- Веб-сервера и обработки HTTP
- Хранения настроек в EEPROM/Flash
- Синхронизации с облачным сервером

✅ Вы пишете только:
- Логику работы с вашим устройством
- Описание параметров (настройки, показания)
- (Опционально) внешний вид виджета

🔧 Поддержка новых платформ

MWOS написана на C++11 и совместима с любым микроконтроллером, поддерживающим Arduino Framework.

>⚠️ Важно: Arduino Framework ≠ платы Arduino.

Фреймворк — это набор библиотек и правил конфигурации, который работает на сотнях платформ: от ATtiny до ESP32, Raspberry Pi Pico и даже Linux.

Если вашей платформы нет в списке:
Проверьте, есть ли поддержка Arduino Framework для вашего МК.
Если нет — вы можете добавить порт самостоятельно (исходники открыты).
Или используйте альтернативный фреймворк с минимальными доработками.

🌟 Бонус: Добавив поддержку новой платформы, вы получаете возможность компиляции под неё тысяч готовых скетчей из экосистемы Arduino и поддержку активного сообщества.

## ⚙️ Самостоятельная компиляция

Для продвинутых пользователей и кастомных сборок:

```bash
git clone https://github.com/yourname/mwos3.git
cd mwos3
# См. подробные инструкции:
open examples/README.md
```

В папке `examples` вы найдёте:
- Примеры сборки под разные платформы
- Инструкции по настройке toolchain
- Шаблоны для добавления новых модулей

## 🔬 Технические особенности MWOS3

| Особенность | Описание                                                                                                                |
|-------------|-------------------------------------------------------------------------------------------------------------------------|
| 🌐 Удалённая конфигурация | Все настройки модулей доступны и изменяются через веб-интерфейс                                                         |
| 🔌 Универсальный протокол | Бинарный протокол абстрагирует физический уровень: WiFi, RS485, CAN, радиоканал и др.                                   |
| 🖥 Серверная часть | Кроссплатформенная (.NET), масштабируемая под тысячи контроллеров и пользователей                                       |
| ⚙️ MWScript | Собственный скриптовый язык, компилируемый в байт-код для сервера и контроллера                                         |
| 🔧 Макроподстановки | Расширенные возможности директив предпроцессора, позволяют адаптировать синтаксис MWScript под C++, Basic, G-коды и др. |
| 🎯 Интерпретатор байт-кода | Позволяет запускать сложные сценарии: от автоматизации датчиков до управления станками через G-коды и M-коды            |
