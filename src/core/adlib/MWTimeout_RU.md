# 📘 Документация: `MWTimeout.h`
**Язык:** C++11 | **Платформа:** Arduino (AVR, ESP32, STM32, RP2040 и др.)

---

## 💡 Зачем нужны неблокирующие таймеры?
В стандартном Arduino-коде часто используется `delay()`, который **полностью останавливает выполнение программы** на заданное время. В этот момент микроконтроллер не может опрашивать датчики, обрабатывать команды по UART/I2C/SPI, обновлять дисплей или реагировать на нажатия кнопок.

`MWTimeout` реализует **неблокирующий подход**: объект запоминает момент старта, а в цикле `loop()` только проверяет, прошло ли заданное время. Это позволяет:
- ✅ Выполнять несколько параллельных процессов без задержек (мигание LED + опрос кнопки + отправка данных).
- ✅ Строить детерминированные конечные автоматы (FSM) и отзывчивые интерфейсы.
- ✅ Экономить такты процессора, избегая простоя и сложных прерываний.

---

## 🆚 Почему `MWTimeout` лучше ручного сравнения с `millis()`?
Типичный ручной подход выглядит так:
```cpp
unsigned long lastTime = 0;
const unsigned long interval = 5000;

void loop() {
    if (millis() - lastTime >= interval) {
        lastTime = millis();
        // действие...
    }
}
```
| Проблема ручного кода | Как решает `MWTimeout` |
|-----------------------|------------------------|
| ❌ **Многословность** | На каждый таймер нужны отдельные переменные `lastTime`, `interval`. Код разрастается, ОЗУ расходуется неэффективно. | ✅ **Инкапсуляция**: один объект хранит всё состояние. API: `start()`, `isTimeout()`. |
| ❌ **Риск ошибок логики** | Легко забыть обновить `lastTime`, перепутать порядок вычитания или использовать знаковый тип, что сломает обработку переполнения. | ✅ **Защита от переполнения**: используется беззнаковая арифметика. Переполнение `millis()` (~49 дней) обрабатывается аппаратно корректно. |
| ❌ **Нет абстракции единиц** | Приходится вручную конвертировать секунды → миллисекунды, каждый раз делая деление в рантайме. | ✅ **Встроенные конвертеры**: `startMS()`, `startSec()` масштабируют значения под `accuracy` автоматически. |
| ❌ **Скрытые затраты** | `unsigned long` всегда занимает 4 байта, даже если таймер на 1 секунду. Деление `millis()/коэффициент` выполняется программно на AVR. | ✅ **Контроль ОЗУ и CPU**: шаблон `classTime` задает размер (1/2/4 байта), а `divOptimization` заменяет деление на 1-тактовый сдвиг. |

💡 **Итог:** `MWTimeout` убирает бойлерплейт, защищает от типичных embedded-ошибок и даёт предсказуемое потребление памяти/тактов, сохраняя гибкость ручного кода.

## ⚡ Ключевые особенности

### 💾 Экономия ОЗУ
| Аспект | Описание |
|--------|----------|
| **Размер экземпляра** | Зависит только от `classTime`. Занимает **1, 2 или 4 байта** на объект. Никаких скрытых полей. |
| **Статический кэш времени** | `static classTime nowTime` **делится между всеми объектами** одного шаблонного инстанса. Исключает дублирование и снижает обращения к системному таймеру. |
| **Отсутствие динамической памяти** | Класс не использует `new`, `malloc` или стандартные контейнеры. Всё выделяется на стеке/в `.data` на этапе компиляции. |

### 🚀 Оптимизация деления
| Аспект | Описание |
|--------|----------|
| **Бинарный сдвиг вместо `/`** | При `divOptimization = true` операция `1000 / accuracy` заменяется на `>> shift`. Степень сдвига вычисляется на этапе компиляции (`constexpr`). |
| **Зачем это нужно?** | 8-битные AVR не имеют аппаратного делителя. Деление выполняется программно за 50–200 тактов. Побитовый сдвиг занимает **1 такт**. |
| **Точность** | Делитель округляется до ближайшей степени двойки. Приоритет: скорость. Для `accuracy=1000` оптимизация отключена автоматически (делитель = 1, компилятор оптимизирует операцию в ноль). |

## 🛠 Параметры шаблона
```cpp
template<class classTime = uint16_t, uint16_t accuracy = 1000, bool divOptimization = false>
class MWTimeout { ... };
```

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `classTime` | `uint8_t`, `uint16_t`, `uint32_t` | `uint16_t` | Тип переменной для хранения дедлайна. Определяет максимальный диапазон таймаута и размер ОЗУ на объект. |
| `accuracy` | `uint16_t` | `1000` | Точность тиков: `1` = 1 сек, `10` = 1/10 сек, `100` = 1/100 сек, `1000` = 1/1000 сек (мс). |
| `divOptimization` | `bool` | `false` | Заменяет деление на сдвиг. Рекомендуется `true` для `accuracy != 1000` на AVR/ESP8266. |

## 📝 Примеры использования

### 1. Базовый вариант (мс, стандартный)
```cpp
MWTimeout<> relayTimer; // uint16_t, accuracy=1000 (1/1000 сек), без оптимизации

void loop() {
    if (digitalRead(BUTTON) == HIGH) {
        relayTimer.startMS(5000); // Таймаут 5 секунд
    }
    if (relayTimer.isTimeout()) {
        digitalWrite(LED, !digitalRead(LED));
    }
}
```

### 2. Экстремальная экономия ОЗУ (1 байт на таймер)
```cpp
// accuracy=100 -> тики по 1/100 сек. Диапазон: 0..255 (макс ~2.55 сек)
MWTimeout<uint8_t, 100, true> shortTimer; 

void loop() {
    shortTimer.startMS(500); // ~500 мс
    while (!shortTimer.isTimeout()) { /* опрос датчиков */ }
}
```

### 3. Максимальная скорость (замена деления)
```cpp
// accuracy=10 -> тики по 1/10 сек. Деление заменено на сдвиг.
MWTimeout<uint16_t, 10, true> fastTimer;

void loop() {
    fastTimer.start(10); // 10 тиков = 1 секунда
    if (fastTimer.isTimeout()) { /* действие */ }
}
```

## 📚 Справочник по API

### Конструкторы
| Метод | Описание |
|-------|----------|
| `MWTimeout()` | Пустой конструктор. Таймер не запущен (`ms_start = 0`). |
| `MWTimeout(int32_t time, bool refresh=true)` | Запускает таймер на `time` тиков. |
| `MWTimeout(float timeSec, bool refresh=true)` | Запускает таймер на `timeSec` секунд. |

### Управление таймером
| Метод | Описание |
|-------|----------|
| `void start(int32_t time=0, bool refresh=true)` | Запуск в тиках (`[сек/accuracy]`). |
| `void startMS(int32_t timeMSec=0, bool refresh=true)` | Запуск в миллисекундах. Автоматически масштабируется под `accuracy`. |
| `void startSec(float timeSec=0, bool refresh=true)` | Запуск в секундах (с дробной частью). |
| `void stop()` | Останавливает таймер. `isTimeout()` вернёт `true`. |
| `bool isStarted()` | `true`, если таймер был запущен (`ms_start > 0`). |

### Проверка состояния
| Метод | Возвращает |
|-------|------------|
| `bool isTimeout(int32_t extra=0, bool refresh=true)` | Таймаут истёк? `extra` добавляет тики к порогу. Также вернёт `true`, если таймер не запущен. |
| `bool isTimeoutMS(float extraMS=0, bool refresh=true)` | То же, но `extra` в мс. |
| `bool isTimeoutSec(float extraSec=0, bool refresh=true)` | То же, но `extra` в секундах. |

### Получение времени
| Метод | Возвращает |
|-------|------------|
| `int32_t timeout(int32_t extra=0, bool refresh=true)` | Тиков, прошедших с момента таймаута. `<0` если таймер ещё работает. |
| `int32_t timeoutMS(int32_t extraMS=0, bool refresh=true)` | Миллисекунд, прошедших с момента таймаута. |
| `float timeoutSec(float extraSec=0, bool refresh=true)` | Секунд, прошедших с момента таймаута. |

### Внутренние/Статические
| Метод | Описание |
|-------|----------|
| `static classTime getNowTime(bool refresh=true)` | Возвращает текущее время в тиках. При `refresh=false` возвращает закэшированное значение. |

## ⚠️ Важные замечания

1. **Кэширование времени (`refresh`)**  
   По умолчанию методы вызывают `getNowTime(true)` и обновляют системное время. В высоконагруженных циклах рекомендуется один раз вызвать 
таймер с `refresh=true` или явно `MWTimeoutBase::refresh()`, а проверки делать с `refresh=false`. Это сократит количество вызовов `millis()` и повысит детерминизм.

2. **Переполнение `millis()`**  
   Используется стандартная техника Arduino: вычитание беззнаковых типов (`ms_now - ms_start`). Переполнение (~49 дней для 32-битного счетчика) обрабатывается корректно без дополнительных флагов.

3. **Точность при `divOptimization=true`**  
   Делитель `1000 / accuracy` округляется до ближайшей степени двойки. Например, для `accuracy=100` (делитель 10) используется сдвиг на 3 (`/8`) или 4 (`/16`). Погрешность может достигать `~12–25%`. Для критичных по времени задач используйте `accuracy=1000` или отключите оптимизацию.

4. **Совместимость с C++11**  
   Используются `static_assert`, `constexpr`, `std::make_signed`, `std::round`. Убедитесь, что в настройках проекта включен стандарт `c++11` или выше (в современных Arduino/PlatformIO включено по умолчанию).

5. **Размер в ОЗУ**
   ```cpp
   sizeof(MWTimeout<uint8_t, 100, true>)   // = 1 байт
   sizeof(MWTimeout<uint16_t, 1000, false>)// = 2 байта
   sizeof(MWTimeout<uint32_t, 1000, false>)// = 4 байта
   ```
   Статическое поле `nowTime` занимает дополнительно 4 байта в секции `.bss`, но **делится между всеми экземплярами MWTimeout**.
