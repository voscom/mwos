# MWOS3

**Имя модуля:** `system` (Ядро системы)

**MWOS3** — главный класс операционной системы фреймворка MWOS. Управляет жизненным циклом модулей, хранилищами данных, портами ввода/вывода и системными событиями. Является точкой входа для инициализации и основного цикла программы.

## Описание

Класс предназначен для:
*   **Управление модулями:** Регистрация, инициализация и обновление всех подключенных модулей системы.
*   **Управление хранилищами:** Организация хранения параметров в EEPROM, NVS, StaticRAM и других типах памяти.
*   **Управление портами:** Централизованный доступ к GPIO пинам микроконтроллера (режимы, чтение, запись).
*   **Системные события:** Диспетчеризация событий (INIT, UPDATE, POWER_OFF) между модулями.
*   **Энергосбережение:** Поддержка перевода контроллера в режим глубокого сна.

**Наследование:** `MWOS3` → `MWOSParent`

**Тип модуля:** `OS` (Операционная система)

**Зависимости:**
*   Библиотека `MWArduinoLib` (абстракция Arduino)
*   Библиотека `MWOSModuleBase` (базовый класс модулей)
*   Библиотека `MWOSStorage` (абстракция хранилищ)
*   Библиотека `MWOSPins` (управление портами)
*   Платформа: Любая совместимая с Arduino (ESP32, ESP8266, STM32, AVR и др.)

## Основные возможности

*   **Автоматическая инициализация хранилищ:** Создание хранилищ EEPROM, NVS, StaticRAM при старте.
*   **Диспетчер событий:** Централизованная рассылка событий всем модулям (`dispatchEvent`).
*   **Управление памятью:** Мониторинг свободной памяти (Heap, PSRAM на ESP32).
*   **Профилирование:** Встроенные инструменты профилирования времени выполнения и утечек памяти (опционально).
*   **Глобальный экземпляр:** Создается единственный глобальный объект `mwos` для доступа из любого места.
*   **Безопасная запись:** Атомарное сохранение параметров в хранилища с контрольными суммами.
*   **Режим сна:** Поддержка Deep Sleep с пробуждением по GPIO.

## Системные конфигурации

Класс использует макросы для настройки системы перед компиляцией:

| Макрос | Значение по умолчанию | Описание |
|:---|:---|:---|
| `MWOS_DEBUG_BOUDRATE` | `115200` | Скорость отладочного порта (Serial) |
| `MWOS_PARAM_STORAGES_COUNT` | `2` или `3` | Максимальное количество хранилищ (зависит от платформы) |
| `MWOS_AUTO_STORAGES_NO` | Не определен | Запрет автоматического создания хранилищ |
| `MWOSStorageEEPROM_NO` | Не определен | Исключить хранилище EEPROM |
| `MWOSStorageStaticRAM_NO` | Не определен | Исключить хранилище StaticRAM |
| `MWOSStorageNVS_NO` | Не определен | Исключить хранилище NVS (ESP32) |

**Примечание:**
*   **Хранилища:** На ESP32 по умолчанию создаются 3 хранилища (EEPROM, StaticRAM, NVS). На AVR — 1 или 2.
*   **Глобальный объект:** В конце файла создается глобальный экземпляр `MWOS3 mwos;`.

## Конфигурация системы

```cpp
; Настройка системы осуществляется через макросы в коде
; Пример для platformio.ini или MWOSConfig.h

; Скорость отладочного порта
#define MWOS_DEBUG_BOUDRATE 115200

; Отключение автоматических хранилищ
; #define MWOS_AUTO_STORAGES_NO

; Отключение конкретных хранилищ
; #define MWOSStorageEEPROM_NO
; #define MWOSStorageStaticRAM_NO
; #define MWOSStorageNVS_NO
```

**Примечание:**
*   **Макросы:** Должны быть определены до включения заголовочного файла `MWOS3.h`.
*   **Хранилища:** Порядок добавления хранилищ важен для адресации параметров.

1.  **По умолчанию**

    Создаёт экземпляр операционной системы. Инициализирует базовые параметры и имя "MWOS3".

## Основные методы

### Жизненный цикл

*   `void start()`
    *   Запускает операционную систему.
    *   Инициализирует хранилища, загружает параметры, вызывает `EVENT_INIT` у всех модулей.
    *   Выводит отладочную информацию (память, время компиляции).

*   `void update()`
    *   Вызывается в основном цикле `loop()`.
    *   Диспетчеризирует событие `EVENT_UPDATE` всем модулям.
    *   Сохраняет измененные параметры в хранилища.

*   `void dispatchEvent(MWOSModeEvent modeEvent)`
    *   Вызывает событие `modeEvent` для всех зарегистрированных модулей.
    *   **События:** `EVENT_INIT`, `EVENT_UPDATE`, `EVENT_CHANGE`, `EVENT_POWER_OFF`, `EVENT_CONNECT`, `EVENT_DISCONNECT`.

*   `void dispatchEvent(MWOSModeEvent modeEvent, MWOSModuleBase * fromModule, MWOSParam * fromParam, uint16_t fromArrayIndex)`
    *   Вызывает событие с указанием источника изменения (модуль, параметр, индекс).

### Управление хранилищами

*   `void AddStorage(MWOSStorage * newStorage)`
    *   Добавляет новое хранилище в систему.
    *   **Параметры:** `newStorage` — указатель на объект хранилища.

*   `void initStorage(uint8_t storageType, uint16_t crc16modules)`
    *   Инициализирует хранилище, записывает контрольную сумму.
    *   Вызывается автоматически при `start()`.

*   `bool IsStorageInited(int8_t storageType)`
    *   Проверяет, было ли инициализировано хранилище.
    *   **Возвращает:** `bool` — true если хранилище активно.

*   `mwos_size saveValueToStorage(uint8_t * buff, MWOSModuleBase * module, MWOSParam * param, int16_t arrayIndex)`
    *   Сохраняет значение параметра в хранилище.
    *   Вызывается автоматически при изменении параметров.

*   `mwos_size loadValueFromStorage(uint8_t * buff, MWOSModuleBase * module, MWOSParam * param, int16_t arrayIndex, int16_t buffSize)`
    *   Загружает значение параметра из хранилища.
    *   Вызывается автоматически при инициализации.

### Управление модулями

*   `MWOSModuleBase * getModule(uint16_t module_id)`
    *   Находит модуль по его ID.
    *   **Возвращает:** Указатель на модуль или NULL.

*   `uint32_t getModulesCount()`
    *   Возвращает общее количество зарегистрированных модулей.
    *   **Возвращает:** `uint32_t` — количество модулей.

*   `uint32_t getParamsCount()`
    *   Возвращает общее количество параметров всех модулей.
    *   **Возвращает:** `uint32_t` — количество параметров.

*   `uint32_t getValuesCount()`
    *   Возвращает общее количество значений всех параметров (с учетом массивов).
    *   **Возвращает:** `uint32_t` — количество значений.

### Управление портами

*   `MWOSPins * getPin(MWOS_PIN_INT pinNum)`
    *   Находит объект порта по номеру PIN.
    *   **Возвращает:** Указатель на объект порта или NULL.

*   `bool mode(MWOS_PIN_INT pinNum, MW_PIN_MODE mode)`
    *   Устанавливает режим порта (вход, выход, ШИМ, и т.д.).
    *   **Возвращает:** `bool` — успех операции.

*   `bool writeValueDigital(MWOS_PIN_INT pinNum, bool value)`
    *   Записывает цифровое значение (HIGH/LOW) в порт.
    *   **Возвращает:** `bool` — успех операции.

*   `bool writeValuePWM(MWOS_PIN_INT pinNum, uint16_t value)`
    *   Записывает значение ШИМ в порт.
    *   **Возвращает:** `bool` — успех операции.

*   `bool writeValueDAC(MWOS_PIN_INT pinNum, uint16_t value)`
    *   Записывает аналоговое значение (ЦАП) в порт.
    *   **Возвращает:** `bool` — успех операции.

*   `uint16_t readValueAnalog(MWOS_PIN_INT pinNum)`
    *   Считывает аналоговое значение (АЦП) из порта.
    *   **Возвращает:** `uint16_t` — значение АЦП.

*   `bool readValueDigital(MWOS_PIN_INT pinNum)`
    *   Считывает цифровое значение из порта.
    *   **Возвращает:** `bool` — значение (true/false).

### Питание и сон

*   `void Sleep(int16_t pinWakeup=-1, uint8_t signalLevel=1)`
    *   Переводит контроллер в режим глубокого сна.
    *   Сохраняет все измененные параметры перед сном.
    *   **Параметры:** `pinWakeup` — пин для пробуждения, `signalLevel` — уровень сигнала для пробуждения.
    *   **Платформа:** Поддерживается на ESP32.

### Логирование и журнал

*   `void saveLogOptions()`
    *   Сохраняет признаки записи в журнал для всех параметров.
    *   Вызывается при инициализации хранилища `MWOS_STORAGE_LOG_OPTIONS`.

*   `void loadLogOptions()`
    *   Загружает признаки записи в журнал из хранилища.

*   `void saveChangedParams()`
    *   Сохраняет признаки измененных параметров перед выключением.

*   `bool toLog(MWOSModuleBase * module, MWOSParam * param, uint16_t arrayIndex)`
    *   Отправляет событие изменения параметра в модули журнала.
    *   **Возвращает:** `bool` — true если найдены модули журнала.

## Пример использования

### Базовое подключение

```cpp
#include "MWOS3.h"
#include "MWOSModule.h"

// Глобальный объект mwos создается в библиотеке
// extern MWOS3 mwos;

void setup() {
    // Запуск операционной системы
    // Инициализирует хранилища, модули, порты
    mwos.start();
}

void loop() {
    // Обновление системы
    // Вызывает EVENT_UPDATE у всех модулей
    mwos.update();
}
```

### Добавление модуля

```cpp
#include "MWOS3.h"
#include "MyCustomModule.h"

// Создаём экземпляр модуля
MyCustomModule myModule;

void setup() {
    // Регистрация модуля в системе
    mwos.addModule(&myModule);
    
    mwos.start();
}

void loop() {
    mwos.update();
}
```

### Управление портами

```cpp
void setup() {
    mwos.start();
    
    // Настройка порта на выход
    mwos.mode(2, MW_PIN_EVENT_OUTPUT);
    
    // Запись значения
    mwos.writeValueDigital(2, HIGH);
}

void loop() {
    // Чтение аналогового значения
    uint16_t val = mwos.readValueAnalog(34);
}
```

### Режим сна (ESP32)

```cpp
void loop() {
    if (needSleep) {
        // Переход в сон с пробуждением по пину 4 на уровне HIGH
        mwos.Sleep(4, HIGH);
    }
    
    mwos.update();
}
```

### Мониторинг памяти

```cpp
void setup() {
    mwos.start();
    
    // Вывод свободной памяти в лог (автоматически при start)
    // MW_LOG(F("Free mem: ")); MW_LOG_LN(mwos.getFreeMemory());
}
```

## Хранение данных

| Тип хранилища | Описание | Платформа |
|:---|:---|:---|
| `EEPROM` | Основное энергонезависимое хранилище | Все |
| `StaticRAM` | Быстрое хранилище в RAM (сохраняется при глубоком сне) | ESP32, STM32 |
| `NVS` | Non-Volatile Storage (ключ-значение) | ESP32 |
| `LOG_OPTIONS` | Специальное хранилище для флагов логирования | Все |

**Примечание:**
*   **Автоматическое создание:** Хранилища создаются автоматически при `start()`, если не определены макросы отключения.
*   **Контрольные суммы:** Каждое хранилище имеет CRC16 для проверки целостности данных.
*   **Адресация:** Параметры модулей распределяются по хранилищам последовательно.

## Важные замечания

1.  **Глобальный объект:** В системе должен быть только один экземпляр `MWOS3`. Используйте глобальный `mwos`.
2.  **Порядок инициализации:** Модули должны быть добавлены до вызова `mwos.start()`.
3.  **Хранилища:** При изменении структуры модулей (добавление/удаление параметров) может потребоваться сброс хранилищ (очистка EEPROM/NVS).
4.  **Память:** На ограниченных платформах (AVR) следите за использованием `StaticRAM` и `EEPROM`.
5.  **Сон:** Метод `Sleep()` блокирует выполнение программы. Пробуждение происходит по перезагрузке или внешнему прерыванию.
6.  **Профилирование:** Включается макросами `MWOS_PROFILER_MS` (время) и `MWOS_PROFILER_MEM` (память). Выводит предупреждения в лог при превышении лимитов.
7.  **События:** Событие `EVENT_EMPTY_STORAGES` вызывается только при первой инициализации хранилищ (для загрузки конфигов по умолчанию).
8.  **Пины:** Управление пинами осуществляется через объект `mwos.pins`. Методы `mode()`, `write...` являются обертками.
9.  **Журнал:** Для работы логирования параметров необходимо наличие модулей типа `MODULE_LOG` в системе.
10. **Watchdog:** В циклах инициализации вызывается `wdt_reset()` для предотвращения перезагрузки по watchdog.
11. **Случайные числа:** Генератор случайных чисел инициализируется при старте через `randomSeed(micros())`.
12. **Время компиляции:** Макрос `MWOS_BUILD_TIME` содержит время компиляции прошивки (для проверки актуальности).
