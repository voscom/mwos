# MWOSPins

**Имя модуля:** `pins` (базовый класс)

**MWOSPins** — базовый класс для организации блоков расширения портов микроконтроллера. Предоставляет абстрактный интерфейс для управления GPIO пинами, поддержки прерываний и сброса расширительных плат.

## Описание

Класс предназначен для:
*   **Управление портами:** Абстрактный интерфейс для настройки режимов пинов (вход, выход, ШИМ, ЦАП, АЦП).
*   **Расширение портов:** Поддержка цепочки блоков расширения для увеличения количества доступных пинов.
*   **Сброс расширителей:** Аппаратный сброс блоков расширения через специальный пин RESET.
*   **Прерывания:** Поддержка подключения и отключения прерываний на пинах.

**Наследование:** `MWOSPins` (базовый класс)

**Тип модуля:** Не является модулем (утилитарный класс)

**Зависимости:**
*   Библиотека `stdlib.h` (стандартная библиотека C)
*   Платформа: Любая совместимая с Arduino (ESP32, STM32, AVR и др.)

## Основные возможности

*   **Цепочка блоков:** Поддержка нескольких блоков портов через связный список (`next`).
*   **Диапазон пинов:** Каждый блок определяет диапазон пинов (`firstPin` - `lastPin`).
*   **Виртуальные методы:** Переопределяется в потомках для конкретной аппаратной реализации.
*   **Сброс по пину:** Аппаратный сброс блока через внешний пин RESET.
*   **Режимы портов:** 9 режимов работы (вход, выход, подтяжка, ШИМ, ЦАП, АЦП и др.).
*   **Прерывания:** Методы `attach()` и `detach()` для работы с прерываниями.
*   **Проверка пина:** Метод `isPin()` для определения принадлежности пина к блоку.

## Параметры конфигурации

Класс не использует макросы `MWOS_PARAM()` так как не является наследником `MWOSModule`. Конфигурация осуществляется через конструктор и методы:

| Параметр | Тип | Описание |
|:---|:---|:---|
| `firstPin` | `MWOS_PIN_INT` | **Первый пин блока.** Начальный номер пина в диапазоне |
| `lastPin` | `MWOS_PIN_INT` | **Последний пин блока.** Конечный номер пина в диапазоне (по умолчанию 127) |
| `pinReset` | `MWOS_PIN_INT` | **Пин сброса.** Пин микроконтроллера для аппаратного сброса блока (по умолчанию -1) |
| `next` | `MWOSPins*` | **Следующий блок.** Указатель на следующий блок расширения в цепочке |

**Примечание:**
*   **Диапазон:** По умолчанию блок занимает пины от 0 до 127.
*   **Автоматическая нумерация:** При добавлении блока через `add()` номер `firstPin` устанавливается автоматически (`lastPin + 1` предыдущего блока).
*   **Виртуальные методы:** Базовая реализация методов возвращает `false` или 0, переопределяется в потомках.

*   **Диапазон пинов:** Автоматически рассчитывается при добавлении блоков.

## Конструкторы

Класс не является шаблонным.

1.  **По умолчанию**
    ```cpp

    ```

## Основные методы

### Управление портами

*   `virtual bool mode(MW_PIN_MODE mode)` *(виртуальный)*
    *   Устанавливает режим работы порта.
    *   **Параметры:** `mode` — режим из enum `MW_PIN_MODE`.
    *   **Возвращает:** `bool` — успех операции (в базовом классе всегда `false`).

*   `virtual bool write(bool dat)` *(виртуальный)*
    *   Записывает цифровое значение в порт.
    *   **Возвращает:** `bool` — успех операции.

*   `virtual bool writePWM(uint16_t dat)` *(виртуальный)*
    *   Записывает значение ШИМ в порт.
    *   **Возвращает:** `bool` — успех операции.

*   `virtual bool writeDAC(uint16_t dat)` *(виртуальный)*
    *   Записывает аналоговое значение (ЦАП) в порт.
    *   **Возвращает:** `bool` — успех операции.

*   `virtual bool read()` *(виртуальный)*
    *   Считывает цифровое значение из порта.
    *   **Возвращает:** `bool` — значение пина.

*   `virtual uint32_t readADC()` *(виртуальный)*
    *   Считывает аналоговое значение (АЦП) из порта.
    *   **Возвращает:** `uint32_t` — значение АЦП.

### Управление цепочкой блоков

*   `MWOSPins * add(MWOSPins * newPorts)`
    *   Добавляет новый блок расширения портов в цепочку.
    *   **Параметры:** `newPorts` — указатель на новый блок.
    *   **Возвращает:** `MWOSPins*` — указатель на добавленный блок.
    *   **Примечание:** Автоматически устанавливает `firstPin` нового блока.

*   `bool isPin(MWOS_PIN_INT pinNum)`
    *   Проверяет принадлежит ли пин этому блоку.
    *   **Возвращает:** `bool` — `true` если пин в диапазоне `firstPin` - `lastPin`.

### Сброс и прерывания

*   `void setResetPin(MWOS_PIN_INT resetPin=-1)`
    *   Устанавливает пин для аппаратного сброса блока.
    *   **Параметры:** `resetPin` — номер пина микроконтроллера.

*   `void reset()`
    *   Выполняет аппаратный сброс блока через пин `pinReset`.
    *   Генерирует импульс LOW на 100 мс.

*   `virtual bool attach(voidFuncPtr funcPtr, int mode)` *(виртуальный)*
    *   Подключает прерывание для порта.
    *   **Возвращает:** `bool` — успех операции.

*   `virtual bool detach()` *(виртуальный)*
    *   Отключает прерывание для порта.
    *   **Возвращает:** `bool` — успех операции.

*   `virtual int8_t getInterrupt()` *(виртуальный)*
    *   Возвращает номер прерывания для порта.
    *   **Возвращает:** `int8_t` — номер прерывания или -1.

## Пример использования

### Базовое подключение

```cpp
#include "MWOSPins.h"

// Создаём основной блок портов
MWOSPins pins(0);

void setup() {
    // Настройка пина на выход
    pins.mode(MW_PIN_EVENT_OUTPUT);
    
    // Запись значения
    pins.write(HIGH);
}

void loop() {
    // Чтение значения
    // bool value = pins.read();
}
```

### Добавление расширителя портов

```cpp
#include "MWOSPins.h"
#include "MWOSPinsExtended.h"  // пример расширителя

// Основной блок
MWOSPins pins(0);

// Расширитель
MWOSPinsExtended extPins;

void setup() {
    // Добавляем расширитель в цепочку
    pins.add(&extPins);
    
    // Теперь пины расширителя доступны с номерами 128-255
}
```

### Настройка сброса расширителя

```cpp
#include "MWOSPins.h"

MWOSPins pins(0);

void setup() {
    // Устанавливаем пин сброса на 5
    pins.setResetPin(5);
    
    // Выполняем сброс
    pins.reset();
}
```

### Работа с прерываниями

```cpp
#include "MWOSPins.h"

MWOSPins pins(2);  // пин 2

void onInterrupt() {
    // Обработчик прерывания
}

void setup() {
    // Подключаем прерывание по нарастающему фронту
    pins.attach(onInterrupt, RISING);
}

void loop() {
    // Отключаем прерывание при необходимости
    // pins.detach();
}
```

### Проверка принадлежности пина

```cpp
#include "MWOSPins.h"

MWOSPins pins(0);  // пины 0-127
MWOSPins extPins(128);  // пины 128-255

void setup() {
    pins.add(&extPins);
    
    // Проверка
    if (pins.isPin(50)) {
        // пин 50 принадлежит основному блоку
    }
    
    if (extPins.isPin(150)) {
        // пин 150 принадлежит расширителю
    }
}
```

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `firstPin` | `RAM` | Не сохраняется, устанавливается в конструкторе |
| `lastPin` | `RAM` | Не сохраняется, по умолчанию 127 |
| `pinReset` | `RAM` | Не сохраняется, устанавливается через `setResetPin()` |
| `next` | `RAM` | Указатель на следующий блок в цепочке |

## Важные замечания

1.  **Виртуальные методы:** Базовая реализация всех методов управления портами возвращает `false` или 0. Класс предназначен для наследования.
2.  **Цепочка блоков:** Блоки соединяются через связный список (`next`). При добавлении нового блока его `firstPin` устанавливается автоматически.
3.  **Диапазон пинов:** По умолчанию блок занимает 128 пинов (0-127). Можно изменить в конструкторе или после создания.
4.  **Сброс:** Пин `pinReset` должен быть настроен как OUTPUT перед использованием. Сброс генерирует импульс LOW на 100 мс.
5.  **Прерывания:** Базовая реализация `attach()` и `detach()` возвращает `false`. Переопределяется в потомках для конкретной платформы.
6.  **Типы режимов:**
    *   `MW_PIN_EVENT_INPUT` (0) — вход
    *   `MW_PIN_EVENT_INPUT_PULLDOWN` (1) — вход с подтяжкой к земле
    *   `MW_PIN_EVENT_INPUT_PULLUP` (2) — вход с подтяжкой к питанию
    *   `MW_PIN_EVENT_INPUT_ADC` (3) — вход АЦП
    *   `MW_PIN_EVENT_OUTPUT` (4) — выход
    *   `MW_PIN_EVENT_OUTPUT_OPEN_DRAIN` (5) — выход с открытым стоком
    *   `MW_PIN_EVENT_OUTPUT_PWM` (6) — выход ШИМ
    *   `MW_PIN_EVENT_OUTPUT_DAC` (7) — выход ЦАП
    *   `MW_PIN_EVENT_DONT_INIT` (255) — не инициализировать
7.  **Callback прерываний:** Используется тип `voidFuncPtr` который может быть указателем на функцию или `std::function` (если доступен).
8.  **Проверка пина:** Метод `isPin()` устанавливает внутреннюю переменную `pin` для последующих операций.
9.  **Массив пинов:** Метод `IsPinInArray()` проверяет наличие пина в массиве PROGMEM.
10. **Наследование:** Класс `MWOSPinsMain` наследует `MWOSPins` для основного блока портов микроконтроллера.
11. **Глобальный объект:** В системе MWOS3 используется глобальный объект `mwos.pins` типа `MWOSPinsMain`.
12. **Расширители:** Для реального управления портами необходимо создавать потомков класса с конкретной реализацией методов.
