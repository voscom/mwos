## События MWOS

Для обработки события, необходимо переопределить виртуальный метод:

```c++
virtual void onEvent(MWOSModeEvent modeEvent, MWValue &data);
```
тут **modeEvent** - режим события (ниже перечислены все режимы событий), а **data** - содержит данные события. 

Необходимо определить следующие события: EVENT_INIT, EVENT_GET_VALUE, EVENT_CHANGE
```c++
String _name="Имя по умолчанию";

virtual void onEvent(MWOSModeEvent modeEvent, MWValue &data) {
   virtual void onEvent(MWOSModeEvent modeEvent, MWValue &data) {
        switch (modeEvent) {
            case EVENT_INIT: { // только при инициализации ОС
                // загрузим имя (если хранилище было пусто, то туда автоматически сохранит значение по умолчанию)
                _name=loadValueString(_name,p_name); // храним эту настройку - будем отдавать ее в EVENT_GET_VALUE
                setName(_name); // что-то делаем с _name ...
                // загрузим разрядность АЦП
                setModeADC(loadValueInt(0,p_modeADC)); // не храним эту настройку - EVENT_GET_VALUE прочитает ее из хранилища  
            } break;
            case EVENT_CHANGE: { // при изменении параметра
                if (data.param_id == id_name) {
                    _name=data.toString();
                    setName(_name);
                }
                if (data.param_id == id_modeADC) setModeADC(data.toInt());
            } break;
            case EVENT_GET_VALUE: { // при запросе параметра
                if (data.param_id == id_name) data.setString(_name.c_str());
            } break;
        }
    // ВАЖНО!!! передадим событие родителям
    // тут родительский класс    
    MWOSModule::onEvent(modeEvent,data); 
}
```

* Если были неинициализированные хранилища (первый запуск контроллера), вызывается **EVENT_INIT**

### EVENT_EMPTY_STORAGES
Вызывается в конце mwos.start(), если было обнаружено пустое хранилище.
Модули могут сохранить свои параметры в это хранилище. При этом хранилище будет заполнено значениями по умолчанию.

data.param_index - битовая маска номеров пустого хранилища. Например 0101 - значит, пустые хранилища 0 и 2.

### EVENT_INIT
Вызывается в конце mwos.start(), когда OS уже стартовала.
Модули могут загрузить свои параметры из хранилища (loadValue) и инициализировать оборудование.

### EVENT_POWER_OFF
Вызывается перед выключение питания или перезагрузкой.
Модули могут завершить операции и сохранить данные.

### EVENT_UPDATE
Вызывается каждый тик операционной системы.
Модуль может обрабатывать свой функционал в реальном времени. 
Но необходимо самостоятельно отслеживать длительность такта обработки.

### EVENT_CHANGE
Вызывается после изменения значения параметра.
Модуль должен обработать новое значение data.getValue()

data - содержит новое значение и адресата.

### EVENT_GET_VALUE
Вызывается если OS или другому модулю нужно запросить значение параметра.
Модуль должен изменить значение data на актуальное - data.setValue(актуальное значение) 

data - содержит последнее значение (или значение по умолчанию) и адресата.

### EVENT_SET_VALUE
Вызывается если OS или другой модуль изменяет значение параметра. 
Если не обработать - новое значение будет автоматом сохранено в хранилище и вызвано событие EVENT_CHANGE. 
Модуль может самостоятельно обработать входящее значение и изменить или отменить запись в хранилище.
Для обычной обработки изменений значения, рекомендуется использовать событие EVENT_CHANGE.

data - содержит новое значение и адресата.

### EVENT_LOG_VALUE
Вызывается если какой-то модуль хочет сохранить свои данные в журнал.

data - содержит значение для журнала и адрес модуля отправителя.

### EVENT_TIME_CHANGE
Вызывается  изменения времени (что-бы модули могли это учесть).

data - содержит ссылку на модуль времени.

### EVENT_CONNECT
Вызывается после подключении к серверу.

data - содержит ссылку на сетевой модуль.

### EVENT_DISCONNECT
Вызывается после отключения от сервера.

data - содержит ссылку на сетевой модуль.
