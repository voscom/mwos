# MWOSPinsMain

**Имя модуля:** `pins` (Системный класс)

**MWOSPinsMain** — основной класс управления портами микроконтроллера в системе MWOS. Реализует физическое взаимодействие с GPIO пинами, проверяя допустимость режимов для конкретных пинов на основе конфигурации платформы.

## Описание

Класс предназначен для:
*   **Управление GPIO:** Настройка режимов работы пинов (вход, выход, ШИМ, ЦАП, АЦП).
*   **Валидация пинов:** Проверка возможности использования пина в заданном режиме через списки допустимых пинов.
*   **Прерывания:** Поддержка подключения и отключения аппаратных прерываний.
*   **Абстракция платформы:** Унифицированный интерфейс для ESP32, ESP8266, STM32, AVR и других платформ.

**Наследование:** `MWOSPinsMain` → `MWOSPins`

**Тип модуля:** Системный класс (не является модулем MWOS)

**Зависимости:**
*   Библиотека `Arduino.h` (Arduino Core)
*   Библиотека `FunctionalInterrupt.h` (ESP32/ESP8266)
*   Платформа: Любая совместимая с Arduino (ESP32, ESP8266, STM32, AVR и др.)

## Основные возможности

*   **Проверка режимов:** Каждый режим (INPUT, OUTPUT, PWM и др.) имеет свой список допустимых пинов в PROGMEM.
*   **Поддержка ШИМ и ЦАП:** Методы `writePWM()` и `writeDAC()` для аналогового управления.
*   **Аналоговый вход:** Метод `readADC()` для чтения значений АЦП.
*   **Прерывания:** Методы `attach()` и `detach()` для работы с внешними прерываниями.
*   **Глобальный доступ:** Используется через объект `mwos.pins` в ядре системы.
*   **Безопасность:** Возвращает `false` при попытке настроить пин в недопустимом режиме.

## Конфигурация системы

Класс использует макросы и массивы в PROGMEM для настройки допустимых пинов. Конфигурация осуществляется через `platformio.ini` или заголовочные файлы:

| Макрос | Описание | Пример |
|:---|:---|:---|
| `PINS_INPUT` | Список пинов, поддерживающих режим входа | `15,2,14,16,17...` |
| `PINS_OUTPUT` | Список пинов, поддерживающих режим выхода | `15,2,14,16,17...` |
| `PINS_ADC` | Список пинов, поддерживающих АЦП | `25,26,27,32,33...` |
| `PINS_PWM` | Список пинов, поддерживающих ШИМ | `15,2,14,16,17...` |
| `PINS_DAC` | Список пинов, поддерживающих ЦАП | `25,26` (ESP32) |
| `PINS_INPUT_PULLUP` | Список пинов с подтяжкой к питанию | `15,2,14...` |
| `PINS_INPUT_PULLDOWN` | Список пинов с подтяжкой к земле | `15,14,16...` |
| `PINS_OPEN_DRAIN` | Список пинов с открытым стоком | `15,14,16...` |

**Примечание:**
*   **PROGMEM:** Списки пинов хранятся в flash-памяти (PROGMEM) для экономии RAM.
*   **Обязательная настройка:** Для корректной работы необходимо определить макросы `PINS_INPUT`, `PINS_OUTPUT` и другие в `platformio.ini` через `build_flags`.
*   **По умолчанию:** Если макросы не определены, используются стандартные значения (например, `LED_BUILTIN` для `PINS_LED`).

## Конфигурация в platformio.ini

```ini
; Пример настройки пинов для ESP32 в platformio.ini
[env:esp32dev]
platform = espressif32
board = esp32dev
framework = arduino

build_flags =
    -D PINS_INPUT=15,2,14,16,17,5,18,19,21,3,1,22,23,13,12,14,27,26,25,33,32,34,35,36,39
    -D PINS_OUTPUT=15,2,14,16,17,5,18,19,21,3,1,22,23,13,12,14,27,26,25,33,32
    -D PINS_ADC=25,26,27,32,33,12,13,14,15,2,4,34,35,36,39
    -D PINS_PWM=15,2,14,16,17,5,18,19,21,3,1,22,23,13,12,14,27,26,25,33,32
    -D PINS_DAC=25,26
    -D PINS_INPUT_PULLUP=15,2,14,16,17,18,19,21,3,1,22,23,13,12,14,27,26,25,33,32
    -D PINS_LED=2
```

**Примечание:**
*   **Формат:** Список пинов перечисляется через запятую без пробелов.
*   **Безопасность:** Попытка использовать пин вне списка вернет `false` в методах настройки.
*   **Платформа:** Наборы пинов зависят от конкретного микроконтроллера.

## Конструкторы

Класс не является шаблонным.

1.  **По умолчанию**
    ```cpp
    MWOSPinsMain();
    ```
    Создаёт экземпляр основного блока портов. Диапазон пинов устанавливается в 0-127.

## Основные методы

### Управление режимами

*   `virtual bool mode(MW_PIN_MODE mode)` *(переопределён)*
    *   Устанавливает режим работы порта.
    *   Проверяет пин на наличие в списке допустимых для данного режима.
    *   **Параметры:** `mode` — режим из enum `MW_PIN_MODE`.
    *   **Возвращает:** `bool` — `true` если режим установлен успешно.

### Цифровое управление

*   `virtual bool write(bool dat)` *(переопределён)*
    *   Записывает цифровое значение (HIGH/LOW) в порт.
    *   **Возвращает:** `bool` — успех операции.

*   `virtual bool read()` *(переопределён)*
    *   Считывает цифровое значение из порта.
    *   **Возвращает:** `bool` — значение пина.

### Аналоговое управление

*   `virtual bool writePWM(uint16_t dat)` *(переопределён)*
    *   Записывает значение ШИМ в порт.
    *   **Возвращает:** `bool` — успех операции.

*   `virtual bool writeDAC(uint16_t dat)` *(переопределён)*
    *   Записывает аналоговое значение (ЦАП) в порт.
    *   **Возвращает:** `bool` — успех операции.

*   `virtual uint32_t readADC()` *(переопределён)*
    *   Считывает аналоговое значение (АЦП) из порта.
    *   **Возвращает:** `uint32_t` — значение АЦП (0-4095 на ESP32).

### Прерывания

*   `virtual int8_t getInterrupt()` *(переопределён)*
    *   Возвращает номер прерывания для порта.
    *   **Возвращает:** `int8_t` — номер прерывания или -1.

*   `virtual bool attach(voidFuncPtr funcPtr, int mode)` *(переопределён)*
    *   Подключает прерывание для порта.
    *   **Параметры:** `funcPtr` — функция обработчика, `mode` — режим (RISING, FALLING, CHANGE).
    *   **Возвращает:** `bool` — успех операции.

*   `virtual bool detach()` *(переопределён)*
    *   Отключает прерывание для порта.
    *   **Возвращает:** `bool` — успех операции.

## Пример использования

### Базовое подключение

```cpp
#include "MWOS3.h"

// Глобальный объект mwos содержит экземпляр pins
// MWOSPinsMain mwos.pins;

void setup() {
    mwos.start();
    
    // Настройка пина 2 на выход (если разрешено в конфиге)
    mwos.mode(2, MW_PIN_EVENT_OUTPUT);
}

void loop() {
    // Мигание светодиодом
    mwos.writeValueDigital(2, HIGH);
    delay(500);
    mwos.writeValueDigital(2, LOW);
    delay(500);
}
```

### Чтение аналогового значения

```cpp
void setup() {
    mwos.start();
    
    // Настройка пина 34 на вход АЦП
    mwos.mode(34, MW_PIN_EVENT_INPUT_ADC);
}

void loop() {
    // Чтение значения АЦП
    uint16_t val = mwos.readValueAnalog(34);
    
    // val будет в диапазоне 0-4095 (ESP32)
}
```

### Подключение прерывания

```cpp
// Глобальная переменная для счётчика
volatile int interruptCounter = 0;

// Обработчик прерывания
void IRAM_ATTR onInterrupt() {
    interruptCounter++;
}

void setup() {
    mwos.start();
    
    // Подключение прерывания на пин 4 по нарастающему фронту
    MWOSPins * pinObj = mwos.getPin(4);
    if (pinObj != NULL) {
        pinObj->attach(onInterrupt, RISING);
    }
}

void loop() {
    // Обработка счётчика
    if (interruptCounter > 0) {
        interruptCounter = 0;
    }
}
```

### Управление ШИМ

```cpp
void setup() {
    mwos.start();
    
    // Настройка пина 5 на ШИМ
    mwos.mode(5, MW_PIN_EVENT_OUTPUT_PWM);
}

void loop() {
    // Плавное изменение яркости
    for (int i = 0; i < 255; i++) {
        mwos.writeValuePWM(5, i);
        delay(10);
    }
}
```

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `PinsINPUT` | `PROGMEM` | Список пинов входа (flash-память) |
| `PinsOUTPUT` | `PROGMEM` | Список пинов выхода (flash-память) |
| `PinsADC` | `PROGMEM` | Список пинов АЦП (flash-память) |
| `PinsPWM` | `PROGMEM` | Список пинов ШИМ (flash-память) |
| `PinsDAC` | `PROGMEM` | Список пинов ЦАП (flash-память) |

**Примечание:**
*   **PROGMEM:** Все массивы пинов хранятся в flash-памяти для экономии оперативной памяти.
*   **Константы:** Значения определяются при компиляции и не изменяются в runtime.

## Важные замечания

1.  **Валидация пинов:** Метод `mode()` проверяет наличие пина в соответствующем массиве (например, `PinsOUTPUT` для режима выхода). Если пина нет в списке — метод вернет `false`.
2.  **Настройка в platformio.ini:** Обязательно задайте списки пинов через `build_flags` в `platformio.ini`, иначе проверка может не работать корректно.
3.  **Платформенные особенности:**
    *   **ESP32:** Поддерживает `INPUT_PULLDOWN`, `DAC`, `TOUCH`.
    *   **ESP8266:** Поддерживает `INPUT_PULLDOWN_16` только для пина 16.
    *   **AVR:** Не поддерживает `DAC`, `TOUCH`.
4.  **Прерывания:** На ESP32/ESP8266 используется `attachInterrupt()`. Функция обработчика должна быть `IRAM_ATTR` на ESP32.
5.  **Глобальный объект:** В системе используется единственный экземпляр `mwos.pins` типа `MWOSPinsMain`.
6.  **Диапазон пинов:** По умолчанию блок занимает пины 0-127. Для расширителей портов используются дополнительные блоки `MWOSPins`.
7.  **ЦАП (DAC):** На ESP32 доступен только на пинах 25 и 26. На других платформах может эмулироваться через ШИМ или отсутствовать.
8.  **АЦП (ADC):** На ESP32 значение 0-4095 (12 бит). На AVR значение 0-1023 (10 бит).
9.  **Логирование:** При ошибке настройки режима выводится сообщение в лог с номером пина и режимом.
10. **Режимы:**
    *   `MW_PIN_EVENT_INPUT` (0) — вход
    *   `MW_PIN_EVENT_INPUT_PULLDOWN` (1) — вход с подтяжкой к земле
    *   `MW_PIN_EVENT_INPUT_PULLUP` (2) — вход с подтяжкой к питанию
    *   `MW_PIN_EVENT_INPUT_ADC` (3) — вход АЦП
    *   `MW_PIN_EVENT_OUTPUT` (4) — выход
    *   `MW_PIN_EVENT_OUTPUT_OPEN_DRAIN` (5) — выход с открытым стоком
    *   `MW_PIN_EVENT_OUTPUT_PWM` (6) — выход ШИМ
    *   `MW_PIN_EVENT_OUTPUT_DAC` (7) — выход ЦАП
11. **Макросы:** Если макрос `PINS_NOT_DAC` определен, метод `writeDAC()` не вызывает `dacWrite()`.
12. **Расширители:** Для подключения внешних расширителей портов используйте метод `mwos.pins.add(new MWOSPinsExtended())`.
