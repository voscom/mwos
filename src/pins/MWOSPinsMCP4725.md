# MWOSPinsMCP4725

**Имя модуля:** `pins` (Расширитель портов)

**MWOSPinsMCP4725** — класс расширения портов для работы с внешним ЦАП (цифро-аналоговым преобразователем) на базе микросхемы MCP4725 через интерфейс I2C. Позволяет добавить один аналоговый выход к системе управления портами MWOS.

## Описание

Класс предназначен для:
*   **Управление ЦАП:** Преобразование цифровых значений (0-4095) в аналоговое напряжение через MCP4725.
*   **I2C интерфейс:** Подключение через шину I2C с настраиваемым адресом устройства.
*   **Интеграция с MWOS:** Встраивание в систему управления портами MWOS как расширитель.
*   **Быстрая запись:** Поддержка режима Fast Mode для записи значений без сохранения в EEPROM чипа.

**Наследование:** `MWOSPinsMCP4725` → `MWOSPins`

**Тип модуля:** Расширитель портов (не является модулем MWOS)

**Зависимости:**
*   Библиотека `Wire.h` (I2C интерфейс)
*   Библиотека `MWOSPins` (базовый класс портов)
*   Библиотека `MWStreamI2C` (I2C поток)
*   Платформа: Любая совместимая с Arduino с поддержкой I2C (ESP32, ESP8266, STM32, AVR и др.)

## Основные возможности

*   **12-битный ЦАП:** Разрешение 4096 уровней (0-4095).
*   **Настраиваемый адрес:** Поддержка 8 адресов I2C (0x60-0x67) через выводы адреса на чипе.
*   **Автоматическая нумерация:** Автоматическое определение номера пина в системе MWOS.
*   **Проверка подключения:** Метод `isConnected()` для проверки наличия устройства на шине I2C.
*   **Цифровой режим:** Поддержка записи логических значений (HIGH/LOW) через эмуляцию ЦАП.
*   **Быстрая запись:** Метод `writeFastMode()` для записи без сохранения в энергонезависимую память чипа.

## Параметры конфигурации

Класс не использует макросы `MWOS_PARAM()` так как является расширителем портов, а не модулем MWOS. Конфигурация осуществляется через конструктор:

| Параметр | Тип | Описание |
|:---|:---|:---|
| `twoWire` | `TwoWire*` | **Интерфейс I2C.** Указатель на объект Wire для связи |
| `devHardwareAddress` | `uint8_t` | **Аппаратный адрес.** Адрес чипа MCP4725 (0-7, реальный адрес 0x60-0x67) |
| `firstPin` | `int16_t` | **Первый пин.** Номер пина в адресном пространстве MWOS (по умолчанию 4725) |

**Примечание:**
*   **Адрес I2C:** Реальный адрес вычисляется как `devHardwareAddress + 0x60` (диапазон 0x60-0x67).
*   **Номер пина:** По умолчанию используется 4725, но может быть изменён для избежания конфликтов с другими расширителями.
*   **Один канал:** MCP4725 имеет один канал ЦАП, поэтому блок занимает один пин (`firstPin = lastPin`).

## Конфигурация модуля

```cpp
; класс MWOSPinsMCP4725 не имеет параметров конфигурации через MWOS_PARAM
; Настройка осуществляется через код:

// Создание расширителя ЦАП
// Wire - интерфейс I2C
// 0 - аппаратный адрес (0x60)
// 4725 - номер пина в системе MWOS
MWOSPinsMCP4725 dac(&Wire, 0, 4725);

// Добавление расширителя в систему
// mwos.pins.add(&dac);
```

**Примечание:**
*   **Параметры:** Класс не использует систему параметров MWOS.
*   **Настройка:** Все настройки задаются программно в конструкторе.
*   **Адресное пространство:** Пин 4725 зарезервирован для MCP4725 по умолчанию.

## Конструкторы

Класс не является шаблонным.

1.  **С указанием параметров**
    ```cpp
    MWOSPinsMCP4725(TwoWire * twoWire, uint8_t devHardwareAddress=0, int16_t _firstPin=4725);
    ```
    Создаёт экземпляр расширителя с указанием интерфейса I2C, адреса устройства и номера пина.
    *   **Параметры:**
        *   `twoWire` — указатель на объект Wire (Wire, Wire1 и др.)
        *   `devHardwareAddress` — адрес чипа (0-7, по умолчанию 0 = 0x60)
        *   `_firstPin` — номер пина в системе MWOS (по умолчанию 4725)

## Основные методы

### Управление портами

*   `virtual bool mode(MW_PIN_MODE mode)` *(переопределён)*
    *   Устанавливает режим работы порта.
    *   Проверяет диапазон адреса I2C (0x60-0x67) и наличие устройства на шине.
    *   **Возвращает:** `bool` — `true` если устройство подключено.

*   `virtual bool writeDAC(uint16_t dat)` *(переопределён)*
    *   Записывает аналоговое значение в ЦАП.
    *   **Параметры:** `dat` — значение 0-4095 (12 бит).
    *   **Возвращает:** `bool` — успех операции.

*   `virtual bool write(bool dat)` *(переопределён)*
    *   Записывает логическое значение (эмуляция через ЦАП).
    *   `true` = 4095 (максимальное напряжение), `false` = 0 (минимальное).
    *   **Возвращает:** `bool` — успех операции.

### Проверка подключения

*   `bool isConnected()`
    *   Проверяет наличие устройства MCP4725 на шине I2C.
    *   **Возвращает:** `bool` — `true` если устройство ответило.

## Пример использования

### Базовое подключение

```cpp
#include "MWOS3.h"
#include "MWOSPinsMCP4725.h"

// Создаём расширитель ЦАП
// Wire - интерфейс I2C
// 0 - адрес чипа (0x60)
// 4725 - номер пина в системе MWOS
MWOSPinsMCP4725 dac(&Wire, 0, 4725);

void setup() {
    Wire.begin();  // Инициализация I2C
    mwos.start();
    
    // Добавляем расширитель в систему портов
    mwos.pins.add(&dac);
    
    // Настройка пина (проверит подключение)
    mwos.mode(4725, MW_PIN_EVENT_OUTPUT_DAC);
}

void loop() {
    // Установка напряжения (0-4095)
    mwos.writeValueDAC(4725, 2048);  // 50% от максимального напряжения
    delay(1000);
    
    mwos.writeValueDAC(4725, 0);  // 0V
    delay(1000);
}
```

### Проверка подключения устройства

```cpp
#include "MWOS3.h"
#include "MWOSPinsMCP4725.h"

MWOSPinsMCP4725 dac(&Wire, 0, 4725);

void setup() {
    Wire.begin();
    mwos.start();
    mwos.pins.add(&dac);
    
    // Проверка подключения
    if (dac.isConnected()) {
        Serial.println("MCP4725 connected!");
    } else {
        Serial.println("MCP4725 not found!");
    }
}
```

### Несколько расширителей MCP4725

```cpp
#include "MWOS3.h"
#include "MWOSPinsMCP4725.h"

// Первый ЦАП на адресе 0x60 (пин 4725)
MWOSPinsMCP4725 dac1(&Wire, 0, 4725);

// Второй ЦАП на адресе 0x61 (пин 4726)
MWOSPinsMCP4725 dac2(&Wire, 1, 4726);

void setup() {
    Wire.begin();
    mwos.start();
    
    // Добавляем оба расширителя
    mwos.pins.add(&dac1);
    mwos.pins.add(&dac2);
}

void loop() {
    // Управление разными ЦАП
    mwos.writeValueDAC(4725, 1024);  // Первый ЦАП
    mwos.writeValueDAC(4726, 3072);  // Второй ЦАП
}
```

### Использование с разными шинами I2C

```cpp
#include "MWOS3.h"
#include "MWOSPinsMCP4725.h"

// ESP32 с двумя шинами I2C
TwoWire I2C_0 = TwoWire(0);
TwoWire I2C_1 = TwoWire(1);

MWOSPinsMCP4725 dac1(&I2C_0, 0, 4725);
MWOSPinsMCP4725 dac2(&I2C_1, 0, 4726);

void setup() {
    I2C_0.begin(21, 22);  // SDA=21, SCL=22
    I2C_1.begin(25, 26);  // SDA=25, SCL=26
    
    mwos.start();
    mwos.pins.add(&dac1);
    mwos.pins.add(&dac2);
}
```

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `nowValue` | `RAM` | Текущее значение ЦАП, не сохраняется |
| `_addr` | `RAM` | Адрес I2C, устанавливается в конструкторе |
| `_wire` | `RAM` | Указатель на объект Wire, не сохраняется |
| `firstPin` | `RAM` | Номер пина, устанавливается в конструкторе |
| `lastPin` | `RAM` | Последний пин (равен firstPin) |

## Важные замечания

1.  **Адрес I2C:** MCP4725 имеет 3 вывода адреса (A0, A1, A2) для настройки адреса от 0x60 до 0x67. Параметр `devHardwareAddress` задаётся как 0-7.
2.  **Проверка подключения:** Метод `mode()` автоматически проверяет наличие устройства на шине I2C через `isConnected()`. Если устройство не найдено — вернёт `false`.
3.  **Разрешение ЦАП:** MCP4725 имеет 12-битное разрешение (0-4095). Значения больше 4095 автоматически ограничиваются до 4095.
4.  **Быстрый режим:** Метод `writeFastMode()` записывает значение без сохранения в EEPROM чипа. Это быстрее, но значение сбрасывается при отключении питания.
5.  **Напряжение выхода:** Выходное напряжение зависит от напряжения питания VDD (обычно 3.3V или 5V). Выход = (значение/4095) × VDD.
6.  **Подтяжка I2C:** Линии SDA и SCL требуют подтягивающих резисторов (обычно 4.7kΩ) к питанию.
7.  **Длина шины:** Для надёжной работы длина проводов I2C не должна превышать 1-2 метра без усилителей.
8.  **Номер пина:** По умолчанию используется 4725. При использовании нескольких расширителей необходимо задать уникальные номера пинов.
9.  **Режим вывода:** Метод `mode()` всегда возвращает `true` если устройство подключено, независимо от режима (ЦАП не имеет режимов входа).
10. **Эмуляция цифрового выхода:** Метод `write(true)` устанавливает максимальное значение (4095), `write(false)` — минимальное (0).
11. **Два провода:** Для подключения требуется только 2 провода (SDA, SCL) плюс питание и земля.
12. **Совместимость:** Работает с любым контроллером, поддерживающим библиотеку Wire и I2C.
