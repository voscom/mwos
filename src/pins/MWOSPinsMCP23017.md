# MWOSPinsMCP23017

**Имя модуля:** `pins` (Расширитель портов)

**MWOSPinsMCP23017** — класс расширения портов для работы с внешним 16-портовым расширителем GPIO на базе микросхемы MCP23017 через интерфейс I2C. Позволяет добавить 16 цифровых входов/выходов к системе управления портами MWOS.

## Описание

Класс предназначен для:
*   **Расширение GPIO:** Добавление 16 дополнительных цифровых портов через I2C.
*   **Управление режимами:** Поддержка режимов входа, выхода и подтяжки к питанию.
*   **I2C интерфейс:** Подключение через шину I2C с настраиваемым адресом устройства.
*   **Интеграция с MWOS:** Встраивание в систему управления портами MWOS как расширитель.

**Наследование:** `MWOSPinsMCP23017` → `MWOSPins`

**Тип модуля:** Расширитель портов (не является модулем MWOS)

**Зависимости:**
*   Библиотека `Wire.h` (I2C интерфейс)
*   Библиотека `MWOSPins` (базовый класс портов)
*   Библиотека `MWStreamI2C` (I2C поток)
*   Платформа: Любая совместимая с Arduino с поддержкой I2C (ESP32, ESP8266, STM32, AVR и др.)

## Основные возможности

*   **16 портов GPIO:** Два банка по 8 портов (A и B).
*   **Настраиваемый адрес:** Поддержка 8 адресов I2C (0x20-0x27) через выводы адреса на чипе.
*   **Автоматическая нумерация:** Автоматическое определение номера пина в системе MWOS.
*   **Проверка подключения:** Метод `mode()` проверяет доступность устройства на шине I2C.
*   **Подтяжка к питанию:** Поддержка внутренних pull-up резисторов.
*   **Кеширование значений:** Локальное кеширование состояния портов для оптимизации.
*   **Пошаговое управление:** Запись и чтение отдельных битов через регистры.

## Параметры конфигурации

Класс не использует макросы `MWOS_PARAM()` так как является расширителем портов, а не модулем MWOS. Конфигурация осуществляется через конструктор:

| Параметр | Тип | Описание |
|:---|:---|:---|
| `twoWire` | `TwoWire*` | **Интерфейс I2C.** Указатель на объект Wire для связи |
| `devHardwareAddress` | `uint8_t` | **Аппаратный адрес.** Адрес чипа MCP23017 (0-7, реальный адрес 0x20-0x27) |
| `firstPin` | `int16_t` | **Первый пин.** Номер пина в адресном пространстве MWOS (по умолчанию 0) |

**Примечание:**
*   **Адрес I2C:** Реальный адрес вычисляется как `devHardwareAddress + 0x20` (диапазон 0x20-0x27).
*   **Номер пина:** По умолчанию используется 0, но может быть изменён для избежания конфликтов с другими расширителями.
*   **Диапазон пинов:** Блок занимает 16 пинов (`firstPin` - `firstPin+15`).

## Конфигурация модуля

```cpp
; класс MWOSPinsMCP23017 не имеет параметров конфигурации через MWOS_PARAM
; Настройка осуществляется через код:

// Создание расширителя портов
// Wire - интерфейс I2C
// 0 - аппаратный адрес (0x20)
// 0 - номер первого пина в системе MWOS
MWOSPinsMCP23017 extPins(&Wire, 0, 0);

// Добавление расширителя в систему
// mwos.pins.add(&extPins);
```

**Примечание:**
*   **Параметры:** Класс не использует систему параметров MWOS.
*   **Настройка:** Все настройки задаются программно в конструкторе.
*   **Адресное пространство:** Пины занимают диапазон от `firstPin` до `firstPin+15`.

## Конструкторы

Класс не является шаблонным.

1.  **С указанием параметров**
    ```cpp
    MWOSPinsMCP23017(TwoWire * twoWire, uint8_t devHardwareAddress, int16_t _firstPin=0);
    ```
    Создаёт экземпляр расширителя с указанием интерфейса I2C, адреса устройства и номера первого пина.
    *   **Параметры:**
        *   `twoWire` — указатель на объект Wire (Wire, Wire1 и др.)
        *   `devHardwareAddress` — адрес чипа (0-7, по умолчанию 0 = 0x20)
        *   `_firstPin` — номер первого пина в системе MWOS (по умолчанию 0)

## Основные методы

### Управление портами

*   `virtual bool mode(MW_PIN_MODE mode)` *(переопределён)*
    *   Устанавливает режим работы порта (вход/выход/подтяжка).
    *   Читает и записывает регистры IODIR, GPPU через I2C.
    *   **Возвращает:** `bool` — `true` если операция успешна.

*   `virtual bool write(bool dat)` *(переопределён)*
    *   Записывает цифровое значение в порт.
    *   **Возвращает:** `bool` — успех операции.

*   `virtual bool read()` *(переопределён)*
    *   Считывает цифровое значение из порта.
    *   **Возвращает:** `bool` — значение пина.

### Внутренние методы

*   `bool writeRegister(uint8_t * word, uint8_t firstReg, bool dat)` *(private)*
    *   Записывает значение в регистровую пару MCP23017.
    *   **Параметры:** `word` — буфер значения, `firstReg` — первый регистр, `dat` — значение бита.
    *   **Возвращает:** `bool` — успех записи.

*   `bool readRegister(uint8_t * word, uint8_t firstReg)` *(private)*
    *   Считывает значение из регистровой пары MCP23017.
    *   **Возвращает:** `bool` — значение бита.

## Пример использования

### Базовое подключение

```cpp
#include "MWOS3.h"
#include "MWOSPinsMCP23017.h"

// Создаём расширитель портов
// Wire - интерфейс I2C
// 0 - адрес чипа (0x20)
// 0 - номер первого пина в системе MWOS
MWOSPinsMCP23017 extPins(&Wire, 0, 0);

void setup() {
    Wire.begin();  // Инициализация I2C
    mwos.start();
    
    // Добавляем расширитель в систему портов
    mwos.pins.add(&extPins);
    
    // Настройка пина 0 на выход (реальный пин MCP23017 GPA0)
    mwos.mode(0, MW_PIN_EVENT_OUTPUT);
}

void loop() {
    // Установка значения на пине 0
    mwos.writeValueDigital(0, HIGH);
    delay(500);
    
    mwos.writeValueDigital(0, LOW);
    delay(500);
}
```

### Несколько расширителей MCP23017

```cpp
#include "MWOS3.h"
#include "MWOSPinsMCP23017.h"

// Первый расширитель на адресе 0x20 (пины 0-15)
MWOSPinsMCP23017 ext1(&Wire, 0, 0);

// Второй расширитель на адресе 0x21 (пины 16-31)
MWOSPinsMCP23017 ext2(&Wire, 1, 16);

void setup() {
    Wire.begin();
    mwos.start();
    
    // Добавляем оба расширителя
    mwos.pins.add(&ext1);
    mwos.pins.add(&ext2);
}

void loop() {
    // Управление разными расширителями
    mwos.writeValueDigital(0, HIGH);   // Первый расширитель, пин 0
    mwos.writeValueDigital(16, LOW);   // Второй расширитель, пин 0
}
```

### Использование с разными шинами I2C

```cpp
#include "MWOS3.h"
#include "MWOSPinsMCP23017.h"

// ESP32 с двумя шинами I2C
TwoWire I2C_0 = TwoWire(0);
TwoWire I2C_1 = TwoWire(1);

MWOSPinsMCP23017 ext1(&I2C_0, 0, 0);
MWOSPinsMCP23017 ext2(&I2C_1, 0, 32);

void setup() {
    I2C_0.begin(21, 22);  // SDA=21, SCL=22
    I2C_1.begin(25, 26);  // SDA=25, SCL=26
    
    mwos.start();
    mwos.pins.add(&ext1);
    mwos.pins.add(&ext2);
}
```

### Режим входа с подтяжкой

```cpp
#include "MWOS3.h"
#include "MWOSPinsMCP23017.h"

MWOSPinsMCP23017 extPins(&Wire, 0, 0);

void setup() {
    Wire.begin();
    mwos.start();
    mwos.pins.add(&extPins);
    
    // Настройка пина 1 на вход с подтяжкой к питанию
    mwos.mode(1, MW_PIN_EVENT_INPUT_PULLUP);
}

void loop() {
    // Чтение значения с пина 1
    bool value = mwos.readValueDigital(1);
    
    if (value) {
        // Кнопка нажата (подтяжка к питанию)
    }
}
```

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `nowValue` | `RAM` | Текущее кешированное значение портов, не сохраняется |
| `i2c` | `RAM` | Указатель на объект I2C потока, не сохраняется |
| `firstPin` | `RAM` | Номер первого пина, устанавливается в конструкторе |
| `lastPin` | `RAM` | Последний пин (firstPin+15) |
| `pinReset` | `RAM` | Пин сброса расширителя (опционально) |

## Важные замечания

1.  **Адрес I2C:** MCP23017 имеет 3 вывода адреса (A0, A1, A2) для настройки адреса от 0x20 до 0x27. Параметр `devHardwareAddress` задаётся как 0-7.
2.  **Диапазон пинов:** Каждый блок занимает 16 пинов. При использовании нескольких расширителей необходимо задать уникальные диапазоны через `firstPin`.
3.  **Режимы портов:** Поддерживаются только `MW_PIN_EVENT_INPUT`, `MW_PIN_EVENT_INPUT_PULLUP`, `MW_PIN_EVENT_OUTPUT`. Другие режимы (PWM, DAC, ADC) не поддерживаются.
4.  **Подтяжка:** Поддерживается только pull-up к питанию. Pull-down не поддерживается аппаратно MCP23017.
5.  **I2C скорость:** Для надёжной работы рекомендуется использовать стандартную скорость I2C (100 kHz) или быструю (400 kHz).
6.  **Подтяжка I2C:** Линии SDA и SCL требуют подтягивающих резисторов (обычно 4.7kΩ) к питанию.
7.  **Длина шины:** Для надёжной работы длина проводов I2C не должна превышать 1-2 метра без усилителей.
8.  **Регистры:** Класс работает с регистрами порта A (IODIRA, GPPUA, GPIOA). Порт B не используется в данной реализации.
9.  **Кеширование:** Значения портов кешируются в `nowValue` для оптимизации операций записи/чтения.
10. **Сброс:** При наличии пина `pinReset` можно выполнить аппаратный сброс расширителя через метод `reset()`.
11. **Прерывания:** В данной реализации прерывания не поддерживаются (методы `attach()` и `detach()` возвращают `false`).
12. **Совместимость:** Работает с любым контроллером, поддерживающим библиотеку Wire и I2C.
