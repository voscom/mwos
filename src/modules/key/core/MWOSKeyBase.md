# MWOSKeyBase

**Имя модуля:** `key`

**MWOSKeyBase** — базовый шаблонный класс для модулей управления ключами/реле. Предназначен только для наследования.

## Описание

Класс предназначен для:
*   **Управление ключами/реле:** Базовая инфраструктура для модулей с бинарными выходами
*   **Множественные ключи:** Поддержка до `keysCount` независимых ключей в одном модуле
*   **Режимы работы:** Инверсия, чекбокс (взаимное исключение), принудительное отключение
*   **Сохранение состояния:** Автоматическое сохранение состояния ключей при выключении

**Наследование:** `MWOSKeyBase<keysCount>` → `MWOSModule` → `MWOSModuleBase`

**Тип модуля:** `MODULE_KEY`

**Зависимости:**
*   Библиотека `MWBitsMaskStat.h` (битовая маска состояний)
*   Библиотека `MWTimeout.h` (таймеры)
*   Платформа с поддержкой портов (ESP32/ESP8266/STM32/Arduino)

## Основные возможности

*   **Множественные ключи:** Поддержка до `keysCount` независимых ключей через шаблон
*   **Битовая структура состояний:** Эффективное хранение состояния всех ключей в битовой маске
*   **Инверсия выходов:** Программная инверсия логического состояния всех ключей
*   **Режим чекбокса:** При включении одного ключа автоматически отключаются все остальные
*   **Принудительное отключение:** Режим `alwaysoff` для временной блокировки всех ключей
*   **Настройка старта:** Выбор поведения ключей при включении контроллера (восстановить/выключить/включить)
*   **Гибкое наследование:** Может использоваться как базовый класс для специализированных модулей ключей

## Параметры конфигурации

Класс использует параметры базовых классов и добавляет свои специфические параметры:

| ID | Название | Тип | Группа | Хранилище | Кол-во | Описание |
|:--:|:---|:---|:---|:---|:---:|:---|
| 0 | `turn` | bits1 | control | RTC | keysCount | **Состояния ключей.** 0=выкл, 1=вкл. Восстанавливается после перезагрузки |
| 1 | `invert` | bits1 | option | EEPROM | 1 | **Инверсия.** Инвертировать логическое состояние всех ключей |
| 2 | `alwaysoff` | bits1 | option | EEPROM | 1 | **Всегда отключено.** Принудительное отключение всех ключей |
| 3 | `checkbox` | bits1 | option | EEPROM | 1 | **Режим чекбокса.** При включении одного ключа отключать остальные |
| 4 | `startTurn` | bits2 | option | EEPROM | 1 | **Стартовое состояние.** 0=восстановить;1=не трогать;2=выключено;3=включено |

**Примечание:**
*   **Кол-во:** Параметр `turn` имеет `keysCount` значений (по одному на каждый ключ). Остальные параметры — по 1 значению.
*   **Хранилище turn:** По умолчанию `MWOS_STORAGE_RTC` (энергонезависимое ОЗУ). Может быть изменено через `MWOS_PARAM_STORAGE_KEY_TURN`.
*   **Параметр 4 (startTurn):** Формат значений: "Восстановить;Не трогать;Выключено;Включено"

## Конфигурация модуля

```
; модуль key - базовый класс для управления ключами/реле
; ID - Описание параметра
; 0 - Состояния ключей (keysCount значений, 0=выкл, 1=вкл)
key:turn = 0, 0, 0, 0, 0
; 1 - Инвертировать значение
key:invert = 0
; 2 - Всегда отключено
key:alwaysoff = 0
; 3 - Режим чекбокса (0=нет, 1=выключить другие при включении этого)
key:checkbox = 0
; 4 - Стартовое состояние (0=восстановить;1=не трогать;2=выключено;3=включено)
key:startTurn = 0
```

**Примечание:**
*   **Кол-во значений:** Параметр `turn` имеет `keysCount` значений (по одному на каждый ключ). Остальные параметры — по 1 значению.
*   **Хранилище:** `turn` сохраняется в RTC (энергонезависимое ОЗУ), остальные параметры в EEPROM.

## Конструкторы

Класс является шаблонным: `MWOSKeyBase<keysCount>`.

1.  **По умолчанию**
    ```cpp
    MWOSKeyBase();
    ```
    Создаёт модуль с параметрами по умолчанию. Имя модуля устанавливается в "key". Тип модуля устанавливается в `MODULE_KEY`.

## Основные методы

### Управление ключами

*   `virtual uint8_t turn(uint8_t mode, int16_t index, bool saveToStorage)`
    *   Изменить значение ключа.
    *   **Параметры:** `mode` (0=выключить, 1=включить, 2=переключить), `index` (номер ключа), `saveToStorage` (сохранять в хранилище).
    *   **Возвращает:** `uint8_t` — фактическое состояние порта с учётом инверсии.

*   `void turnBool(bool modeTurn, int16_t index)`
    *   Переключить ключ только если состояние отличается от текущего.
    *   Кеширует значение для избежания лишних переключений.

*   `bool getValueBool(int16_t index)`
    *   Получить текущее бинарное значение ключа.
    *   **Возвращает:** `bool` — состояние ключа (с учётом инверсии).

*   `void start(int16_t index)`
    *   Включить ключ.

*   `void stop(int16_t index)`
    *   Выключить ключ.

*   `void restart(int16_t index)`
    *   Переключить ключ в противоположное состояние.

### Инициализация

*   `virtual void onEvent(MWOSModeEvent modeEvent, MWValue &data)` *(переопределён)*
    *   **`EVENT_INIT` / `EVENT_CHANGE`:** Загрузка настроек из хранилища, инициализация состояний ключей согласно `startTurn`.
    *   **`EVENT_GET_VALUE`:** Возврат текущих значений параметров.
    *   **`EVENT_SET_VALUE`:** Обработка команд установки значений ключей.

*   `void setMode(MW_PIN_MODE pull)`
    *   Задать тип выхода ключа (Выход;Открытый коллектор;ШИМ;ЦАП).

## Пример использования

### Базовое подключение

```cpp
#include "MWOSKeyBase.h"

// Создаём тип для 4 ключей
MWOSKeyBase<4> keyModule;

void setup() {
    // Настройки по умолчанию (загружаются только если хранилище не инициализировано)
    keyModule._checkbox = 1;      // Режим чекбокса (взаимное исключение)
    keyModule._startTurn = 0;     // Восстанавливать состояние после перезагрузки
    
    // Запуск системы - загружает настройки из хранилища
    mwos.start();
}

void loop() {
    // Все действия выполняются внутри модулей через обработчики событий
    mwos.update();
}
```

### Управление ключами

```cpp
#include "MWOSKeyBase.h"

// Создаём модуль с 3 ключами
MWOSKeyBase<3> keyModule;

void setup() {
    // Настройка инверсии
    keyModule._invert = 1;  // Инвертировать все выходы
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
}
```

### Режим чекбокса

```cpp
#include "MWOSKeyBase.h"

// Создаём модуль с 5 ключами (например, для выбора источника сигнала)
MWOSKeyBase<5> keyModule;

void setup() {
    // Включаем режим чекбокса
    keyModule._checkbox = 1;  // При включении одного ключа остальные отключаются
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
}
```

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `turn` | RTC | Состояния ключей, восстанавливаются после перезагрузки |
| `invert` | EEPROM | Флаг инверсии, сохраняется |
| `alwaysoff` | EEPROM | Флаг принудительного отключения, сохраняется |
| `checkbox` | EEPROM | Режим чекбокса, сохраняется |
| `startTurn` | EEPROM | Поведение при старте, сохраняется |

## Важные замечания

1.  **Только для наследования:** Класс `MWOSKeyBase` предназначен только для наследования и не используется самостоятельно. Конкретные модули ключей (реле, транзисторы, ШИМ) наследуются от этого класса и переопределяют метод `turn()` для физической коммутации.

2.  **Порядок инициализации:** Все настройки по умолчанию должны задаваться **до** вызова `mwos.start()`. После запуска системы значения загружаются из хранилища, и повторная установка значений в коде будет перезаписывать сохранённые настройки при каждой перезагрузке.

3.  **Режим чекбокса:** При включённом параметре `checkbox` включение любого ключа автоматически отключает все остальные ключи модуля. Это полезно для взаимно исключающих состояний (например, выбор источника сигнала).

4.  **Стартовое состояние:** Параметр `startTurn` управляет поведением ключей при включении контроллера:
    *   0 = Восстановить состояние до выключения (по умолчанию)
    *   1 = Не трогать (оставить как есть после инициализации портов)
    *   2 = Принудительно выключить все ключи
    *   3 = Принудительно включить все ключи

5.  **Инверсия:** Параметр `invert` инвертирует логическое состояние всех ключей. Это удобно для реле с нормально замкнутыми контактами или активной низкой логикой.

6.  **Принудительное отключение:** Режим `alwaysoff` временно блокирует все ключи независимо от их состояния. Полезно для аварийной остановки или режима безопасности.

7.  **Архитектура MWOS:** Все действия выполняются внутри модулей через обработчики событий. Пользовательский код в `loop()` должен содержать только вызов `mwos.update()`. Для расширения функциональности создавайте собственные модули, наследуясь от `MWOSModule`.

8.  **Сохранение состояния:** Состояния ключей сохраняются в RTC (энергонезависимое ОЗУ) при каждом изменении. Это обеспечивает восстановление состояния после перезагрузки без износа EEPROM.

9.  **Битовая маска:** Все состояния ключей хранятся в битовой маске `MWBitsMaskStat<keysCount>`, что позволяет эффективно использовать память даже при большом количестве ключей.

10. **Физическая коммутация:** Для реальной коммутации портов необходимо создать класс-наследник и переопределить метод `turn()`, в котором управлять портами через `mwos.writeValueDigital()` или аналогичные методы.

## См. также

*   **MWOSModule**: Базовый класс для всех модулей контроллера. Реализует общую инфраструктуру модулей, параметры и события.
*   **MWOSModuleBase**: Базовый класс модулей. Предоставляет методы работы с параметрами и событиями.
*   **MWBitsMaskStat**: Класс битовой маски для эффективного хранения состояний ключей.
*   **MWOSKey**: Пример конкретного модуля ключей, наследующегося от `MWOSKeyBase`.
*   **MWOSKeyAnalog**: Модуль аналоговых ключей с поддержкой ШИМ и ЦАП.
