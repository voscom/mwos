# MWOSKeySwitch

**Имя модуля:** `key`

**MWOSKeySwitch** — модуль ключей с автоматическим переключением от внешнего датчика. При изменении бинарного значения датчика ключ автоматически переключается в противоположное состояние.

## Описание

Класс предназначен для:
*   **Автоматическое переключение:** Ключ переключается при изменении бинарного значения датчика
*   **Управление от датчика:** Датчик и индекс значения задаются в параметрах модуля
*   **Множественные ключи:** Поддержка до `keysCount` независимых ключей, каждый может управляться своим датчиком
*   **Гибкое наследование:** Работает с любым типом ключей, наследующимся от MWOSKeyBase

**Наследование:** `MWOSKeySwitch<keysCount, MWOSKeyForSchedule>` → `MWOSKeyForSchedule` → `MWOSKey` → `MWOSKeyBase<keysCount>` → `MWOSModule`

**Тип модуля:** `MODULE_KEY`

**Зависимости:**
*   Библиотека `MWOSKeySchedule.h` (базовый класс ключей)
*   Библиотека `MWOSSensorBase.h` (для работы с датчиками)
*   Библиотека `MWTimeout.h` (таймеры инициализации)

## Основные возможности

*   **Автоматическое переключение:** При изменении бинарного значения датчика ключ автоматически переключается (вкл→выкл или выкл→вкл)
*   **Поддержка множественных ключей:** До `keysCount` независимых ключей с индивидуальными настройками датчиков
*   **Гибкое наследование:** Работает с MWOSKey, MWOSKeyAnalog, MWOSKeySchedule и другими наследниками MWOSKeyBase
*   **Сохранение настроек:** Все параметры сохраняются в EEPROM и восстанавливаются после перезагрузки
*   **Отложенная инициализация:** Автоматическая загрузка ссылки на модуль датчика после старта системы

## Параметры конфигурации

Класс использует параметры базовых классов и добавляет свои специфические параметры:

| ID | Название | Тип | Группа | Хранилище | Кол-во | Описание |
|:--:|:---|:---|:---|:---|:---:|:---|
| 0 | `turn` | bits1 | control | RTC | keysCount | **Состояния ключей.** 0=выкл, 1=вкл. Восстанавливается после перезагрузки |
| 1 | `invert` | bits1 | option | EEPROM | 1 | **Инверсия.** Инвертировать логическое состояние всех ключей |
| 2 | `alwaysoff` | bits1 | option | EEPROM | 1 | **Всегда отключено.** Принудительное отключение всех ключей |
| 3 | `checkbox` | bits1 | option | EEPROM | 1 | **Режим чекбокса.** При включении одного ключа отключать остальные |
| 4 | `startTurn` | bits2 | option | EEPROM | 1 | **Стартовое состояние.** 0=восстановить;1=не трогать;2=выключено;3=включено |
| 6 | `pull` | bits3 | option | EEPROM | 1 | **Подтяжка портов.** Режим (Выход;Открытый коллектор;ШИМ;ЦАП) |
| 7 | `pin` | int16 | pin | FOR_PIN | keysCount | **Порты ключей.** Номера портов для каждого ключа |
| 21 | `sensorModuleId` | int16 | module_id+option | EEPROM | 1 | **ID модуля датчика.** Номер модуля датчика для управления ключом |
| 22 | `sensorIndex` | int16 | option | EEPROM | keysCount | **Индекс датчика.** Индекс параметра 0 датчика для каждого ключа |

**Примечание:**
*   **Кол-во:** Параметры с `keysCount` имеют индивидуальное значение для каждого ключа. Остальные параметры — по 1 значению.
*   **Параметр 21 (sensorModuleId):** ID модуля датчика в системе MWOS (минимум 2).
*   **Параметр 22 (sensorIndex):** Индекс бинарного значения датчика (параметр 0). Если датчик множественный, каждый ключ может управляться своим индексом.

## Конфигурация модуля

```
; модуль key - ключи с переключением от датчика
; 0 - Состояния ключей (keysCount значений, 0=выкл, 1=вкл)
key:turn = 0, 0, 0, 0, 0
; 1 - Инвертировать значение
key:invert = 0
; 2 - Всегда отключено
key:alwaysoff = 0
; 3 - Режим чекбокса (0=нет, 1=выключить другие при включении этого)
key:checkbox = 0
; 4 - Стартовое состояние (0=восстановить;1=не трогать;2=выключено;3=включено)
key:startTurn = 0
; 6 - Подтяжка портов (0=Выход;1=Открытый коллектор;2=ШИМ;3=ЦАП)
key:pull = 0
; 7 - Порты ключей (keysCount значений)
key:pin = 5, 6, 7, 8, -1
; 21 - ID модуля датчика для управления ключом
key:sensorModuleId = 10
; 22 - Индексы параметра 0 датчика для каждого ключа (keysCount значений)
key:sensorIndex = 0, 1, 2, 3, 4
```

**Примечание:**
*   **Кол-во значений:** Параметры `turn`, `pin`, `sensorIndex` имеют `keysCount` значений (по одному на каждый ключ). Остальные параметры — по 1 значению.
*   **ID модуля датчика:** Должен быть не менее 2 (системные модули имеют ID 0 и 1).
*   **Индекс датчика:** Должен быть в пределах количества датчиков в модуле датчика.

## Конструкторы

Класс является шаблонным: `MWOSKeySwitch<keysCount, MWOSKeyForSwitch>`.

1.  **По умолчанию**
    ```cpp
    MWOSKeySwitch();
    ```
    Создаёт модуль с параметрами по умолчанию. Все порты инициализируются значением -1 (не подключены).

2.  **Из массива в PROGMEM**
    ```cpp
    MWOSKeySwitch(const uint8_t * pins);
    ```
    Создаёт модуль с портами из массива в памяти PROGMEM. Позволяет задать конфигурацию портов на этапе компиляции.

3.  **Одиночный пин**
    ```cpp
    MWOSKeySwitch(MWOS_PIN_INT pin);
    ```
    Создаёт модуль с одним ключом на указанном порту. Удобно для простых конфигураций с одним реле.

4.  **С указанием типа ключа**
    ```cpp
    MWOSKeySwitch<4, MWOSKeySchedule<4>> keySwitch;
    ```
    Создаёт модуль с ключами расширенной функциональности (таймеры, расписание).

## Основные методы

### Управление ключами

*   `virtual void onEvent(MWOSModeEvent modeEvent)` *(переопределён)*
    *   **`EVENT_INIT` / `EVENT_CHANGE`:** Загрузка настроек из хранилища, инициализация ссылки на датчик.
    *   **`EVENT_UPDATE`:** Проверка состояния датчика, переключение ключа при изменении бинарного значения.

*   `void loadSensors()`
    *   Загружает ссылку на модуль датчика по ID.
    *   Проверяет корректность типа датчика (параметр 0 должен быть readonly bits1).
    *   Считывает текущие значения датчиков для синхронизации состояний.

### Инициализация

*   `void initKeySwitch()`
    *   Регистрирует параметры модуля (sensorModuleId, sensorIndex).
    *   Инициализирует массивы индексов датчиков значением -1.

## Пример использования

### Базовое подключение

```cpp
#include "MWOSKeySwitch.h"

// Создаём тип для 4 ключей с переключением от датчика
MWOSKeySwitch<4> keySwitch;

// Массив пинов в PROGMEM
const uint8_t PROGMEM pins[] = {5, 6, 7, 8};

// Экземпляр модуля
keySwitch(pins);

void setup() {
    // Настройки по умолчанию (загружаются только если хранилище не инициализировано)
    // keySwitch._sensorModuleId = 10;  // ID модуля датчика
    
    // Запуск системы - загружает настройки из хранилища
    mwos.start();
}

void loop() {
    // Все действия выполняются внутри модулей через обработчики событий
    mwos.update();
}
```

### Подключение с ключами расширенной функциональности

```cpp
#include "MWOSKeySwitch.h"
#include "MWOSKeySchedule.h"

// Создаём модуль с ключами с таймерами и расписанием
MWOSKeySwitch<4, MWOSKeySchedule<4>> keySwitch;

void setup() {
    // Настройка ID датчика (загружается только если хранилище не инициализировано)
    keySwitch._sensorModuleId = 10;  // ID модуля бинарного датчика
    
    // Настройка индексов датчиков для каждого ключа
    keySwitch._sensorIndex[0] = 0;  // Ключ 0 управляется датчиком 0
    keySwitch._sensorIndex[1] = 1;  // Ключ 1 управляется датчиком 1
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
}
```

### Подключение с аналоговыми ключами

```cpp
#include "MWOSKeySwitch.h"
#include "MWOSKeyAnalog.h"

// Создаём модуль с аналоговыми ключами (ШИМ)
MWOSKeySwitch<2, MWOSKeyAnalog<2>> keySwitch;

void setup() {
    // Настройка ID датчика
    keySwitch._sensorModuleId = 15;  // ID модуля бинарного датчика
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
}
```

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `turn` | RTC | Состояния ключей, восстанавливаются после перезагрузки |
| `pin` | FOR_PIN | EEPROM или NO в зависимости от конфигурации |
| `pull` | EEPROM | Режим подтяжки портов |
| `sensorModuleId` | EEPROM | ID модуля датчика, сохраняется |
| `sensorIndex` | EEPROM | Индексы датчиков, сохраняются |
| `invert`, `alwaysoff`, `checkbox` | EEPROM | Флаги режима работы, сохраняются |
| `startTurn` | EEPROM | Поведение при старте, сохраняется |

## Важные замечания

1.  **ID модуля датчика:** Параметр `sensorModuleId` должен быть не менее 2. Системные модули имеют ID 0 и 1 и не могут использоваться как датчики.

2.  **Тип параметра датчика:** Модуль проверяет, что параметр 0 датчика имеет тип `mwos_param_readonly` и `mwos_param_bits1`. Если тип не соответствует, ссылка на датчик сбрасывается.

3.  **Отложенная инициализация:** Ссылка на модуль датчика загружается не сразу, а через таймаут 0.5 секунды после старта. Это позволяет всем модулям системы инициализироваться перед попыткой подключения.

4.  **Автоматическое переключение:** Ключ переключается только при **изменении** бинарного значения датчика. Если датчик уже имеет значение 1, ключ не переключится повторно до изменения на 0 и обратно.

5.  **Множественные ключи:** Каждый ключ может управляться своим индексом датчика. Это позволяет одному модулю датчика с несколькими каналами управлять несколькими ключами.

6.  **Гибкое наследование:** Класс работает с любым типом ключей, наследующимся от MWOSKeyBase. Можно использовать MWOSKey, MWOSKeyAnalog, MWOSKeySchedule и другие.

7.  **Архитектура MWOS:** Все действия выполняются внутри модулей через обработчики событий. Пользовательский код в `loop()` должен содержать только вызов `mwos.update()`.

8.  **Проверка датчика:** При инициализации модуль проверяет существование датчика и корректность типа параметра. При ошибке ссылка на датчик сбрасывается и повторяется попытка через таймаут.

9.  **Сохранение состояния:** Состояния ключей сохраняются в RTC (энергонезависимое ОЗУ) при каждом изменении. Это обеспечивает восстановление состояния после перезагрузки без износа EEPROM.

10. **Режим чекбокса:** При включённом параметре `checkbox` переключение одного ключа автоматически отключает все остальные ключи модуля.

## См. также

*   **MWOSKeyBase**: Базовый класс для всех модулей ключей. Реализует общую логику управления состояниями ключей, инверсию и режим чекбокса.
*   **MWOSKey**: Базовый модуль ключей для управления портами микроконтроллера. Предоставляет физическую коммутацию реле и транзисторов.
*   **MWOSKeySchedule**: Модуль ключей с таймерами и расписанием. Может использоваться вместо MWOSKey для расширенной функциональности.
*   **MWOSKeyAnalog**: Модуль аналоговых ключей с поддержкой ШИМ и ЦАП. Может использоваться вместо MWOSKey для плавного управления.
*   **MWOSSensorBase**: Базовый класс для всех датчиков. Предоставляет бинарные значения для управления ключами.
