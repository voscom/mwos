# MWOSKey

**Имя модуля:** `key`

**MWOSKey** — базовый модуль ключей для управления портами микроконтроллера. Предоставляет физическую коммутацию реле, транзисторов и других исполнительных устройств через цифровые выходы.

## Описание

Класс предназначен для:
*   **Управление ключами/реле:** Физическая коммутация портов микроконтроллера для управления внешними устройствами
*   **Множественные ключи:** Поддержка до `keysCount` независимых ключей в одном модуле
*   **Режимы работы:** Инверсия, чекбокс (взаимное исключение), принудительное отключение
*   **Сохранение состояния:** Автоматическое сохранение состояния ключей при выключении

**Наследование:** `MWOSKey<keysCount>` → `MWOSKeyBase<keysCount>` → `MWOSModule`

**Тип модуля:** `MODULE_KEY`

**Зависимости:**
*   Библиотека `MWOSKeyBase.h` (базовый класс)
*   Платформа с поддержкой цифровых выходов (ESP32/ESP8266/STM32/Arduino)

## Основные возможности

*   **Множественные ключи:** Поддержка до `keysCount` независимых ключей через шаблон
*   **Битовая структура состояний:** Эффективное хранение состояния всех ключей в битовой маске
*   **Инверсия выходов:** Программная инверсия логического состояния всех ключей
*   **Режим чекбокса:** При включении одного ключа автоматически отключаются все остальные
*   **Принудительное отключение:** Режим `alwaysoff` для временной блокировки всех ключей
*   **Настройка старта:** Выбор поведения ключей при включении контроллера (восстановить/выключить/включить)
*   **Настройка типа выхода:** Поддержка режимов Выход, Открытый коллектор, ШИМ, ЦАП

## Параметры конфигурации

Класс использует параметры базовых классов и добавляет свои специфические параметры:

| ID | Название | Тип | Группа | Хранилище | Кол-во | Описание |
|:--:|:---|:---|:---|:---|:---:|:---|
| 0 | `turn` | bits1 | control | RTC | keysCount | **Состояния ключей.** 0=выкл, 1=вкл. Восстанавливается после перезагрузки |
| 1 | `invert` | bits1 | option | EEPROM | 1 | **Инверсия.** Инвертировать логическое состояние всех ключей |
| 2 | `alwaysoff` | bits1 | option | EEPROM | 1 | **Всегда отключено.** Принудительное отключение всех ключей |
| 3 | `checkbox` | bits1 | option | EEPROM | 1 | **Режим чекбокса.** При включении одного ключа отключать остальные |
| 4 | `startTurn` | bits2 | option | EEPROM | 1 | **Стартовое состояние.** 0=восстановить;1=не трогать;2=выключено;3=включено |
| 6 | `pull` | bits3 | option | EEPROM | 1 | **Подтяжка портов.** Режим (Выход;Открытый коллектор;ШИМ;ЦАП) |
| 7 | `pin` | int16 | pin | FOR_PIN | keysCount | **Порты ключей.** Номера портов для каждого ключа |

**Примечание:**
*   **Кол-во:** Параметры с `keysCount` имеют индивидуальное значение для каждого ключа. Остальные параметры — по 1 значению.
*   **Параметр 6 (pull):** Формат значений: "Выход;Открытый коллектор;ШИМ;ЦАП"
*   **Параметр 4 (startTurn):** Определяет поведение ключей при включении контроллера.

## Конфигурация модуля

```
; модуль key - ключи для управления портами микроконтроллера
; 0 - Состояния ключей (keysCount значений, 0=выкл, 1=вкл)
key:turn = 0, 0, 0, 0, 0
; 1 - Инвертировать значение
key:invert = 0
; 2 - Всегда отключено
key:alwaysoff = 0
; 3 - Режим чекбокса (0=нет, 1=выключить другие при включении этого)
key:checkbox = 0
; 4 - Стартовое состояние (0=восстановить;1=не трогать;2=выключено;3=включено)
key:startTurn = 0
; 6 - Подтяжка портов (0=Выход;1=Открытый коллектор;2=ШИМ;3=ЦАП)
key:pull = 0
; 7 - Порты ключей (keysCount значений)
key:pin = 5, 6, 7, 8, -1
```

**Примечание:**
*   **Кол-во значений:** Параметры `turn`, `pin` имеют `keysCount` значений (по одному на каждый ключ). Остальные параметры — по 1 значению.
*   **Значения портов:** `-1` = порт не подключён, `-3` = ошибка инициализации порта.

## Конструкторы

Класс является шаблонным: `MWOSKey<keysCount>`.

1.  **По умолчанию**
    ```cpp
    MWOSKey();
    ```
    Создаёт модуль с параметрами по умолчанию. Все порты инициализируются значением -1 (не подключены).

2.  **Из массива в PROGMEM**
    ```cpp
    MWOSKey(const uint8_t * pins);
    ```
    Создаёт модуль с портами из массива в памяти PROGMEM. Позволяет задать конфигурацию портов на этапе компиляции.

3.  **Одиночный пин**
    ```cpp
    MWOSKey(MWOS_PIN_INT pin);
    ```
    Создаёт модуль с одним ключом на указанном порту. Удобно для простых конфигураций с одним реле.

## Основные методы

### Управление ключами

*   `virtual uint8_t turn(uint8_t mode, int16_t index, bool saveToStorage)` *(переопределён)*
    *   Изменяет значение ключа и физически переключает порт.
    *   **Параметры:** `mode` (0=выключить, 1=включить, 2=переключить), `index` (номер ключа), `saveToStorage` (сохранять в хранилище).
    *   **Возвращает:** `uint8_t` — фактическое состояние порта с учётом инверсии.

*   `void turnBool(bool modeTurn, int16_t index)` *(унаследован)*
    *   Переключить ключ только если состояние отличается от текущего.
    *   Кеширует значение для избежания лишних переключений.

*   `bool getValueBool(int16_t index)` *(унаследован)*
    *   Получить текущее бинарное значение ключа.
    *   **Возвращает:** `bool` — состояние ключа (с учётом инверсии).

*   `void start(int16_t index)` *(унаследован)*
    *   Включить ключ.

*   `void stop(int16_t index)` *(унаследован)*
    *   Выключить ключ.

*   `void restart(int16_t index)` *(унаследован)*
    *   Переключить ключ в противоположное состояние.

### Инициализация

*   `virtual void onEvent(MWOSModeEvent modeEvent)` *(переопределён)*
    *   **`EVENT_INIT` / `EVENT_CHANGE`:** Загрузка настроек из хранилища, инициализация портов.
    *   **`EVENT_GET_VALUE`:** Возврат текущих значений параметров.

*   `void initKey(int16_t index)`
    *   Настраивает порт ключа в заданном режиме.
    *   При ошибке настройки записывает значение -3 в параметр `pin` и устанавливает флаг изменения.

*   `void setMode(MW_PIN_MODE pull)` *(унаследован)*
    *   Задать тип выхода ключа (Выход;Открытый коллектор;ШИМ;ЦАП).

## Пример использования

### Базовое подключение

```cpp
#include "MWOSKey.h"

// Создаём тип для 4 ключей
MWOSKey<4> keyModule;

// Массив пинов в PROGMEM
const uint8_t PROGMEM pins[] = {5, 6, 7, 8};

// Экземпляр модуля
keyModule(pins);

void setup() {
    // Настройки по умолчанию (загружаются только если хранилище не инициализировано)
    // keyModule._checkbox = 1;  // Режим чекбокса (взаимное исключение)
    
    // Запуск системы - загружает настройки из хранилища
    mwos.start();
}

void loop() {
    // Все действия выполняются внутри модулей через обработчики событий
    mwos.update();
}
```

### Управление ключами

```cpp
#include "MWOSKey.h"

// Создаём модуль с 3 ключами
MWOSKey<3> keyModule;

void setup() {
    // Настройка инверсии
    keyModule._invert = 1;  // Инвертировать все выходы
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
    
    // Включение ключа 0
    keyModule.start(0);
    
    // Выключение ключа 1
    keyModule.stop(1);
    
    // Переключение ключа 2
    keyModule.restart(2);
    
    // Проверка состояния
    if (keyModule.getValueBool(0)) {
        // Ключ 0 включён
    }
}
```

### Режим чекбокса

```cpp
#include "MWOSKey.h"

// Создаём модуль с 5 ключами (например, для выбора источника сигнала)
MWOSKey<5> keyModule;

void setup() {
    // Включаем режим чекбокса
    keyModule._checkbox = 1;  // При включении одного ключа остальные отключаются
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
    
    // При включении ключа 2 все остальные автоматически отключатся
    keyModule.start(2);
}
```

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `turn` | RTC | Состояния ключей, восстанавливаются после перезагрузки |
| `pin` | FOR_PIN | EEPROM или NO в зависимости от конфигурации |
| `pull` | EEPROM | Режим подтяжки портов, сохраняется |
| `invert`, `alwaysoff`, `checkbox` | EEPROM | Флаги режима работы, сохраняются |
| `startTurn` | EEPROM | Поведение при старте, сохраняется |

## Важные замечания

1.  **Порядок инициализации:** Все настройки по умолчанию должны задаваться **до** вызова `mwos.start()`. После запуска системы значения загружаются из хранилища, и повторная установка значений в коде будет перезаписывать сохранённые настройки при каждой перезагрузке.

2.  **Режим чекбокса:** При включённом параметре `checkbox` включение любого ключа автоматически отключает все остальные ключи модуля. Это полезно для взаимно исключающих состояний (например, выбор источника сигнала).

3.  **Стартовое состояние:** Параметр `startTurn` управляет поведением ключей при включении контроллера:
    *   0 = Восстановить состояние до выключения (по умолчанию)
    *   1 = Не трогать (оставить как есть после инициализации портов)
    *   2 = Принудительно выключить все ключи
    *   3 = Принудительно включить все ключи

4.  **Инверсия:** Параметр `invert` инвертирует логическое состояние всех ключей. Это удобно для реле с нормально замкнутыми контактами или активной низкой логикой.

5.  **Принудительное отключение:** Режим `alwaysoff` временно блокирует все ключи независимо от их состояния. Полезно для аварийной остановки или режима безопасности.

6.  **Архитектура MWOS:** Все действия выполняются внутри модулей через обработчики событий. Пользовательский код в `loop()` должен содержать только вызов `mwos.update()`. Для расширения функциональности создавайте собственные модули, наследуясь от `MWOSModule`.

7.  **Сохранение состояния:** Состояния ключей сохраняются в RTC (энергонезависимое ОЗУ) при каждом изменении. Это обеспечивает восстановление состояния после перезагрузки без износа EEPROM.

8.  **Битовая маска:** Все состояния ключей хранятся в битовой маске `MWBitsMaskStat<keysCount>`, что позволяет эффективно использовать память даже при большом количестве ключей.

9.  **Физическая коммутация:** Метод `turn()` автоматически управляет портами через `mwos.writeValueDigital()`. При ошибке инициализации порта записывается значение -3 в параметр `pin`.

10. **Режимы выхода:** Параметр `pull` определяет режим работы порта: 0=Выход, 1=Открытый коллектор, 2=ШИМ, 3=ЦАП. Для аналогового управления используйте MWOSKeyAnalog.

## См. также

*   **MWOSKeyBase**: Базовый класс для всех модулей ключей. Реализует общую логику управления состояниями ключей, инверсию и режим чекбокса.
*   **MWOSKeyAnalog**: Модуль аналоговых ключей с поддержкой ШИМ и ЦАП. Наследуется от MWOSKey для плавного управления мощностью.
*   **MWOSKeySchedule**: Модуль ключей с таймерами и расписанием. Наследуется от MWOSKey для расширенной функциональности.
*   **MWOSKeySwitch**: Модуль ключей, управляемых внешним датчиком. Переключает ключ при изменении бинарного значения датчика.
*   **MWOSKeyIR**: Модуль управления ИК-приборами через инфракрасный светодиод. Наследуется от MWOSKeyBase.
