# Документация по модулям ключей MWOS

## Обзор архитектуры ключей

Все модули ключей в системе MWOS построены на единой иерархической архитектуре с наследованием от базовых классов. Каждый модуль представляет собой шаблонный класс, поддерживающий множественные ключи на одном экземпляре модуля.

## Модули ключей

| Модуль                                | Наследование             | Описание |
|---------------------------------------|--------------------------|----------|
| [MWOSKeyBase](core/MWOSKeyBase.md)    | MWOSModule               | Базовый класс для всех модулей ключей |
| [MWOSKey](MWOSKey.md)                 | MWOSKeyBase              | Базовые цифровые ключи для управления портами |
| [MWOSKeyAnalog](MWOSKeyAnalog.md)     | MWOSKey                  | Аналоговые ключи с поддержкой ШИМ и ЦАП |
| [MWOSKeySchedule](MWOSKeySchedule.md) | наследники MWOSKey       | Ключи с таймерами и суточным расписанием |
| [MWOSKeySwitch](MWOSKeySwitch.md)     | наследники MWOSKey       | Ключи, переключаемые внешним датчиком |
| [MWOSKeyFeedback](MWOSKeyFeedback.md) | наследники MWOSKeyAnalog | Ключи управления мощностью с обратной связью от датчика |
| [MWOSKeyIR](MWOSKeyIR.md)             | MWOSKeyBase              | Управление ИК-приборами через инфракрасный светодиод |
| [MWOSHzESP32](MWOSHzESP32.md)         | MWOSKey                  | Регулятор частоты и скважности для ESP32 (LEDC) |
| [MWOSHzSTM32](MWOSHzSTM32.md)         | MWOSKey                  | Регулятор частоты и скважности для STM32 (HardwareTimer) |

---

### Иерархия классов ключей

```
MWOSModule
└── MWOSKeyBase<keysCount> (базовый класс для всех ключей)
    ├── MWOSKey<keysCount> (базовые цифровые ключи)
    │   ├── MWOSKeyAnalog<keysCount> (аналоговые ключи ШИМ/ЦАП)
    │   │   ├── MWOSKeyFeedback<keysCount> (ключи с обратной связью от датчика)
    │   │   ├── MWOSHzESP32<keysCount> (генератор частоты ESP32)
    │   │   └── MWOSHzSTM32 (генератор частоты STM32)
    │   ├── MWOSKeySchedule<keysCount> (ключи с таймерами и расписанием)
    │   ├── MWOSKeySwitch<keysCount> (ключи, переключаемые датчиком)
    │   └── MWOSKeyIR<keysCount> (ИК-управление приборами)
```

## Композиция модулей через шаблоны

Одной из ключевых особенностей архитектуры ключей MWOS является возможность гибкой композиции функциональности через шаблонные параметры. Классы `MWOSKeySchedule` и `MWOSKeySwitch` могут принимать в качестве шаблонного параметра `MWOSKeyForSwitch` любого наследника `MWOSKeyBase`, а `MWOSKeyFeedback` принимает любого наследника `MWOSKeyAnalog` в параметре `MWOSKeyForSwitch`. Это позволяет комбинировать свойства различных модулей в одном экземпляре, создавая сложные устройства с минимальным количеством кода. Например, можно создать ключ с аналоговым управлением, таймерами и переключением от датчика одновременно:

```cpp
#include "modules/key/MWOSKeyAnalog.h"
#include "modules/key/MWOSKeySchedule.h"
#include "modules/key/MWOSKeySwitch.h"

// Ключ с аналоговым управлением + таймеры/расписание + переключение от датчика
MWOSKeySwitch<5, MWOSKeySchedule<5, MWOSKeyAnalog<5>>> key0;
```

Или объединить регулировку мощности с генерацией частоты на ESP32:

```cpp
#include "modules/key/MWOSKeyFeedback.h"
#include "modules/key/MWOSHzESP32.h"

// Ключ с обратной связью от датчика + генерация ШИМ с настраиваемой частотой
MWOSKeyFeedback<5, MWOSHzESP32<5>> key1;
```

Таким образом, каждый уровень наследования добавляет свою функциональность: базовый класс предоставляет физическую коммутацию портов, промежуточные классы добавляют таймеры, расписание или управление от датчиков, а верхний уровень завершает композицию. Это позволяет создавать уникальные конфигурации под конкретные задачи, комбинируя готовые модули как конструктор, без необходимости писать новый код для каждого сочетания функций.

## Базовый класс MWOSKeyBase

Класс `MWOSKeyBase` является фундаментальным базовым классом для всех модулей ключей в системе. Он реализует общую логику управления состояниями ключей, инверсию, режим чекбокса и базовую инфраструктуру для наследования.

### Назначение класса

Этот класс предназначен только для наследования и не используется самостоятельно. Он предоставляет универсальный механизм работы с множественными ключами одного типа, где количество ключей задаётся шаблоном при объявлении класса.

### Параметры модуля

| Параметр | Тип | Группа | Хранилище | Описание |
|----------|-----|--------|-----------|----------|
| turn | bits1 | control | RTC | Состояния ключей (0=выкл, 1=вкл). Восстанавливается после перезагрузки |
| invert | bits1 | option | EEPROM | Инвертировать логическое состояние всех ключей |
| alwaysoff | bits1 | option | EEPROM | Принудительное отключение всех ключей |
| checkbox | bits1 | option | EEPROM | При включении одного ключа отключать остальные |
| startTurn | bits2 | option | EEPROM | Поведение при старте (0=восстановить;1=не трогать;2=выключено;3=включено) |
| pull | bits3 | option | EEPROM | Режим подтяжки (0=Выход;1=Открытый коллектор;2=ШИМ;3=ЦАП) |

### Внутренняя структура состояний ключа

Для каждого ключа модуль хранит состояние в битовой маске `MWBitsMaskStat<keysCount>`, что позволяет эффективно использовать память даже при большом количестве ключей.

### Методы управления состоянием

| Метод | Описание |
|-------|----------|
| turn(mode, index, saveToStorage) | Изменить значение ключа (0=выключить, 1=включить, 2=переключить) |
| turnBool(modeTurn, index) | Переключить ключ только если состояние отличается от текущего |
| getValueBool(index) | Получить текущее бинарное значение ключа |
| start(index) | Включить ключ |
| stop(index) | Выключить ключ |
| restart(index) | Переключить ключ в противоположное состояние |
| setMode(pull) | Задать тип выхода ключа (Выход;Открытый коллектор;ШИМ;ЦАП) |

### Алгоритм управления ключами

Метод `turn()` вызывается при изменении состояния ключа. При включённом режиме `checkbox` включение одного ключа автоматически отключает все остальные ключи модуля. Состояния ключей сохраняются в RTC (энергонезависимое ОЗУ) при каждом изменении, что обеспечивает восстановление состояния после перезагрузки без износа EEPROM.

## Класс MWOSKey

Класс `MWOSKey` наследуется от `MWOSKeyBase` и добавляет поддержку физических портов микроконтроллера для управления реле, транзисторами и другими исполнительными устройствами.

### Назначение класса

Модуль предназначен для физической коммутации портов микроконтроллера. Предоставляет базовую инфраструктуру для всех специализированных модулей ключей.

### Параметры модуля

| Параметр | Тип | Группа | Хранилище | Описание |
|----------|-----|--------|-----------|----------|
| pin | int16 | pin | FOR_PIN | Номера портов для каждого ключа |
| pull | bits3 | option | EEPROM | Режим подтяжки (Выход;Открытый коллектор;ШИМ;ЦАП) |

### Методы

| Метод | Описание |
|-------|----------|
| initKey(index) | Инициализация порта ключа в заданном режиме |
| turn(mode, index, saveToStorage) | Изменить значение ключа и физически переключить порт |

---

## Класс MWOSKeyAnalog

Класс `MWOSKeyAnalog` наследуется от `MWOSKey` и добавляет поддержку аналоговых значений для управления мощностью через ШИМ или ЦАП.

### Назначение класса

Модуль предназначен для плавной регулировки мощности подключенных устройств через ШИМ (PWM) или ЦАП (DAC).

### Параметры модуля

| Параметр | Тип | Группа | Хранилище | Описание |
|----------|-----|--------|-----------|----------|
| value | uint16 | option | EEPROM | Аналоговое значение мощности (0-255 для ШИМ, 0-4095 для ЦАП) |

### Методы

| Метод | Описание |
|-------|----------|
| setKeyValue(mode, index) | Установить аналоговое значение ключа (ШИМ/ЦАП) |
| turn(mode, index, saveToStorage) | Изменить значение ключа и установить аналоговое значение |

---

## Класс MWOSKeySchedule

Класс `MWOSKeySchedule` наследуется от `MWOSKey` и добавляет поддержку таймеров и суточного расписания для автоматического управления ключами.

### Назначение класса

Модуль предназначен для автоматического включения/выключения ключей по таймеру или расписанию.

### Параметры модуля

| Параметр | Тип | Группа | Хранилище | Описание |
|----------|-----|--------|-----------|----------|
| timeon | uint32 | option | EEPROM | Максимальное время включения ключа [сек/10] |
| timeoff | uint32 | option | EEPROM | Максимальное время выключения ключа [сек/10] |
| schedule | int16 | time+option | EEPROM | Время для расписания (минут с полуночи) |
| scheduleTurn | bits2 | option | EEPROM | Действие расписания (0=Без;1=Выключать;2=Включать;3=Переключать) |

### Методы

| Метод | Описание |
|-------|----------|
| turn(mode, index, saveToStorage) | Изменить значение ключа и запустить соответствующий таймер |

## Класс MWOSKeySwitch

Класс `MWOSKeySwitch` наследуется от `MWOSKeySchedule` и добавляет поддержку автоматического переключения ключа от внешнего датчика.

### Назначение класса

Модуль предназначен для автоматического переключения ключа при изменении бинарного значения внешнего датчика.

### Параметры модуля

| Параметр | Тип | Группа | Хранилище | Описание |
|----------|-----|--------|-----------|----------|
| sensorModuleId | int16 | module_id+option | EEPROM | ID модуля датчика для управления ключом |
| sensorIndex | int16 | option | EEPROM | Индекс параметра 0 датчика для каждого ключа |

### Методы

| Метод | Описание |
|-------|----------|
| loadSensors() | Загрузить ссылку на модуль датчика по ID |
| onEvent(modeEvent) | Проверка состояния датчика, переключение ключа при изменении |

### Назначение класса

### Параметры модуля

| Параметр | Тип | Группа | Хранилище | Описание |
|----------|-----|--------|-----------|----------|
| sensorModuleId | int16 | module_id+option | EEPROM | ID модуля аналогового датчика |
| sensorIndex | int16 | option | EEPROM | Индекс параметра 9 датчика для каждого ключа |
| power | int16 | option | EEPROM | Диапазон мощности (мин/макс) |
| changeTimeMS | uint16 | option | EEPROM | Время между изменениями мощности [мс] |

### Методы

| Метод | Описание |

| updateKey(sensorIndex, keyIndex) | Обновить мощность ключа в зависимости от показания датчика |

## Класс MWOSKeyIR

Класс `MWOSKeyIR` наследуется от `MWOSKeyBase` и добавляет поддержку управления ИК-приборами через инфракрасный светодиод.

### Назначение класса

Модуль предназначен для отправки ИК-команд включения/выключения для кондиционеров и других устройств с ИК-пультом.

### Параметры модуля

| Параметр | Тип | Группа | Хранилище | Описание |
|----------|-----|--------|-----------|----------|
| pin | int16 | pin | FOR_PIN | Порт ИК-светодиода |
| typeIR | uint8 | option | EEPROM | Тип ИК-протокола для каждого ключа |

### Методы

| Метод | Описание |
|-------|----------|
| turn(mode, index, saveToStorage) | Изменить значение ключа и отправить ИК-команду |

### Зависимости

*   Библиотека `IRremoteESP8266` (https://github.com/crankyoldgit/IRremoteESP8266.git)
*   Платформа ESP32/ESP8266

| hz | uint32 | option | EEPROM | Частота ШИМ в герцах для каждого канала |
| channel | bits4 | option | EEPROM | Номер канала LEDC (0-15) |

### Методы

| Метод | Описание |

## Хранилища параметров

Система MWOS поддерживает несколько типов хранилищ для сохранения параметров модулей.

| Хранилище | Значение | Описание |
|-----------|----------|----------|
| MWOS_STORAGE_EEPROM | 0 | Энергонезависимая память EEPROM. Для настроек, сохраняющихся между перезагрузками |
| MWOS_STORAGE_RTC | 1 | Статическое ОЗУ с питанием от резервного источника. Для состояний ключей |
| MWOS_STORAGE_NVS | 2 | Раздел NVS на платформе ESP32. Альтернатива EEPROM |
| MWOS_STORAGE_NO | 7 | Параметр не хранится в хранилище. Для вычисляемых значений |
| MWOS_STORAGE_FOR_PIN | 0/7 | Хранилище для параметров портов. EEPROM или NO в зависимости от конфигурации |

| Группа | Значение | Описание |

| PARAM_TYPE_TIME | 2 | Параметры времени. Для расписания и таймеров |

## Макросы объявления параметров

Система MWOS предоставляет макросы для объявления параметров модулей.

### MWOS_PARAM

Объявляет параметр с идентификатором, именем, типом значения, группой, хранилищем и длиной массива.

```cpp
MWOS_PARAM(id, name, valueType, paramGroup, storage, arrayLength)
```

### MWOS_PARAM_FF

Объявляет параметр с форматом перечисления. Принимает дополнительный параметр формата со значениями через точку с запятой.

```cpp
MWOS_PARAM_FF(id, name, valueType, paramGroup, storage, arrayLength, "format")
```

## Шаблоны модулей ключей

Все модули ключей реализованы как шаблонные классы с параметром `keysCount`. Этот параметр определяет количество ключей, поддерживаемых экземпляром модуля.

```cpp
template<MWOS_PARAM_INDEX_UINT keysCount>
class MWOSKey : public MWOSKeyBase<keysCount> {
    // keysCount определяет размер внутренних массивов
};
```

## Наследование и переопределение методов

Модули ключей используют механизм наследования для расширения функциональности. Базовый класс `MWOSKeyBase` реализует общую логику управления состояниями ключей. Класс `MWOSKey` добавляет поддержку физических портов. Специализированные модули наследуются от `MWOSKey` или `MWOSKeyAnalog` и переопределяют виртуальные методы для реализации специфичной логики.

### Виртуальные методы для переопределения

| Метод | Класс | Описание |
|-------|-------|----------|
| turn(mode, index, saveToStorage) | MWOSKeyBase | Изменить значение ключа |
| onEvent(modeEvent) | MWOSModule | Обработка системных событий |
| getValue(param, arrayIndex) | MWOSModule | Запрос значения параметра |
| initKey(index) | MWOSKey | Инициализация порта ключа |

---

## Общие рекомендации

### 1. Обработка событий
*   `EVENT_INIT` — загрузка **всех** настроек при старте
*   `EVENT_CHANGE` — реакция на изменение **конкретного** параметра
*   `EVENT_UPDATE` — тиковая обработка (таймеры, расписание, датчики)
*   `EVENT_GET_VALUE` — возврат значений через `getValue()`
*   Вызов базового класса `Parent::onEvent()` **в конце** метода

### 2. Работа с хранилищами
```cpp
// Загрузка при EVENT_INIT
_value = MWOSModule::loadValue(_value, &p_value, index);

// Сохранение при изменении (автоматически системой)
// Дополнительно — только для RTC или особых случаев
MWOSModule::saveValue(_value, &p_value, index);
```

### 3. Флаги изменений
```cpp
// Пометить параметр как изменённый для отправки
MWOSModule::SetParamChanged(&p_value, index);

// Проверить необходимость отправки
if (MWOSModule::IsParamChanged(&p_value)) {
    // Отправить значение
}
```

### 4. Режим чекбокса
При включённом параметре `checkbox` включение одного ключа автоматически отключает все остальные ключи модуля. Это полезно для взаимно исключающих состояний.

### 5. Стартовое состояние
Параметр `startTurn` управляет поведением ключей при включении контроллера:
*   0 = Восстановить состояние до выключения (по умолчанию)
*   1 = Не трогать (оставить как есть после инициализации портов)
*   2 = Принудительно выключить все ключи
*   3 = Принудительно включить все ключи

### 6. Логирование
```cpp
// MW_LOG авто-инклудится из MWOS, не нужно подключать отдельно
MW_LOG_MODULE(this, &p_turn, index);
MW_LOG(F("Значение: "));
MW_LOG_LN(_turn.getBit(index));
```

## Пример полного модуля ключа

```cpp
#ifndef MWOS3_MWOSKEYCUSTOM_H
#define MWOS3_MWOSKEYCUSTOM_H

#include "core/MWOSKey.h"

/**
* Пользовательский ключ (Версия MWOS3)
* 
* Параметры регистрируются автоматически через макросы MWOS_PARAM.
* Логика обработки данных объединена в onEvent.
*/
template<MWOS_PARAM_INDEX_UINT keysCount>
class MWOSKeyCustom : public MWOSKey<keysCount> {
public:
    // номер порта для каждого ключа
    MWOS_PIN_INT _pin[keysCount];

    // порт ключа
    MWOS_PARAM(7, pin, MWOS_PIN_INT_PTYPE, mwos_param_pin, MWOS_STORAGE_FOR_PIN, keysCount);

    MWOSKeyCustom() : MWOSKey<keysCount>() {
        for (MWOS_PARAM_INDEX_UINT i = 0; i < keysCount; ++i) {
            _pin[i] = -1;
        }
    }

    virtual void onEvent(MWOSModeEvent modeEvent) override {
        if (modeEvent == EVENT_INIT || modeEvent == EVENT_CHANGE) {
            for (MWOS_PARAM_INDEX_UINT i = 0; i < keysCount; ++i) {
                _pin[i] = MWOSModule::loadValue(_pin[i], &p_pin, i);
                initKey(i);
            }
        }
        
        MWOSKey<keysCount>::onEvent(modeEvent);
    }

    virtual uint8_t turn(uint8_t mode, int16_t index, bool saveToStorage) override {
        uint8_t v = MWOSKeyBase<keysCount>::turn(mode, index, saveToStorage);
        if (_pin[index] >= 0) mwos.writeValueDigital(_pin[index], v);
        return v;
    }
};

#endif
```

## См. также

*   **MWOSModule**: Базовый класс для всех модулей контроллера. Реализует общую инфраструктуру модулей, параметры и события.
*   **MWOSModuleBase**: Базовый класс модулей. Предоставляет методы работы с параметрами и событиями.
*   **MWBitsMaskStat**: Класс битовой маски для эффективного хранения состояний ключей.
*   **MWOSSensorBase**: Базовый класс для всех датчиков. Может использоваться вместе с модулями ключей для автоматизации.
