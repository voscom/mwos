# MWOSKeySchedule

**Имя модуля:** `key`

**MWOSKeySchedule** — расширение ключей с расширенной функциональностью для автоматического управления по таймеру и расписанию. Может самостоятельно выключаться и/или включаться через заданный таймаут, а также работать по суточному расписанию.

## Описание

Класс предназначен для:
*   **Автоматическое выключение:** Ключ автоматически выключается после заданного времени включения
*   **Автоматическое включение:** Ключ автоматически включается после заданного времени выключения
*   **Суточное расписание:** Ключ включается/выключается ежедневно в заданное время (минут с полуночи)
*   **Гибкое наследование:** Работает с любым типом ключей, наследующимся от MWOSKeyBase

**Наследование:** `MWOSKeySchedule<keysCount, MWOSKeyForSchedule>` → `MWOSKeyForSchedule` → `MWOSKeyBase<keysCount>` → `MWOSModule`

**Тип модуля:** `MODULE_KEY`

**Зависимости:**
*   Библиотека `MWOSKey.h` (базовый класс ключей)
*   Библиотека `MWTimeout.h` (таймеры)
*   Библиотека `MWBitsMaskStat.h` (битовая маска состояний)
*   Модуль времени `MWOSTime` (для работы расписания)

## Основные возможности

*   **Таймаут включения:** Автоматическое выключение ключа после заданного времени работы
*   **Таймаут выключения:** Автоматическое включение ключа после заданного времени простоя
*   **Суточное расписание:** Принудительное включение/выключение/переключение в заданную минуту суток
*   **Поддержка множественных ключей:** До `keysCount` независимых ключей с индивидуальными настройками
*   **Гибкое наследование:** Работает с MWOSKey, MWOSKeyAnalog и другими наследниками MWOSKeyBase
*   **Сохранение настроек:** Все параметры сохраняются в EEPROM и восстанавливаются после перезагрузки

## Параметры конфигурации

Класс использует параметры базовых классов и добавляет свои специфические параметры:

| ID | Название | Тип | Группа | Хранилище | Кол-во | Описание |
|:--:|:---|:---|:---|:---|:---:|:---|
| 0 | `turn` | bits1 | control | RTC | keysCount | **Состояния ключей.** 0=выкл, 1=вкл. Восстанавливается после перезагрузки |
| 1 | `invert` | bits1 | option | EEPROM | 1 | **Инверсия.** Инвертировать логическое состояние всех ключей |
| 2 | `alwaysoff` | bits1 | option | EEPROM | 1 | **Всегда отключено.** Принудительное отключение всех ключей |
| 3 | `checkbox` | bits1 | option | EEPROM | 1 | **Режим чекбокса.** При включении одного ключа отключать остальные |
| 4 | `startTurn` | bits2 | option | EEPROM | 1 | **Стартовое состояние.** 0=восстановить;1=не трогать;2=выключено;3=включено |
| 6 | `pull` | bits3 | option | EEPROM | 1 | **Подтяжка портов.** Режим (Выход;Открытый коллектор;ШИМ;ЦАП) |
| 7 | `pin` | int16 | pin | FOR_PIN | keysCount | **Порты ключей.** Номера портов для каждого ключа |
| 11 | `timeon` | uint32 | option | EEPROM | keysCount | **Время включения.** Максимальное время включения ключа [сек/10] |
| 12 | `timeoff` | uint32 | option | EEPROM | keysCount | **Время выключения.** Максимальное время выключения ключа [сек/10] |
| 13 | `schedule` | int16 | time+option | EEPROM | keysCount | **Расписание.** Время для расписания (минут с полуночи) |
| 14 | `scheduleTurn` | bits2 | option | EEPROM | keysCount | **Действие расписания.** 0=Без расписания;1=Выключать;2=Включать;3=Переключать |

**Примечание:**
*   **Кол-во:** Параметры с `keysCount` имеют индивидуальное значение для каждого ключа. Остальные параметры — по 1 значению.
*   **Единицы измерения:** `timeon`, `timeoff` — десятые доли секунды. `schedule` — минуты с полуночи (0-1439).
*   **Модуль времени:** Для работы расписания необходим инициализированный модуль времени `MWOSTime`.

## Конфигурация модуля

```
; модуль key - ключи с таймерами и расписанием
; 0 - Состояния ключей (keysCount значений, 0=выкл, 1=вкл)
key:turn = 0, 0, 0, 0, 0
; 1 - Инвертировать значение
key:invert = 0
; 2 - Всегда отключено
key:alwaysoff = 0
; 3 - Режим чекбокса (0=нет, 1=выключить другие при включении этого)
key:checkbox = 0
; 4 - Стартовое состояние (0=восстановить;1=не трогать;2=выключено;3=включено)
key:startTurn = 0
; 6 - Подтяжка портов (0=Выход;1=Открытый коллектор;2=ШИМ;3=ЦАП)
key:pull = 0
; 7 - Порты ключей (keysCount значений)
key:pin = 5, 6, 7, 8, -1
; 11 - Максимальное время включения ключа [сек/10] (keysCount значений)
key:timeon = 0, 0, 0, 0, 0
; 12 - Максимальное время выключения ключа [сек/10] (keysCount значений)
key:timeoff = 0, 0, 0, 0, 0
; 13 - Время для расписания (минут с полуночи) (keysCount значений)
key:schedule = 0, 0, 0, 0, 0
; 14 - Как реагировать на расписание (0=Без расписания;1=Выключать;2=Включать;3=Переключать)
key:scheduleTurn = 0, 0, 0, 0, 0
```

**Примечание:**
*   **Кол-во значений:** Параметры `turn`, `pin`, `timeon`, `timeoff`, `schedule`, `scheduleTurn` имеют `keysCount` значений (по одному на каждый ключ). Остальные параметры — по 1 значению.
*   **Единицы измерения:** `timeon`, `timeoff` — десятые доли секунды (10 = 1 секунда, 600 = 1 минута, 36000 = 1 час). `schedule` — минуты с полуночи (0 = 00:00, 720 = 12:00, 1439 = 23:59).
*   **Отключение функций:** Значение 0 в параметрах `timeon`, `timeoff`, `scheduleTurn` отключает соответствующую функцию.

## Конструкторы

Класс является шаблонным: `MWOSKeySchedule<keysCount, MWOSKeyForSchedule>`.

1.  **По умолчанию**
    ```cpp
    MWOSKeySchedule();
    ```
    Создаёт модуль с параметрами по умолчанию. Все порты инициализируются значением -1 (не подключены).

2.  **Из массива в PROGMEM**
    ```cpp
    MWOSKeySchedule(const uint8_t * pins);
    ```
    Создаёт модуль с портами из массива в памяти PROGMEM. Позволяет задать конфигурацию портов на этапе компиляции.

3.  **Одиночный пин**
    ```cpp
    MWOSKeySchedule(MWOS_PIN_INT pin);
    ```
    Создаёт модуль с одним ключом на указанном порту. Удобно для простых конфигураций с одним реле.

4.  **С указанием типа ключа**
    ```cpp
    MWOSKeySchedule<4, MWOSKeyAnalog<4>> keySchedule;
    ```
    Создаёт модуль с аналоговыми ключами (ШИМ/ЦАП) вместо стандартных цифровых.

## Основные методы

### Управление ключами

*   `virtual uint8_t turn(uint8_t mode, int16_t index, bool saveToStorage)` *(переопределён)*
    *   Изменяет значение ключа и запускает соответствующий таймер.
    *   **Параметры:** `mode` (0=выключить, 1=включить, 2=переключить), `index` (номер ключа), `saveToStorage` (сохранять в хранилище).
    *   При включении ключа запускает таймер `timeon`.
    *   При выключении ключа запускает таймер `timeoff`.
    *   **Возвращает:** `uint8_t` — фактическое состояние порта с учётом инверсии.

*   `void start(int16_t index)` *(унаследован)*
    *   Включить ключ.

*   `void stop(int16_t index)` *(унаследован)*
    *   Выключить ключ.

*   `void restart(int16_t index)` *(унаследован)*
    *   Переключить ключ в противоположное состояние.

### Инициализация

*   `virtual void onEvent(MWOSModeEvent modeEvent)` *(переопределён)*
    *   **`EVENT_INIT` / `EVENT_CHANGE`:** Загрузка настроек из хранилища (`timeon`, `timeoff`, `schedule`, `scheduleTurn`).
    *   **`EVENT_UPDATE`:** Проверка таймеров включения/выключения, обработка расписания по минутам суток.

## Пример использования

### Базовое подключение с таймерами

```cpp
#include "MWOSKeySchedule.h"

// Создаём тип для 4 ключей с расписанием
MWOSKeySchedule<4> keySchedule;

// Массив пинов в PROGMEM
const uint8_t PROGMEM pins[] = {5, 6, 7, 8};

// Экземпляр модуля
keySchedule(pins);

void setup() {
    // Настройки по умолчанию (загружаются только если хранилище не инициализировано)
    keySchedule._timeon[0] = 36000;    // Ключ 0: выключить через 1 час (36000 × 0.1 сек)
    keySchedule._timeoff[0] = 72000;   // Ключ 0: включить через 2 часа
    keySchedule._schedule[1] = 480;    // Ключ 1: расписание на 8:00 (480 минут с полуночи)
    keySchedule._scheduleTurn[1] = 2;  // Ключ 1: действие - включить
    
    // Запуск системы - загружает настройки из хранилища
    mwos.start();
}

void loop() {
    // Все действия выполняются внутри модулей через обработчики событий
    mwos.update();
}
```

### Подключение с аналоговыми ключами

```cpp
#include "MWOSKeySchedule.h"
#include "MWOSKeyAnalog.h"

// Создаём модуль с аналоговыми ключами (ШИМ)
MWOSKeySchedule<4, MWOSKeyAnalog<4>> keySchedule;

void setup() {
    // Настройка таймеров для каждого ключа
    keySchedule._timeon[0] = 6000;     // 10 минут
    keySchedule._timeoff[0] = 12000;   // 20 минут
    
    // Настройка расписания
    keySchedule._schedule[0] = 420;    // 7:00 утра
    keySchedule._scheduleTurn[0] = 2;  // Включить
    
    keySchedule._schedule[1] = 1320;   // 22:00 вечера
    keySchedule._scheduleTurn[1] = 1;  // Выключить
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
}
```

### Настройка суточного расписания

```cpp
#include "MWOSKeySchedule.h"

// Создаём модуль с 2 ключами
MWOSKeySchedule<2> keySchedule;

void setup() {
    // Ключ 0: включение в 8:00, выключение в 20:00
    keySchedule._schedule[0] = 480;     // 8:00 (480 минут с полуночи)
    keySchedule._scheduleTurn[0] = 2;   // Включить
    
    // Ключ 1: выключение в 23:00
    keySchedule._schedule[1] = 1380;    // 23:00 (1380 минут с полуночи)
    keySchedule._scheduleTurn[1] = 1;   // Выключить
    
    // Таймауты отключены (0)
    keySchedule._timeon[0] = 0;
    keySchedule._timeoff[0] = 0;
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
}
```

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `turn` | RTC | Состояния ключей, восстанавливаются после перезагрузки |
| `pin` | FOR_PIN | EEPROM или NO в зависимости от конфигурации |
| `pull` | EEPROM | Режим подтяжки портов |
| `timeon`, `timeoff` | EEPROM | Таймеры включения/выключения, сохраняются |
| `schedule`, `scheduleTurn` | EEPROM | Настройки расписания, сохраняются |
| `invert`, `alwaysoff`, `checkbox` | EEPROM | Флаги режима работы, сохраняются |
| `startTurn` | EEPROM | Поведение при старте, сохраняется |

## Важные замечания

1.  **Модуль времени:** Для работы суточного расписания необходим инициализированный модуль времени `MWOSTime`. Без него расписание не будет выполняться. Проверьте `MWOSTime::getTimeModule()!=NULL` и `IsTimeSetting()`.

2.  **Порядок инициализации:** Все настройки по умолчанию должны задаваться **до** вызова `mwos.start()`. После запуска системы значения загружаются из хранилища, и повторная установка значений в коде будет перезаписывать сохранённые настройки при каждой перезагрузке.

3.  **Единицы измерения таймеров:** Параметры `timeon` и `timeoff` задаются в десятых долях секунды. Значение 600 = 60 секунд = 1 минута. Значение 36000 = 3600 секунд = 1 час.

4.  **Формат расписания:** Параметр `schedule` задаётся в минутах с полуночи (0-1439). 0 = 00:00, 60 = 01:00, 720 = 12:00, 1380 = 23:00.

5.  **Режимы расписания:** Параметр `scheduleTurn` определяет действие:
    *   0 = Без расписания (отключено)
    *   1 = Выключать ключ в заданное время
    *   2 = Включать ключ в заданное время
    *   3 = Переключить ключ в заданное время

6.  **Таймеры и расписание одновременно:** Модуль поддерживает одновременную работу таймеров и расписания. Таймеры запускаются при изменении состояния ключа, расписание срабатывает в заданную минуту суток.

7.  **Архитектура MWOS:** Все действия выполняются внутри модулей через обработчики событий. Пользовательский код в `loop()` должен содержать только вызов `mwos.update()`. Для расширения функциональности создавайте собственные модули, наследуясь от `MWOSModule`.

8.  **Сохранение состояния:** Состояния ключей сохраняются в RTC (энергонезависимое ОЗУ) при каждом изменении. Это обеспечивает восстановление состояния после перезагрузки без износа EEPROM.

9.  **Проверка времени:** Расписание срабатывает только если время установлено (`IsTimeSetting()`). До синхронизации времени расписание не выполняется.

10. **Физическая коммутация:** Для реальной коммутации портов необходимо указать тип ключа в шаблоне (MWOSKey, MWOSKeyAnalog и др.). Модуль автоматически вызовет метод `turn()` родительского класса для физической коммутации.

## См. также

*   **MWOSKey**: Базовый модуль ключей для управления портами микроконтроллера. Предоставляет физическую коммутацию реле и транзисторов.
*   **MWOSKeyBase**: Базовый класс для всех модулей ключей. Реализует общую логику управления состояниями ключей, инверсию и режим чекбокса.
*   **MWOSKeyAnalog**: Модуль аналоговых ключей с поддержкой ШИМ и ЦАП. Может использоваться вместо MWOSKey для плавного управления.
*   **MWOSKeySwitch**: Модуль ключей, управляемых внешним датчиком. Переключает ключ при изменении бинарного значения датчика.
*   **MWOSTime**: Модуль времени и даты. Необходим для работы суточного расписания.
