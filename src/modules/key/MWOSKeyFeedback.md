# MWOSKeyFeedback

**Имя модуля:** `key`

**MWOSKeyFeedback** — модуль ключа управления мощностью на основе показаний аналогового датчика. При изменении аналогового значения датчика автоматически регулирует мощность ключа (ШИМ/ЦАП) в заданном диапазоне.

## Описание

Класс предназначен для:
*   **Автоматическая регулировка мощности:** Плавное изменение мощности ключа в зависимости от показаний датчика
*   **Обратная связь от датчика:** Использование аналогового значения датчика (параметр 9) для управления ключом
*   **Настраиваемый диапазон:** Настройка минимальной и максимальной мощности регулировки
*   **Плавность регулировки:** Настройка скорости и шага изменения мощности

**Наследование:** `MWOSKeyFeedback<keysCount, MWOSKeyForSwitch>` → `MWOSKeyForSwitch` → `MWOSKeyAnalog<keysCount>` → `MWOSKey<keysCount>` → `MWOSKeyBase<keysCount>` → `MWOSModule`

**Тип модуля:** `MODULE_KEY`

**Зависимости:**
*   Библиотека `MWOSKeyAnalog.h` (базовый класс ключей)
*   Библиотека `MWTimeout.h` (таймеры регулировки)
*   Платформа с поддержкой ШИМ/ЦАП (для аналоговых ключей)

## Основные возможности

*   **Регулировка по датчику:** Автоматическое изменение мощности ключа при изменении показаний аналогового датчика
*   **Настраиваемый диапазон мощности:** Установка минимального и максимального значения мощности (1-255)
*   **Плавность регулировки:** Настройка времени между изменениями мощности и шага изменения
*   **Поддержка множественных ключей:** До `keysCount` независимых ключей, каждый может управляться своим датчиком
*   **Гибкое наследование:** Работает с MWOSKeyAnalog и другими наследниками MWOSKeyBase
*   **Сохранение настроек:** Все параметры сохраняются в EEPROM и восстанавливаются после перезагрузки

## Параметры конфигурации

Класс использует параметры базовых классов и добавляет свои специфические параметры:

| ID | Название | Тип | Группа | Хранилище | Кол-во | Описание |
|:--:|:---|:---|:---|:---|:---:|:---|
| 0 | `turn` | bits1 | control | RTC | keysCount | **Состояния ключей.** 0=выкл, 1=вкл. Восстанавливается после перезагрузки |
| 1 | `invert` | bits1 | option | EEPROM | 1 | **Инверсия.** Инвертировать логическое состояние всех ключей |
| 2 | `alwaysoff` | bits1 | option | EEPROM | 1 | **Всегда отключено.** Принудительное отключение всех ключей |
| 3 | `checkbox` | bits1 | option | EEPROM | 1 | **Режим чекбокса.** При включении одного ключа отключать остальные |
| 4 | `startTurn` | bits2 | option | EEPROM | 1 | **Стартовое состояние.** 0=восстановить;1=не трогать;2=выключено;3=включено |
| 6 | `pull` | bits3 | option | EEPROM | 1 | **Подтяжка портов.** Режим (Выход;Открытый коллектор;ШИМ;ЦАП) |
| 7 | `pin` | int16 | pin | FOR_PIN | keysCount | **Порты ключей.** Номера портов для каждого ключа |
| 8 | `value` | uint16 | option | EEPROM | keysCount | **Аналоговое значение.** Мощность ШИМ/ЦАП (0-255 или 0-4095) |
| 21 | `sensorModuleId` | int16 | module_id+option | EEPROM | 1 | **ID модуля датчика.** Номер модуля аналогового датчика для управления ключом |
| 22 | `sensorIndex` | int16 | option | EEPROM | keysCount | **Индекс датчика.** Индекс параметра 9 датчика для каждого ключа |
| 30 | `power` | int16 | option | EEPROM | 2 | **Диапазон мощности.** Минимальное и максимальное значение мощности |
| 31 | `changeTimeMS` | uint16 | option | EEPROM | 1 | **Время между изменениями.** Плавность регулировки [мс] |
| 32 | `stepPower` | int16 | option | EEPROM | 1 | **Шаг мощности.** Изменение мощности за один шаг |

**Примечание:**
*   **Кол-во:** Параметры с `keysCount` имеют индивидуальное значение для каждого ключа. Остальные параметры — по 1 значению.
*   **Параметр 30 (power):** Имеет 2 значения — минимальное (индекс 0) и максимальное (индекс 1) мощность.
*   **Параметр датчика:** Используется параметр 9 (analog value) датчика для управления мощностью.

## Конфигурация модуля

```
; модуль key - ключ управления мощностью от датчика
; 0 - Состояния ключей (keysCount значений, 0=выкл, 1=вкл)
key:turn = 0, 0, 0, 0, 0
; 1 - Инвертировать значение
key:invert = 0
; 2 - Всегда отключено
key:alwaysoff = 0
; 3 - Режим чекбокса (0=нет, 1=выключить другие при включении этого)
key:checkbox = 0
; 4 - Стартовое состояние (0=восстановить;1=не трогать;2=выключено;3=включено)
key:startTurn = 0
; 6 - Подтяжка портов (0=Выход;1=Открытый коллектор;2=ШИМ;3=ЦАП)
key:pull = 2
; 7 - Порты ключей (keysCount значений)
key:pin = 5, 6, 7, 8, -1
; 8 - Аналоговое значение мощности (keysCount значений)
key:value = 128, 128, 128, 128, 128
; 21 - ID модуля датчика для управления ключом
key:sensorModuleId = 10
; 22 - Индексы параметра 9 датчика для каждого ключа (keysCount значений)
key:sensorIndex = 0, 1, 2, 3, 4
; 30 - Диапазон мощности (2 значения: мин, макс)
key:power = 1, 255
; 31 - Время между изменениями мощности [мс]
key:changeTimeMS = 20
; 32 - Шаг изменения мощности за один раз
key:stepPower = 1
```

**Примечание:**
*   **Кол-во значений:** Параметры `turn`, `pin`, `value`, `sensorIndex` имеют `keysCount` значений (по одному на каждый ключ). Параметр `power` имеет 2 значения (мин/макс). Остальные параметры — по 1 значению.
*   **Единицы измерения:** `changeTimeMS` — миллисекунды. `stepPower` — единицы мощности (0-255 для ШИМ, 0-4095 для ЦАП). `power` — диапазон мощности.
*   **ID модуля датчика:** Должен быть не менее 2 (системные модули имеют ID 0 и 1).

## Конструкторы

Класс является шаблонным: `MWOSKeyFeedback<keysCount, MWOSKeyForSwitch>`.

1.  **По умолчанию**
    ```cpp
    MWOSKeyFeedback();
    ```
    Создаёт модуль с параметрами по умолчанию. Все порты инициализируются значением -1 (не подключены).

2.  **Из массива в PROGMEM**
    ```cpp
    MWOSKeyFeedback(const uint8_t * pins);
    ```
    Создаёт модуль с портами из массива в памяти PROGMEM. Позволяет задать конфигурацию портов на этапе компиляции.

3.  **Одиночный пин с датчиком**
    ```cpp
    MWOSKeyFeedback(MWOS_PIN_INT pin, int16_t sensor_id = -1);
    ```
    Создаёт модуль с одним ключом на указанном порту и опциональным ID датчика. Удобно для простых конфигураций.

4.  **С указанием типа ключа**
    ```cpp
    MWOSKeyFeedback<4, MWOSKeyAnalog<4>> keyFeedback;
    ```
    Создаёт модуль с аналоговыми ключами (ШИМ/ЦАП) для плавной регулировки мощности.

## Основные методы

### Управление ключами

*   `virtual void onEvent(MWOSModeEvent modeEvent)` *(переопределён)*
    *   **`EVENT_INIT` / `EVENT_CHANGE`:** Загрузка настроек из хранилища, инициализация ссылки на датчик.
    *   **`EVENT_UPDATE`:** Проверка показаний датчика, плавная регулировка мощности ключа.

*   `void updateKey(int16_t sensorIndex, int16_t keyIndex)`
    *   Обновляет мощность ключа в зависимости от показания датчика.
    *   Сравнивает значение датчика с диапазонами fromValue/toValue.
    *   Плавно изменяет мощность с заданным шагом и интервалом.

*   `void loadSensors()`
    *   Загружает ссылку на модуль датчика по ID.
    *   Проверяет корректность типа датчика (параметр 0 должен быть readonly bits1, параметр 9 — analog value).
    *   Считывает текущие значения датчиков для синхронизации состояний.

### Инициализация

*   `void initKeySwitch()`
    *   Регистрирует параметры модуля (sensorModuleId, sensorIndex, power, changeTimeMS, stepPower).
    *   Инициализирует массивы индексов датчиков.

## Пример использования

### Базовое подключение

```cpp
#include "MWOSKeyFeedback.h"

// Создаём тип для 4 ключей с регулировкой от датчика
MWOSKeyFeedback<4> keyFeedback;

// Массив пинов в PROGMEM
const uint8_t PROGMEM pins[] = {5, 6, 7, 8};

// Экземпляр модуля
keyFeedback(pins);

void setup() {
    // Настройки по умолчанию (загружаются только если хранилище не инициализировано)
    // keyFeedback._sensorModuleId = 10;  // ID модуля аналогового датчика
    
    // Запуск системы - загружает настройки из хранилища
    mwos.start();
}

void loop() {
    // Все действия выполняются внутри модулей через обработчики событий
    mwos.update();
}
```

### Настройка диапазона мощности

```cpp
#include "MWOSKeyFeedback.h"

// Создаём модуль с аналоговыми ключами (ШИМ)
MWOSKeyFeedback<2, MWOSKeyAnalog<2>> keyFeedback;

void setup() {
    // Настройка ID датчика (загружается только если хранилище не инициализировано)
    keyFeedback._sensorModuleId = 10;  // ID модуля аналогового датчика
    
    // Настройка диапазона мощности
    keyFeedback._powerMin = 50;    // Минимальная мощность (50 из 255)
    keyFeedback._powerMax = 200;   // Максимальная мощность (200 из 255)
    
    // Настройка плавности регулировки
    keyFeedback._changeTimeMS = 50;  // 50 мс между изменениями
    keyFeedback._stepPower = 2;      // Шаг изменения 2 единицы
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
}
```

### Подключение с несколькими датчиками

```cpp
#include "MWOSKeyFeedback.h"

// Создаём модуль с 4 ключами
MWOSKeyFeedback<4> keyFeedback;

void setup() {
    // Настройка ID датчика
    keyFeedback._sensorModuleId = 15;  // ID модуля с 4 аналоговыми датчиками
    
    // Настройка индексов датчиков для каждого ключа
    keyFeedback._sensorIndex[0] = 0;  // Ключ 0 управляется датчиком 0
    keyFeedback._sensorIndex[1] = 1;  // Ключ 1 управляется датчиком 1
    keyFeedback._sensorIndex[2] = 2;  // Ключ 2 управляется датчиком 2
    keyFeedback._sensorIndex[3] = 3;  // Ключ 3 управляется датчиком 3
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
}
```

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `turn` | RTC | Состояния ключей, восстанавливаются после перезагрузки |
| `pin` | FOR_PIN | EEPROM или NO в зависимости от конфигурации |
| `pull` | EEPROM | Режим подтяжки портов |
| `value` | EEPROM | Аналоговые значения мощности, сохраняются |
| `sensorModuleId` | EEPROM | ID модуля датчика, сохраняется |
| `sensorIndex` | EEPROM | Индексы датчиков, сохраняются |
| `power` | EEPROM | Диапазон мощности (мин/макс), сохраняется |
| `changeTimeMS`, `stepPower` | EEPROM | Параметры регулировки, сохраняются |
| `invert`, `alwaysoff`, `checkbox` | EEPROM | Флаги режима работы, сохраняются |
| `startTurn` | EEPROM | Поведение при старте, сохраняется |

## Важные замечания

1.  **ID модуля датчика:** Параметр `sensorModuleId` должен быть не менее 2. Системные модули имеют ID 0 и 1 и не могут использоваться как датчики.

2.  **Тип параметра датчика:** Модуль использует параметр 9 (analog value) датчика для управления мощностью. Убедитесь, что датчик поддерживает этот параметр (MWOSSensorAnalog и наследники).

3.  **Отложенная инициализация:** Ссылка на модуль датчика загружается не сразу, а через таймаут 0.5 секунды после старта. Это позволяет всем модулям системы инициализироваться перед попыткой подключения.

4.  **Плавность регулировки:** Параметры `changeTimeMS` и `stepPower` управляют плавностью регулировки мощности. Меньшие значения дают более быструю, но менее плавную регулировку.

5.  **Диапазон мощности:** Параметр `power` имеет два значения — минимальное (индекс 0) и максимальное (индекс 1). Мощность ключа не выйдет за пределы этого диапазона.

6.  **Аналоговые ключи:** Для плавной регулировки мощности используйте MWOSKeyAnalog вместо MWOSKey. Это обеспечит поддержку ШИМ/ЦАП для плавного изменения мощности.

7.  **Архитектура MWOS:** Все действия выполняются внутри модулей через обработчики событий. Пользовательский код в `loop()` должен содержать только вызов `mwos.update()`.

8.  **Проверка датчика:** При инициализации модуль проверяет существование датчика и корректность типа параметра. При ошибке ссылка на датчик сбрасывается и повторяется попытка через таймаут.

9.  **Сохранение состояния:** Состояния ключей сохраняются в RTC (энергонезависимое ОЗУ) при каждом изменении. Это обеспечивает восстановление состояния после перезагрузки без износа EEPROM.

10. **Режим чекбокса:** При включённом параметре `checkbox` изменение мощности одного ключа может влиять на состояние остальных ключей модуля.

11. **Параметры датчика:** Модуль использует параметры 9 (value), 10 (fromValue), 11 (toValue) датчика для определения диапазона регулировки мощности.

## См. также

*   **MWOSKeyBase**: Базовый класс для всех модулей ключей. Реализует общую логику управления состояниями ключей, инверсию и режим чекбокса.
*   **MWOSKey**: Базовый модуль ключей для управления портами микроконтроллера. Предоставляет физическую коммутацию реле и транзисторов.
*   **MWOSKeyAnalog**: Модуль аналоговых ключей с поддержкой ШИМ и ЦАП. Необходим для плавной регулировки мощности.
*   **MWOSKeySchedule**: Модуль ключей с таймерами и расписанием. Может использоваться вместо MWOSKey для расширенной функциональности.
*   **MWOSSensorAnalog**: Базовый класс для аналоговых датчиков. Предоставляет параметр 9 (analog value) для управления ключом.
