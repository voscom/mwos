# MWOSKeyAnalog

**Имя модуля:** `key`

**MWOSKeyAnalog** — модуль ключей с аналоговыми значениями для управления мощностью через ШИМ или ЦАП. Позволяет плавно регулировать мощность подключенных устройств.

## Описание

Класс предназначен для:
*   **Управление мощностью:** Плавная регулировка мощности через ШИМ (PWM) или ЦАП (DAC)
*   **Множественные ключи:** Поддержка до `keysCount` независимых аналоговых ключей
*   **Аналоговые значения:** Каждому ключу соответствует аналоговое значение (0-255 для ШИМ, 0-4095 для ЦАП)
*   **Гибкое наследование:** Может использоваться как базовый класс для более специализированных модулей ключей

**Наследование:** `MWOSKeyAnalog<keysCount>` → `MWOSKey<keysCount>` → `MWOSKeyBase<keysCount>` → `MWOSModule`

**Тип модуля:** `MODULE_KEY`

**Зависимости:**
*   Библиотека `MWOSKey.h` (базовый класс ключей)
*   Платформа с поддержкой ШИМ/ЦАП (ESP32, STM32 и др.)

## Основные возможности

*   **Аналоговое управление:** Поддержка ШИМ (PWM) и ЦАП (DAC) для плавной регулировки мощности
*   **Множественные каналы:** До `keysCount` независимых аналоговых ключей в одном модуле
*   **Сохранение значений:** Аналоговые значения сохраняются в EEPROM и восстанавливаются после перезагрузки
*   **Режимы работы:** Поддержка режимов Выход, Открытый коллектор, ШИМ, ЦАП
*   **Гибкое наследование:** Может использоваться как базовый класс для MWOSKeyFeedback, MWOSHzESP32 и других модулей

## Параметры конфигурации

Класс использует параметры базовых классов и добавляет свои специфические параметры:

| ID | Название | Тип | Группа | Хранилище | Кол-во | Описание |
|:--:|:---|:---|:---|:---|:---:|:---|
| 0 | `turn` | bits1 | control | RTC | keysCount | **Состояния ключей.** 0=выкл, 1=вкл. Восстанавливается после перезагрузки |
| 1 | `invert` | bits1 | option | EEPROM | 1 | **Инверсия.** Инвертировать логическое состояние всех ключей |
| 2 | `alwaysoff` | bits1 | option | EEPROM | 1 | **Всегда отключено.** Принудительное отключение всех ключей |
| 3 | `checkbox` | bits1 | option | EEPROM | 1 | **Режим чекбокса.** При включении одного ключа отключать остальные |
| 4 | `startTurn` | bits2 | option | EEPROM | 1 | **Стартовое состояние.** 0=восстановить;1=не трогать;2=выключено;3=включено |
| 6 | `pull` | bits3 | option | EEPROM | 1 | **Подтяжка портов.** Режим (Выход;Открытый коллектор;ШИМ;ЦАП) |
| 7 | `pin` | int16 | pin | FOR_PIN | keysCount | **Порты ключей.** Номера портов для каждого ключа |
| 8 | `value` | uint16 | option | EEPROM | keysCount | **Аналоговое значение.** Мощность ШИМ (0-255) или ЦАП (0-4095) |

**Примечание:**
*   **Кол-во:** Параметры с `keysCount` имеют индивидуальное значение для каждого ключа. Остальные параметры — по 1 значению.
*   **Параметр 8 (value):** Диапазон зависит от режима: 0-255 для ШИМ, 0-4095 для ЦАП (ESP32).
*   **Параметр 6 (pull):** Формат значений: "Выход;Открытый коллектор;ШИМ;ЦАП"

## Конфигурация модуля

```
; модуль key - ключи с аналоговыми значениями (ШИМ/ЦАП)
; 0 - Состояния ключей (keysCount значений, 0=выкл, 1=вкл)
key:turn = 0, 0, 0, 0, 0
; 1 - Инвертировать значение
key:invert = 0
; 2 - Всегда отключено
key:alwaysoff = 0
; 3 - Режим чекбокса (0=нет, 1=выключить другие при включении этого)
key:checkbox = 0
; 4 - Стартовое состояние (0=восстановить;1=не трогать;2=выключено;3=включено)
key:startTurn = 0
; 6 - Подтяжка портов (0=Выход;1=Открытый коллектор;2=ШИМ;3=ЦАП)
key:pull = 2
; 7 - Порты ключей (keysCount значений)
key:pin = 5, 6, 7, 8, -1
; 8 - Аналоговое значение мощности (keysCount значений, 0-255 для ШИМ)
key:value = 128, 128, 128, 128, 128
```

**Примечание:**
*   **Кол-во значений:** Параметры `turn`, `pin`, `value` имеют `keysCount` значений (по одному на каждый ключ). Остальные параметры — по 1 значению.
*   **Единицы измерения:** `value` — единицы мощности (0-255 для ШИМ, 0-4095 для ЦАП на ESP32).
*   **Режим ШИМ/ЦАП:** Задаётся параметром `pull` (2=ШИМ, 3=ЦАП).

## Конструкторы

Класс является шаблонным: `MWOSKeyAnalog<keysCount>`.

1.  **По умолчанию**
    ```cpp
    MWOSKeyAnalog();
    ```
    Создаёт модуль с параметрами по умолчанию. Все порты инициализируются значением -1 (не подключены).

2.  **Из массива в PROGMEM**
    ```cpp
    MWOSKeyAnalog(const uint8_t * pins);
    ```
    Создаёт модуль с портами из массива в памяти PROGMEM. Позволяет задать конфигурацию портов на этапе компиляции.

3.  **Одиночный пин**
    ```cpp
    MWOSKeyAnalog(MWOS_PIN_INT pin);
    ```
    Создаёт модуль с одним ключом на указанном порту. Удобно для простых конфигураций с одним аналоговым ключом.

## Основные методы

### Чтение значений

*   `virtual int64_t getValue(MWOSParam * param, int16_t arrayIndex)` *(переопределён)*
    *   Возвращает значение запрошенного параметра.
    *   Для параметра 8 (value) возвращает аналоговое значение мощности.
    *   **Возвращает:** `int64_t` — значение параметра.

*   `void setKeyValue(uint8_t mode, int16_t index)`
    *   Устанавливает аналоговое значение ключа (ШИМ/ЦАП).
    *   **Параметры:** `mode` (режим вывода), `index` (номер ключа).

### Инициализация

*   `virtual void onEvent(MWOSModeEvent modeEvent)` *(переопределён)*
    *   **`EVENT_INIT` / `EVENT_CHANGE`:** Загрузка настроек из хранилища, инициализация аналоговых значений.
    *   **`EVENT_UPDATE`:** Вызывает обработку базового класса для управления ключами.

*   `virtual uint8_t turn(uint8_t mode, int16_t index, bool saveToStorage)` *(переопределён)*
    *   Изменяет значение ключа и устанавливает аналоговое значение.
    *   **Параметры:** `mode` (0=выключить, 1=включить, 2=переключить), `index` (номер ключа), `saveToStorage` (сохранять в хранилище).
    *   **Возвращает:** `uint8_t` — фактическое состояние порта с учётом инверсии.

## Пример использования

### Базовое подключение

```cpp
#include "MWOSKeyAnalog.h"

// Создаём тип для 4 аналоговых ключей
MWOSKeyAnalog<4> keyAnalog;

// Массив пинов в PROGMEM
const uint8_t PROGMEM pins[] = {5, 6, 7, 8};

// Экземпляр модуля
keyAnalog(pins);

void setup() {
    // Настройки по умолчанию (загружаются только если хранилище не инициализировано)
    // keyAnalog._pull = 2;  // Режим ШИМ
    
    // Запуск системы - загружает настройки из хранилища
    mwos.start();
}

void loop() {
    // Все действия выполняются внутри модулей через обработчики событий
    mwos.update();
}
```

### Настройка аналоговых значений

```cpp
#include "MWOSKeyAnalog.h"

// Создаём модуль с 4 аналоговыми ключами (ШИМ)
MWOSKeyAnalog<4> keyAnalog;

void setup() {
    // Настройка режима ШИМ (загружается только если хранилище не инициализировано)
    keyAnalog._pull = 2;  // 2=ШИМ, 3=ЦАП
    
    // Настройка начальных значений мощности (0-255 для ШИМ)
    keyAnalog._value[0] = 128;  // 50% мощности
    keyAnalog._value[1] = 192;  // 75% мощности
    keyAnalog._value[2] = 255;  // 100% мощности
    keyAnalog._value[3] = 64;   // 25% мощности
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
    
    // Изменение мощности первого ключа
    keyAnalog._value[0] = 200;
    keyAnalog.turn(1, 0, false);
}
```

### Подключение с режимом ЦАП (ESP32)

```cpp
#include "MWOSKeyAnalog.h"

#ifdef ESP32
// Создаём модуль с 2 ключами в режиме ЦАП
MWOSKeyAnalog<2> keyAnalog;

void setup() {
    // Настройка режима ЦАП
    keyAnalog._pull = 3;  // 3=ЦАП (только ESP32)
    
    // Настройка значений (0-4095 для ЦАП ESP32)
    keyAnalog._value[0] = 2048;  // 50% от 4095
    keyAnalog._value[1] = 3072;  // 75% от 4095
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
}
#endif
```

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `turn` | RTC | Состояния ключей, восстанавливаются после перезагрузки |
| `pin` | FOR_PIN | EEPROM или NO в зависимости от конфигурации |
| `pull` | EEPROM | Режим подтяжки портов, сохраняется |
| `value` | EEPROM | Аналоговые значения мощности, сохраняются |
| `invert`, `alwaysoff`, `checkbox` | EEPROM | Флаги режима работы, сохраняются |
| `startTurn` | EEPROM | Поведение при старте, сохраняется |

## Важные замечания

1.  **Режим ШИМ/ЦАП:** Параметр `pull` определяет режим работы ключа: 0=Выход, 1=Открытый коллектор, 2=ШИМ, 3=ЦАП. Для аналогового управления используйте режим 2 или 3.

2.  **Диапазон значений:** Для ШИМ диапазон значений 0-255. Для ЦАП на ESP32 диапазон 0-4095. Убедитесь, что значения `_value` соответствуют выбранному режиму.

3.  **Порядок инициализации:** Все настройки по умолчанию должны задаваться **до** вызова `mwos.start()`. После запуска системы значения загружаются из хранилища, и повторная установка значений в коде будет перезаписывать сохранённые настройки при каждой перезагрузке.

4.  **Поддержка платформ:** ШИМ поддерживается на большинстве платформ. ЦАП (DAC) доступен только на некоторых платформах (например, ESP32 имеет 2 канала ЦАП).

5.  **Гибкое наследование:** Класс MWOSKeyAnalog предназначен для наследования. На его основе построены MWOSKeyFeedback (регулировка от датчика), MWOSHzESP32 (генератор частоты) и другие модули.

6.  **Архитектура MWOS:** Все действия выполняются внутри модулей через обработчики событий. Пользовательский код в `loop()` должен содержать только вызов `mwos.update()`.

7.  **Сохранение состояния:** Состояния ключей сохраняются в RTC (энергонезависимое ОЗУ) при каждом изменении. Аналоговые значения сохраняются в EEPROM. Это обеспечивает восстановление состояния после перезагрузки.

8.  **Режим чекбокса:** При включённом параметре `checkbox` включение одного ключа автоматически отключает все остальные ключи модуля. Это полезно для взаимно исключающих состояний.

9.  **Физическая коммутация:** Метод `setKeyValue()` автоматически выбирает режим коммутации (ШИМ/ЦАП/цифровой) на основе параметра `pull`.

10. **Инициализация портов:** При инициализации модуль проверяет возможность настройки порта в заданном режиме. При ошибке записывает значение -3 в параметр `pin`.

## См. также

*   **MWOSKeyBase**: Базовый класс для всех модулей ключей. Реализует общую логику управления состояниями ключей, инверсию и режим чекбокса.
*   **MWOSKey**: Базовый модуль ключей для управления портами микроконтроллера. Предоставляет физическую коммутацию реле и транзисторов.
*   **MWOSKeyFeedback**: Модуль ключа управления мощностью на основе показаний аналогового датчика. Наследуется от MWOSKeyAnalog.
*   **MWOSHzESP32**: Регулятор частоты и скважности для ESP32. Наследуется от MWOSKeyAnalog.
*   **MWOSKeySchedule**: Модуль ключей с таймерами и расписанием. Может использоваться вместе с MWOSKeyAnalog для расширенной функциональности.
