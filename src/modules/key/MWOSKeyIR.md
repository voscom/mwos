# MWOSKeyIR

**Имя модуля:** `keyIR`

**MWOSKeyIR** — модуль управления ИК-приборами через инфракрасный светодиод. Отправляет команды включения/выключения для кондиционеров и других устройств с ИК-пультом.

## Описание

Класс предназначен для:
*   **Управление ИК-приборами:** Отправка ИК-команд для включения/выключения устройств
*   **Поддержка протоколов:** Работа с различными протоколами ИК-пультов (Daikin, Panasonic, LG и др.)
*   **Множественные устройства:** Поддержка до `keysCount` независимых ИК-устройств через один светодиод
*   **Автоматическая инициализация:** Настройка ИК-передатчика при инициализации модуля

**Наследование:** `MWOSKeyIR<keysCount>` → `MWOSKeyBase<keysCount>` → `MWOSModule`

**Тип модуля:** `MODULE_KEY`

**Зависимости:**
*   Библиотека `IRremoteESP8266` (https://github.com/crankyoldgit/IRremoteESP8266.git)
*   Библиотека `IRac.h`
*   Платформа ESP32/ESP8266 (требуется для работы библиотеки)

## Основные возможности

*   **Поддержка множественных протоколов:** Каждый ключ может использовать свой протокол ИК-пульта
*   **Один ИК-светодиод:** Все ключи управляют одним ИК-передатчиком
*   **Стандартные команды:** Включение/выключение/переключение состояния устройств
*   **Сохранение настроек:** Тип протокола для каждого ключа сохраняется в EEPROM
*   **Гибкая конфигурация:** Настройка порта ИК-светодиода через параметры

## Параметры конфигурации

Класс использует параметры базовых классов и добавляет свои специфические параметры:

| ID | Название | Тип | Группа | Хранилище | Кол-во | Описание |
|:--:|:---|:---|:---|:---|:---:|:---|
| 0 | `turn` | bits1 | control | RTC | keysCount | **Состояния ключей.** 0=выкл, 1=вкл. Восстанавливается после перезагрузки |
| 1 | `invert` | bits1 | option | EEPROM | 1 | **Инверсия.** Инвертировать логическое состояние всех ключей |
| 2 | `alwaysoff` | bits1 | option | EEPROM | 1 | **Всегда отключено.** Принудительное отключение всех ключей |
| 3 | `checkbox` | bits1 | option | EEPROM | 1 | **Режим чекбокса.** При включении одного ключа отключать остальные |
| 4 | `startTurn` | bits2 | option | EEPROM | 1 | **Стартовое состояние.** 0=восстановить;1=не трогать;2=выключено;3=включено |
| 6 | `pin` | int16 | pin | FOR_PIN | 1 | **Порт ИК-светодиода.** Номер порта для подключения ИК-передатчика |
| 7 | `typeIR` | uint8 | option | EEPROM | keysCount | **Тип ИК-протокола.** Протокол ИК-пульта для каждого ключа |

**Примечание:**
*   **Кол-во:** Параметры с `keysCount` имеют индивидуальное значение для каждого ключа. Остальные параметры — по 1 значению.
*   **Параметр 7 (typeIR):** Использует значения из библиотеки IRremoteESP8266 (decode_type_t). Например: DAIKIN=64, PANASONIC=53, LG=19.

## Конфигурация модуля

```
; модуль keyIR - управление ИК-приборами через инфракрасный светодиод
; 0 - Состояния ключей (keysCount значений, 0=выкл, 1=вкл)
keyIR:turn = 0, 0, 0, 0, 0
; 1 - Инвертировать значение
keyIR:invert = 0
; 2 - Всегда отключено
keyIR:alwaysoff = 0
; 3 - Режим чекбокса (0=нет, 1=выключить другие при включении этого)
keyIR:checkbox = 0
; 4 - Стартовое состояние (0=восстановить;1=не трогать;2=выключено;3=включено)
keyIR:startTurn = 0
; 6 - Порт ИК-светодиода
keyIR:pin = 4
; 7 - Тип ИК-протокола для каждого ключа (keysCount значений)
keyIR:typeIR = 64, 53, 19, 0, 0
```

**Примечание:**
*   **Кол-во значений:** Параметры `turn`, `typeIR` имеют `keysCount` значений (по одному на каждый ключ). Остальные параметры — по 1 значению.
*   **Типы протоколов:** Значения из библиотеки IRremoteESP8266 (decode_type_t). 0=UNKNOWN, 19=LG, 53=PANASONIC, 64=DAIKIN.
*   **Порт ИК-светодиода:** Должен поддерживать вывод PWM для работы ИК-передатчика.

## Конструкторы

Класс является шаблонным: `MWOSKeyIR<keysCount>`.

1.  **По умолчанию**
    ```cpp
    MWOSKeyIR();
    ```
    Создаёт модуль с параметрами по умолчанию. Порт ИК-светодиода инициализируется значением -1 (не подключён).

2.  **С указанием порта**
    ```cpp
    MWOSKeyIR(MWOS_PIN_INT pin);
    ```
    Создаёт модуль с ИК-светодиодом на указанном порту. Удобно для простых конфигураций с одним ИК-передатчиком.

## Основные методы

### Управление ключами

*   `virtual uint8_t turn(uint8_t mode, int16_t index, bool saveToStorage)` *(переопределён)*
    *   Изменяет значение ключа и отправляет ИК-команду.
    *   **Параметры:** `mode` (0=выключить, 1=включить, 2=переключить), `index` (номер ключа), `saveToStorage` (сохранять в хранилище).
    *   При изменении состояния отправляет ИК-сигнал через библиотеку IRac.
    *   **Возвращает:** `uint8_t` — фактическое состояние порта с учётом инверсии.

*   `void start(int16_t index)` *(унаследован)*
    *   Включить ключ (отправить ИК-команду включения).

*   `void stop(int16_t index)` *(унаследован)*
    *   Выключить ключ (отправить ИК-команду выключения).

*   `void restart(int16_t index)` *(унаследован)*
    *   Переключить ключ в противоположное состояние.

### Инициализация

*   `virtual void onEvent(MWOSModeEvent modeEvent)` *(переопределён)*
    *   **`EVENT_INIT` / `EVENT_CHANGE`:** Загрузка настроек из хранилища, инициализация объекта IRac при изменении порта.
    *   **`EVENT_GET_VALUE`:** Возврат текущих значений параметров.

## Пример использования

### Базовое подключение

```cpp
#include "MWOSKeyIR.h"

// Создаём тип для 3 ИК-устройств
MWOSKeyIR<3> keyIR;

// Экземпляр модуля с портом 4 для ИК-светодиода
keyIR(4);

void setup() {
    // Настройки по умолчанию (загружаются только если хранилище не инициализировано)
    // keyIR._typeIR[0] = 64;  // DAIKIN для первого устройства
    
    // Запуск системы - загружает настройки из хранилища
    mwos.start();
}

void loop() {
    // Все действия выполняются внутри модулей через обработчики событий
    mwos.update();
}
```

### Настройка протоколов для разных устройств

```cpp
#include "MWOSKeyIR.h"

// Создаём модуль для 3 ИК-устройств
MWOSKeyIR<3> keyIR;

void setup() {
    // Настройка протоколов для каждого устройства (загружаются только если хранилище не инициализировано)
    keyIR._typeIR[0] = 64;  // DAIKIN для кондиционера 1
    keyIR._typeIR[1] = 53;  // PANASONIC для кондиционера 2
    keyIR._typeIR[2] = 19;  // LG для кондиционера 3
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
    
    // Включение первого кондиционера
    keyIR.start(0);
    
    // Выключение второго кондиционера
    keyIR.stop(1);
}
```

### Управление с датчика

```cpp
#include "MWOSKeyIR.h"
#include "MWOSKeySwitch.h"

// Создаём модуль ИК-управления
MWOSKeyIR<1> keyIR(4);

// Создаём переключатель от датчика
MWOSKeySwitch<1, MWOSKeyIR<1>> keySwitch;

void setup() {
    // Настройка ID датчика для управления ИК-ключом
    keySwitch._sensorModuleId = 10;  // ID модуля бинарного датчика
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
}
```

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `turn` | RTC | Состояния ключей, восстанавливаются после перезагрузки |
| `pin` | FOR_PIN | EEPROM или NO в зависимости от конфигурации |
| `typeIR` | EEPROM | Типы ИК-протоколов, сохраняются |
| `invert`, `alwaysoff`, `checkbox` | EEPROM | Флаги режима работы, сохраняются |
| `startTurn` | EEPROM | Поведение при старте, сохраняется |

## Важные замечания

1.  **Библиотека IRremoteESP8266:** Требуется установка библиотеки https://github.com/crankyoldgit/IRremoteESP8266.git для работы модуля.

2.  **Платформа:** Модуль работает только на платформах ESP32/ESP8266, поддерживаемых библиотекой IRremoteESP8266.

3.  **Один ИК-светодиод:** Все ключи используют один ИК-передатчик. Команды отправляются последовательно, не одновременно.

4.  **Типы протоколов:** Используйте значения из перечисления `decode_type_t` библиотеки IRremoteESP8266. Популярные: DAIKIN=64, PANASONIC=53, LG=19, SAMSUNG=17.

5.  **Настройки ИК-команд:** По умолчанию используются стандартные настройки (температура 25°C, режим охлаждения, средняя скорость вентилятора). Для изменения настроек необходимо модифицировать код инициализации `ac->next`.

6.  **Порядок инициализации:** Все настройки по умолчанию должны задаваться **до** вызова `mwos.start()`. После запуска системы значения загружаются из хранилища.

7.  **Режим чекбокса:** При включённом параметре `checkbox` включение одного ключа автоматически отключает все остальные ключи модуля.

8.  **Архитектура MWOS:** Все действия выполняются внутри модулей через обработчики событий. Пользовательский код в `loop()` должен содержать только вызов `mwos.update()`.

9.  **Сохранение состояния:** Состояния ключей сохраняются в RTC (энергонезависимое ОЗУ) при каждом изменении. Это обеспечивает восстановление состояния после перезагрузки.

10. **Дальность ИК-сигнала:** Зависит от мощности ИК-светодиода и расстояния до прибора. Рекомендуется использовать ИК-светодиод с линзой для увеличения дальности.

## См. также

*   **MWOSKeyBase**: Базовый класс для всех модулей ключей. Реализует общую логику управления состояниями ключей, инверсию и режим чекбокса.
*   **MWOSKey**: Базовый модуль ключей для управления портами микроконтроллера. Предоставляет физическую коммутацию реле и транзисторов.
*   **MWOSKeySwitch**: Модуль ключей, управляемых внешним датчиком. Может использоваться для управления ИК-приборами от датчиков.
*   **Библиотека IRremoteESP8266**: https://github.com/crankyoldgit/IRremoteESP8266 — библиотека для отправки и получения ИК-сигналов.
*   **Библиотека IRac**: Класс для упрощённого управления кондиционерами через ИК.
