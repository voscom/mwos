# MWOSHzSTM32

**Имя модуля:** `hz`

**MWOSHzSTM32** — регулятор частоты и скважности на базе аппаратного таймера STM32. Позволяет генерировать ШИМ-сигналы с настраиваемой частотой на платформах STM32.

## Описание

Класс предназначен для:
*   **Генерация ШИМ:** Создание широтно-импульсной модуляции на портах STM32
*   **Настройка частоты:** Регулировка частоты ШИМ в широком диапазоне
*   **Аппаратный таймер:** Использование встроенных таймеров STM32 для точной генерации
*   **Одиночный канал:** Поддержка одного ШИМ-канала на модуль

**Наследование:** `MWOSHzSTM32` → `MWOSKey<1>` → `MWOSKeyBase<1>` → `MWOSModule`

**Тип модуля:** `MODULE_KEY`

**Зависимости:**
*   Платформа STM32 (требуется для работы библиотеки HardwareTimer)
*   Библиотека `HardwareTimer.h` (встроена в ядро STM32)
*   Библиотека `MWOSKey.h` (базовый класс ключей)

## Основные возможности

*   **Аппаратный ШИМ STM32:** Использование встроенных таймеров для генерации ШИМ
*   **Настраиваемая частота:** Установка частоты ШИМ (по умолчанию 490 Гц)
*   **Автоматическая конфигурация:** Автоматическое определение канала таймера по пину
*   **Сохранение настроек:** Частота сохраняется в EEPROM и восстанавливается после перезагрузки
*   **Управление через MWOS:** Полная интеграция с системой параметров и событий MWOS

## Параметры конфигурации

Класс использует параметры базовых классов и добавляет свои специфические параметры:

| ID | Название | Тип | Группа | Хранилище | Кол-во | Описание |
|:--:|:---|:---|:---|:---|:---:|:---|
| 0 | `turn` | bits1 | control | RTC | 1 | **Состояние ключа.** 0=выкл, 1=вкл. Восстанавливается после перезагрузки |
| 1 | `invert` | bits1 | option | EEPROM | 1 | **Инверсия.** Инвертировать логическое состояние ключа |
| 2 | `alwaysoff` | bits1 | option | EEPROM | 1 | **Всегда отключено.** Принудительное отключение ключа |
| 3 | `checkbox` | bits1 | option | EEPROM | 1 | **Режим чекбокса.** При включении отключать другие ключи |
| 4 | `startTurn` | bits2 | option | EEPROM | 1 | **Стартовое состояние.** 0=восстановить;1=не трогать;2=выключено;3=включено |
| 6 | `pull` | bits3 | option | EEPROM | 1 | **Подтяжка портов.** Режим (Выход;Открытый коллектор;ШИМ;ЦАП) |
| 7 | `pin` | int16 | pin | FOR_PIN | 1 | **Порт ключа.** Номер порта для ключа |
| 8 | `hz` | uint32 | option | EEPROM | 1 | **Частота ШИМ.** Частота в герцах (по умолчанию 490 Гц) |

**Примечание:**
*   **Кол-во:** Все параметры имеют 1 значение (одиночный канал ШИМ).
*   **Параметр 8 (hz):** Частота ШИМ в герцах. По умолчанию 490 Гц.
*   **Платформа:** Работает только на STM32. На других платформах код не компилируется.

## Конфигурация модуля

```

; 0 - Состояние ключа (0=выкл, 1=вкл)
hz:turn = 0
; 1 - Инвертировать значение
hz:invert = 0
; 2 - Всегда отключено
hz:alwaysoff = 0
; 3 - Режим чекбокса (0=нет, 1=выключить другие при включении этого)
hz:checkbox = 0
; 4 - Стартовое состояние (0=восстановить;1=не трогать;2=выключено;3=включено)
hz:startTurn = 0
; 6 - Подтяжка портов (0=Выход;1=Открытый коллектор;2=ШИМ;3=ЦАП)
hz:pull = 2
; 7 - Порт ключа
hz:pin = 5
; 8 - Частота ШИМ в герцах
hz:hz = 490
```

**Примечание:**
*   **Кол-во значений:** Все параметры имеют по 1 значению (одиночный канал).
*   **Единицы измерения:** `hz` — герцы.
*   **Режим ШИМ:** Параметр `pull` должен быть установлен в 2 (ШИМ) для корректной работы.

## Конструкторы

Класс не является шаблонным (фиксированный 1 канал).

1.  **По умолчанию**
    ```cpp
    MWOSHzSTM32();
    ```
    Создаёт модуль с параметрами по умолчанию. Частота инициализируется 490 Гц.

2.  **Из массива в PROGMEM**
    ```cpp
    MWOSHzSTM32(const uint8_t * pins);
    ```
    Создаёт модуль с портом из массива в памяти PROGMEM. Позволяет задать конфигурацию порта на этапе компиляции.

3.  **Одиночный пин**
    ```cpp
    MWOSHzSTM32(MWOS_PIN_INT pin);
    ```
    Создаёт модуль с одним ШИМ-каналом на указанном порту. Удобно для простых конфигураций.

## Основные методы

### Чтение значений

*   `virtual int64_t getValue(MWOSParam * param, int16_t arrayIndex)` *(переопределён)*
    *   Возвращает значение запрошенного параметра.
    *   Для параметра 8 (hz) возвращает частоту из локальной переменной.
    *   **Возвращает:** `int64_t` — значение параметра.

### Инициализация

*   `virtual void onEvent(MWOSModeEvent modeEvent)` *(переопределён)*
    *   **`EVENT_INIT` / `EVENT_CHANGE`:** Загрузка настроек из хранилища, инициализация аппаратного таймера STM32.
    *   **`EVENT_UPDATE`:** Вызывает обработку базового класса для управления ШИМ.

*   `virtual uint8_t turn(uint8_t mode, int16_t index, bool saveToStorage)` *(переопределён)*
    *   Изменяет значение ключа и настраивает ШИМ через библиотеку HardwareTimer.
    *   При включении ключа настраивает таймер с заданной частотой.
    *   При выключении ключа останавливает таймер.
    *   **Возвращает:** `uint8_t` — фактическое состояние порта с учётом инверсии.

```cpp
#include "MWOSHzSTM32.h"

```

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `turn` | RTC | Состояние ключа, восстанавливается после перезагрузки |
| `pin` | FOR_PIN | EEPROM или NO в зависимости от конфигурации |
| `pull` | EEPROM | Режим подтяжки портов, сохраняется |
| `hz` | EEPROM | Частота ШИМ, сохраняется |
| `invert`, `alwaysoff`, `checkbox` | EEPROM | Флаги режима работы, сохраняются |
| `startTurn` | EEPROM | Поведение при старте, сохраняется |

## Важные замечания

1.  **Только STM32:** Модуль работает только на платформе STM32. Использует библиотеку HardwareTimer, которая недоступна на других платформах. Код обёрнут в `#ifdef STM32_MCU_SERIES`.

2.  **Одиночный канал:** В отличие от MWOSHzESP32, этот модуль поддерживает только 1 канал ШИМ. Для нескольких каналов необходимо создавать несколько экземпляров модуля.

3.  **Автоматическая конфигурация таймера:** При инициализации модуль автоматически определяет канал таймера STM32 на основе выбранного пина через функции `pinmap_peripheral()` и `pinmap_function()`.

4.  **Порядок инициализации:** Все настройки по умолчанию должны задаваться **до** вызова `mwos.start()`. После запуска системы значения загружаются из хранилища, и повторная установка значений в коде будет перезаписывать сохранённые настройки при каждой перезагрузке.

5.  **Базовый класс MWOSKey:** Модуль наследуется от MWOSKey, что означает поддержку всех функций управления ключами (инверсия, чекбокс, принудительное отключение и т.д.).

6.  **Управление скважностью:** Скважность ШИМ управляется через параметр `_periodValue` базового класса MWOSKey.

7.  **Архитектура MWOS:** Все действия выполняются внутри модулей через обработчики событий. Пользовательский код в `loop()` должен содержать только вызов `mwos.update()`.

8.  **Сохранение состояния:** Состояние ключа сохраняется в RTC (энергонезависимое ОЗУ) при каждом изменении. Это обеспечивает восстановление состояния после перезагрузки без износа EEPROM.

9.  **Режим ШИМ:** Параметр `pull` должен быть установлен в 2 (ШИМ) для корректной работы модуля.

10. **Перенастройка частоты:** Изменение частоты ШИМ требует перенастройки таймера. Модуль автоматически выполняет перенастройку при изменении параметра `hz`.

## См. также

*   **MWOSKey**: Базовый модуль ключей для управления портами микроконтроллера. Предоставляет физическую коммутацию и базовые функции управления.
*   **MWOSKeyBase**: Базовый класс для всех модулей ключей. Реализует общую логику управления состояниями ключей, инверсию и режим чекбокса.
*   **MWOSKeyAnalog**: Модуль аналоговых ключей с поддержкой ШИМ и ЦАП. MWOSHzSTM32 использует аналогичный механизм для управления скважностью.
*   **MWOSHzESP32**: Аналогичный модуль для платформы ESP32 с использованием аппаратных таймеров LEDC.
*   **Библиотека HardwareTimer**: Встроенная библиотека STM32 для управления аппаратными таймерами.
