## Правила оформления

### Таблицы параметров
*   **ID** — номер параметра из `MWOS_PARAM()`
*   **Название** — имя параметра из `MWOS_PARAM()`
*   **Тип** — тип данных (`bits1`, `uint8`, `int16`, `uint32`, `float32`, `double64` и т.д.)
*   **Группа** — группа параметра (`control`, `option`, `pin`, `readonly`, `realtime`)
*   **Хранилище** — где хранится (`NO`, `EEPROM`, `RTC`, `FOR_PIN`, `NVS`)
*   **Кол-во** — `1` или `keysCount`/`sensorsCount`
*   **Описание** — краткое + детальное через **жирный текст**

### Конфигурация модуля
*   Использовать `;` для комментариев (не `//`)
*   Одна строка на комментарий, одна строка на параметр
*   Формат: `module:param = значение, значение, ...`
*   Для параметров с несколькими значениями — перечислять через запятую (5 значений по умолчанию)
*   Для параметров с 1 значением — одно значение
*   Для параметров с 2 значениями (min/max) — два значения через запятую

### Код
*   Все блоки кода в ```cpp
*   Примеры должны быть рабочими и минимальными
*   Комментарии в примерах на русском языке

### Стиль
*   Заголовки H1, H2, H3
*   Списки через `*`
*   Нумерованные списки для замечаний
*   **Жирный текст** для акцентов
*   `код` для имён параметров, методов, классов

### Особые требования
1.  **Извлекать имя модуля** из `MWOSModule::setName()` в конструкторе
2.  **Извлекать все параметры** из макросов `MWOS_PARAM()` включая родительские классы
3.  **Указывать тип модуля** из `moduleType=MODULE_XXX`
4.  **Показывать иерархию наследования** из объявления класса
5.  **Включать все параметры родительских классов** в раздел "Конфигурация модуля"
