# MWOSFS

**Имя модуля:** `fs`

**MWOSFS** — модуль работы с файловой системой контроллера. Предоставляет доступ к файлам и папкам, управление конфигурацией и выполнение системных команд.

## Описание

Класс предназначен для:
*   **Управление файловой системой:** Создание, удаление, переименование файлов и папок
*   **Конфигурация контроллера:** Загрузка и сохранение параметров в файлах конфигурации
*   **Системные команды:** Перезагрузка контроллера, вход в режим bootloader, форматирование ФС
*   **Мониторинг памяти:** Отслеживание общего, используемого и свободного места

**Наследование:** `MWOSFS` → `MWOSModule`

**Тип модуля:** `MODULE_FS`

**Зависимости:**
*   Библиотека `MWFS.h` (абстракция файловой системы)
*   Платформа ESP8266/ESP32 (для системных команд и работы с LittleFS/SPIFFS)

## Основные возможности

*   **Файловые операции:** Создание/удаление папок (md/rd), удаление файлов (del), переименование (rn)
*   **Листинг содержимого:** Получение списка файлов и папок (ls/dir)
*   **Чтение/запись файлов:** Чтение содержимого файлов (type), запись в файл (echo)
*   **Конфигурация:** Сохранение и загрузка параметров контроллера (cfg save/load)
*   **Системное управление:** Перезагрузка контроллера (reboot), вход в режим загрузчика (bootloader)
*   **Форматирование:** Полное форматирование файловой системы (format)
*   **Мониторинг:** Отслеживание размера, использованного и свободного места

## Параметры конфигурации

Класс использует параметры базовых классов и добавляет свои специфические параметры:

| ID | Название | Тип | Группа | Хранилище | Кол-во | Описание |
|:--:|:---|:---|:---|:---|:---:|:---|
| 0 | `cmd` | string | readonly | NO | 16400 | **Команды и ответы.** Текстовые команды файловой системы и результаты их выполнения |
| 1 | `size` | int64 | readonly | NO | 1 | **Общий размер.** Общий размер файловой системы в байтах |
| 2 | `used` | int64 | readonly | NO | 1 | **Использовано.** Использованное место файловой системы в байтах |
| 3 | `free` | int64 | readonly | NO | 1 | **Свободно.** Свободное место файловой системы в байтах |

**Примечание:**
*   **Кол-во:** Все параметры имеют 1 значение (глобальные для файловой системы).
*   **Параметр 0 (cmd):** Буфер команд имеет размер 16400 байт для поддержки длинных путей и ответов.
*   **Только чтение:** Все параметры доступны только для чтения, команды передаются через запись значения в параметр `cmd`.

## Конфигурация модуля

```
; модуль fs - работа с файловой системой контроллера
; 0 - Команды и ответы на них (буфер 16400 байт)
fs:cmd = 
; 1 - Общий размер файловой системы в байтах
fs:size = 1048576
; 2 - Использованное место файловой системы в байтах
fs:used = 524288
; 3 - Свободное место файловой системы в байтах
fs:free = 524288
```

**Примечание:**
*   **Кол-во значений:** Все параметры имеют по 1 значению.
*   **Единицы измерения:** `size`, `used`, `free` — байты.
*   **Команды:** Передаются через запись в параметр `cmd` в формате `команда;аргументы`.

## Конструкторы

Класс не является шаблонным (одиночный модуль файловой системы).

1.  **По умолчанию**
    ```cpp
    MWOSFS();
    ```
    Создаёт модуль с параметрами по умолчанию. Файловая система не задана.

2.  **С указанием файловой системы**
    ```cpp
    MWOSFS(MWFS * _fs);
    ```
    Создаёт модуль с указанием объекта файловой системы. Позволяет подключить конкретную ФС (LittleFS, SPIFFS и др.).

## Основные методы

### Чтение значений

*   `virtual int64_t getValue(MWOSParam * param, int16_t arrayIndex)` *(переопределён)*
    *   Возвращает значение запрошенного параметра.
    *   Для параметра `cmd` возвращает символы результата выполнения команды.
    *   Для параметров `size`, `used`, `free` возвращает значения от объекта файловой системы.
    *   **Возвращает:** `int64_t` — значение параметра.

### Управление файловой системой

*   `bool fsCommand(const String &command, const String &args)`
    *   Выполняет команду файловой системы.
    *   **Параметры:** `command` — команда (md, rd, del, rn, ls, dir, type, echo, cfg, reboot, bootloader, format), `args` — аргументы команды.
    *   **Возвращает:** `bool` — true при успешном выполнении.

*   `void initFS()`
    *   Инициализирует файловую систему.
    *   При отсутствии ФС выполняет форматирование.

*   `uint32_t configFileParams(bool saveToFile, String configFileName)`
    *   Сохраняет или загружает параметры контроллера из файла конфигурации.
    *   **Параметры:** `saveToFile` — true для сохранения, false для загрузки, `configFileName` — имя файла.
    *   **Возвращает:** `uint32_t` — количество обработанных параметров.

### Инициализация

*   `virtual void onEvent(MWOSModeEvent modeEvent)` *(переопределён)*
    *   **`EVENT_EMPTY_STORAGES`:** Загрузка параметров из файлов конфигурации (defaults.cfg, type.cfg, CID.cfg, user.cfg).
    *   **`EVENT_INIT`:** Инициализация файловой системы.
    *   **`EVENT_SET_VALUE`:** Обработка команд, записанных в параметр `cmd`.

## Пример использования

### Базовое подключение

```cpp
#include "fs/MWOSFS.h"
#include "fs/MWFS_LittleFS.h"

// Создаём объект файловой системы
MWFS_LittleFS littleFS;

// Создаём модуль работы с ФС
MWOSFS fsModule(&littleFS);

void setup() {
    mwos.start();
}

void loop() {
    mwos.update();
}
```

### Выполнение команд через терминал

```
; Создать папку
fs:cmd=md;/data

; Удалить папку
fs:cmd=rd;/data

; Удалить файл
fs:cmd=del;/data/config.txt

; Переименовать файл
fs:cmd=rn;/data/old.txt;/data/new.txt

; Листинг папки
fs:cmd=ls;/data

; Чтение файла
fs:cmd=type;/data/config.txt

; Запись в файл
fs:cmd=echo;/data/log.txt;текст для записи

; Сохранить конфигурацию
fs:cmd=cfg;save

; Загрузить конфигурацию
fs:cmd=cfg;load

; Перезагрузка контроллера
fs:cmd=reboot

; Форматирование файловой системы
fs:cmd=format;fs
```

### Программное выполнение команд

```cpp
#include "fs/MWOSFS.h"

// Получаем ссылку на модуль файловой системы
MWOSFS * fsModule = (MWOSFS *) mwos.FindChildByModuleType(MODULE_FS);

void setup() {
    mwos.start();
    
    // Создание папки программно
    if (fsModule != NULL) {
        fsModule->fsCommand("md", "/data");
    }
}

void loop() {
    mwos.update();
}
```

### Работа с конфигурацией

```cpp
#include "fs/MWOSFS.h"

// Получаем ссылку на модуль файловой системы
MWOSFS * fsModule = (MWOSFS *) mwos.FindChildByModuleType(MODULE_FS);

void setup() {
    mwos.start();
    
    // Сохранение параметров в файл конфигурации
    if (fsModule != NULL) {
        uint32_t count = fsModule->saveConfigFileParams();
        // count = количество сохранённых параметров
    }
}

void loop() {
    mwos.update();
}
```

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `cmd` | NO | Буфер команд, не хранится |
| `size` | NO | Вычисляется от файловой системы, не хранится |
| `used` | NO | Вычисляется от файловой системы, не хранится |
| `free` | NO | Вычисляется от файловой системы, не хранится |

## Важные замечания

1.  **Команды через параметр cmd:** Все команды файловой системы передаются через запись значения в параметр `cmd` (ID 0). Формат: `команда;аргументы`.

2.  **Файлы конфигурации:** Модуль автоматически загружает параметры из файлов при старте в порядке:
    *   `/defaults.cfg` — заводские настройки по умолчанию
    *   `/t{TYPE}.cfg` — настройки для типа контроллера
    *   `/{CID}.cfg` — индивидуальные настройки контроллера
    *   `/u{CID}.cfg` — пользовательские настройки (сохраняются перед сбросом)

3.  **Системные команды:** Команды `reboot` и `bootloader` работают только на платформах ESP8266/ESP32. На других платформах игнорируются.

4.  **Форматирование:** Команда `format;fs` удаляет все файлы и папки на файловой системе. Используйте с осторожностью.

5.  **Чтение файлов:** Команда `type` поддерживает чтение по частям с указанием позиции и размера: `type;/file.txt,0,1000` (читать 1000 байт с позиции 0).

6.  **Листинг файлов:** Команда `ls` запускает постраничную отправку списка файлов. Команда `dir` возвращает весь список сразу с возможностью фильтрации: `dir;/data*.txt`.

7.  **Мониторинг памяти:** Параметры `size`, `used`, `free` обновляются автоматически при выполнении команд. Для получения актуальных значений запросите их через сеть или терминал.

8.  **Буфер команд:** Размер буфера параметра `cmd` задан константой `FS_RESULT_MAX_SIZE` (MWOS_SEND_BUFFER_SIZE-512). При необходимости можно увеличить в конфигурации.

9.  **Автоматическая инициализация:** При старте модуль автоматически инициализирует файловую систему. Если ФС не найдена или пуста — выполняет форматирование.

10. **Событие EVENT_EMPTY_STORAGES:** По этому событию модуль загружает параметры из файлов конфигурации. Если хранилища не инициализированы, параметрам вызывается `EVENT_SET_VALUE` для сохранения в хранилище.

11. **Перенос старых конфигов:** При наличии CID контроллера модуль автоматически переименовывает старые файлы конфигурации (`/0.cfg` → `/{CID}.cfg`, `/u0.cfg` → `/u{CID}.cfg`).

12. **Логирование:** Все операции файловой системы логируются через `MW_LOG_MODULE`. Для отладки установите `MWOS_LOG_CMD > 0`.

## См. также

*   **MWOSModule**: Базовый класс для всех модулей контроллера. Реализует общую инфраструктуру модулей, параметры и события.
*   **MWFS**: Библиотека абстракции файловой системы. Предоставляет единый интерфейс для работы с разными ФС (LittleFS, SPIFFS, SD и др.).
*   **MWOSCmd**: Модуль простого соединения через поток. Может использоваться для отправки команд файловой системы через последовательный порт.
*   **MWOSTime**: Модуль учёта времени. Файлы конфигурации могут содержать параметры времени для синхронизации.
