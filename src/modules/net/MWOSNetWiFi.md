# MWOSNetWiFi

**Имя модуля:** `net`

**MWOSNetWiFi** — модуль сетевого подключения контроллера к серверу MWOS через WiFi. Поддерживает автоматическое подключение к WiFi сети, режим точки доступа для конфигурации, статическую настройку сети и автоматическое переподключение при разрыве соединения.

## Описание

Класс предназначен для:
*   **WiFi подключение:** Организация соединения с WiFi сетью на ESP8266/ESP32.
*   **Автоматическое переподключение:** Многошаговый алгоритм подключения с режимом точки доступа при неудаче.
*   **Режим настройки:** Запуск точки доступа для первоначальной конфигурации WiFi через веб-интерфейс.
*   **Подключение к серверу:** Соединение с сервером MWOS через TCP-сокет после подключения к WiFi.

**Наследование:** `MWOSNetWiFi` → `MWOSNetIP` → `MWOSNetDevice` → `MWOSNetModule` → `MWOSModule`, `MWOSNetReceiver`

**Тип модуля:** `MODULE_NET`

**Зависимости:**
*   Библиотека `WiFi` (ESP8266/ESP32 Core)
*   Библиотека `MWAsyncClient` (асинхронный клиент)
*   Библиотека `MWSetupWiFi` (веб-интерфейс настройки)
*   Платформа: **ESP8266, ESP32** (требуется поддержка WiFi)

## Основные возможности

*   **Автоматическое подключение:** Подключение к последней сохранённой сети или к сети по умолчанию.
*   **Режим точки доступа:** Автоматический запуск AP при неудачном подключении к WiFi (через 120 сек).
*   **Веб-интерфейс настройки:** Конфигурация WiFi через веб-страницу в режиме AP.
*   **Статический IP:** Поддержка DHCP и статической настройки сети (IP, шлюз, маска, DNS).
*   **Многоадресное подключение:** Последовательная попытка подключения к нескольким серверам.
*   **SSL поддержка:** Опциональное SSL-подключение к серверу (требуется настройка сертификатов).
*   **Хранение credentials:** SSID и пароль сохраняются в NVS (ESP32) или EEPROM.
*   **Таймауты подключения:** Настраиваемые интервалы между попытками подключения.

## Параметры конфигурации

Класс использует параметры базовых классов и добавляет свои специфические параметры:

| ID | Название | Тип | Группа | Хранилище | Кол-во | Описание |
|:--:|:---|:---|:---|:---|:---:|:---|
| 0 | `controllerId` | `uint32` | `readonly+option` | `EEPROM` | 1 | **Код контроллера.** CID присвоенный сервером при первой авторизации. 0 — не выдан |
| 1 | `reconnectCount` | `uint8` | `option` | `RTC` | 1 | **Количество повторных подключений.** Неудачных попыток перед переключением на резервный сервер |
| 2 | `dhcp` | `bits2` | `option` | `EEPROM` | 1 | **Тип подключения.** 0=DHCP, 1=STATIC IP4 |
| 3 | `staticIP` | `uint8` | `option` | `EEPROM` | 4 | **Статический IP-адрес.** 4 байта IP-адреса контроллера |
| 4 | `gateway` | `uint8` | `option` | `EEPROM` | 4 | **Шлюз по умолчанию.** 4 байта IP-адреса шлюза |
| 5 | `subnet` | `uint8` | `option` | `EEPROM` | 4 | **Маска подсети.** 4 байта маски сети |
| 6 | `dns` | `uint8` | `option` | `EEPROM` | 4 | **DNS-сервер.** 4 байта IP-адреса DNS |
| 8 | `netInit` | `bits2` | `option` | `EEPROM` | 1 | **Инициализация сети.** 0=Нет, 1=Выключено, 2=Включено, 3=Точка доступа |
| 9 | `netNow` | `bits2` | `option` | `NO` | 1 | **Текущее состояние.** 0=Выключено, 1=Включено, 2=Точка доступа (только чтение) |
| 10 | `ssid` | `string` | `option` | `NVS` | 50 | **SSID сети WiFi.** Имя WiFi сети для подключения |
| 11 | `pass` | `string` | `option+secret` | `NVS` | 16 | **Пароль сети WiFi.** Пароль от WiFi сети |
| 12 | `name` | `string` | `option` | `NVS` | 50 | **Сетевое имя контроллера.** Имя контроллера в сети |

**Примечание:**
*   **Кол-во:** Параметры `staticIP`, `gateway`, `subnet`, `dns` имеют по 4 значения (по одному байту на октет IP-адреса). Остальные — по 1 значению, кроме `ssid`, `pass`, `name` (строки).
*   **Хранилище:** WiFi credentials (`ssid`, `pass`, `name`) сохраняются в NVS на ESP32.
*   **По умолчанию:** Если `reconnectCount=0` или >200, устанавливается значение 3.

## Конфигурация модуля

```cpp
; модуль net - сетевое подключение к серверу MWOS через WiFi
; ID - Описание параметра
; 0 - Код контроллера (CID, присваивается сервером)
net:controllerId = 0
; 1 - Количество повторных подключений (неудачных попыток)
net:reconnectCount = 3
; 2 - Тип подключения (0=DHCP, 1=STATIC IP4)
net:dhcp = 0
; 3 - Статический IP-адрес (4 байта)
net:staticIP = 192, 168, 0, 54
; 4 - Шлюз по умолчанию (4 байта)
net:gateway = 192, 168, 0, 1
; 5 - Маска подсети (4 байта)
net:subnet = 255, 255, 255, 0
; 6 - DNS-сервер (4 байта)
net:dns = 192, 168, 0, 1
; 8 - Инициализация сети (0=Нет, 1=Выключено, 2=Включено, 3=Точка доступа)
net:netInit = 2
; 9 - Текущее состояние сети (0=Выключено, 1=Включено, 2=Точка доступа)
net:netNow = 0
; 10 - SSID сети WiFi (до 50 символов)
net:ssid = "MyWiFiNetwork"
; 11 - Пароль сети WiFi (до 16 символов)
net:pass = "password123"
; 12 - Сетевое имя контроллера (до 50 символов)
net:name = "MWOS_Controller"
```

**Примечание:**
*   **Кол-во значений:** Параметры IP-адресов имеют по 4 значения (октеты). Строковые параметры — переменная длина.
*   **Порт сервера:** По умолчанию используется порт 8082 (можно изменить через `MWOS_SERVER_PORT`).
*   **Хост сервера:** По умолчанию `voscom.online` (можно изменить через `MWOS_SERVER_HOST`).
*   **Строковые параметры:** Значения строковых параметров заключаются в двойные кавычки `"`.

## Основные методы

### Чтение значений

*   `virtual int64_t getValue(MWOSParam * param, int16_t arrayIndex)` *(переопределён)*
    *   Возвращает значение параметра модуля.
    *   **Возвращает:** `int64_t` — текущее значение параметра (ssid, pass, name и др.).

*   `static MWOSNetModule * getNetModule()` *(наследован)*
    *   Находит активный сетевой модуль в системе.
    *   **Возвращает:** `MWOSNetModule*` — указатель на сетевой модуль или NULL.

*   `uint32_t getCID()` *(наследован)*
    *   Получает глобальный CID контроллера из EEPROM или NVS.
    *   **Возвращает:** `uint32_t` — идентификатор контроллера.

### Инициализация и обработка

*   `virtual void onEvent(MWOSModeEvent modeEvent)` *(переопределён)*
    *   **`EVENT_INIT` / `EVENT_CHANGE`:** Загрузка параметров, инициализация WiFi согласно `netInit`.
    *   **`EVENT_UPDATE`:** Мониторинг статуса WiFi, обработка веб-сервера в режиме AP, проверка таймаутов.

*   `virtual void onConnect()` *(переопределён)*
    *   Вызывается при успешном подключении к серверу.
    *   Сбрасывает счётчик шагов переподключения.

*   `virtual void onDisconnect(MWOSDisconnectCode disconnectCode)` *(переопределён)*
    *   Вызывается при разрыве соединения.
    *   Увеличивает счётчик ошибок, при превышении лимита — запуск режима AP.

### Управление подключением

*   `virtual void connectToServer()` *(переопределён)*
    *   Начинает подключение к серверу MWOS.
    *   Перебирает список серверов при неудачных попытках.

*   `virtual bool isConnectedServer(bool firstTime)` *(переопределён)*
    *   Проверяет наличие подключения к серверу.
    *   **Возвращает:** `bool` — true если сервер подключён.

*   `virtual void startNet()` *(переопределён)*
    *   Аппаратно включает сеть через `startWiFi()`.
    *   Пытается подключиться к сохранённой или заданной WiFi сети.

*   `virtual void connectNet()` *(переопределён)*
    *   Начинает подключение к WiFi сети.
    *   Логирует статус подключения и IP-адрес.

*   `virtual void closeNet()` *(переопределён)*
    *   Закрывает подключение к сети.
    *   Останавливает точку доступа и WiFi соединение.

*   `void startWiFiAp()`
    *   Запускает точку доступа для настройки WiFi.
    *   IP точки доступа: 192.168.4.1
    *   SSID: `voscom_{CID}` (или заданное имя)

*   `void stopWiFiAp()`
    *   Останавливает точку доступа.
    *   Отключает веб-сервер настроек.

*   `void saveWiFi()`
    *   Сохраняет новые WiFi credentials от веб-сервера.
    *   Перезапускает подключение с новыми настройками.

## Важные замечания

1.  **Алгоритм подключения WiFi:**
    *   Сначала попытка подключения к последней сохранённой сети
    *   Если не удалось — подключение к сети из параметров `ssid`/`pass`
    *   Если не удалось через 120 сек (`MW_NoConnectWiFi_Sec`) — запуск точки доступа
    *   Точка доступа работает 30 минут (`MW_ServerWiFi_Minutes`)
    *   Если за это время не заданы новые настройки — выключение AP и повтор попытки
2.  **Порт сервера:** По умолчанию используется порт 8082. Можно изменить через макрос `MWOS_SERVER_PORT`.
3.  **Хост сервера:** По умолчанию `voscom.online`. Можно изменить через макрос `MWOS_SERVER_HOST`.
4.  **Резервные серверы:** При неудаче перебираются: `voscom.online`, `voscom.ru`, `voscom.ddns.net`.
5.  **Статический IP:** При `dhcp=1` используются настройки из параметров `staticIP`, `gateway`, `subnet`, `dns`.
6.  **Таймауты:** Между попытками подключения выдерживается интервал 5 секунд (50 дсек).
7.  **ESP32 NVS:** На ESP32 WiFi credentials и CID сохраняются в NVS для сохранения при очистке EEPROM.
8.  **Кнопка сброса:** При наличии `MWOS_RESET_TO_FACTORY_SETTINGS_PIN` и нажатии кнопки при старте — EEPROM очищается, включается режим AP.
9.  **Режим AP:** Точка доступа создаётся с SSID `voscom_{CID}` и IP 192.168.4.1.
10. **Веб-сервер:** В режиме AP запускается веб-сервер для настройки WiFi через браузер.
11. **SSL подключение:** Опциональная поддержка SSL через `WiFiClientSecure` (требуется настройка сертификатов).
12. **Логирование:** Включается через флаг `MWOS_LOG_NET`. Уровни: >0 (события), >1 (шаги подключения), >2 (ошибки таймаутов), >3 (детали WiFi).
13. **Макросы конфигурации:**
    *   `MWOW_AP_AFTER_NORECONNECTS` — через сколько неудач включать AP (по умолчанию 3)
    *   `MW_ServerWiFi_Minutes` — сколько минут работает AP (по умолчанию 30)
    *   `MW_NoConnectWiFi_Sec` — через сколько секунд включать AP (по умолчанию 120)
    *   `MWOS_TO_WIFI_NAME` — WiFi сеть по умолчанию
    *   `MWOS_TO_WIFI_PASS` — Пароль WiFi по умолчанию
14. **Отключение резервных URL:** Макрос `MWOS_NOT_RECONNECT_URLS` отключает перебор резервных серверов.
15. **Быстрое переподключение:** Макрос `MWOS_WIFI_NOFAST_RECONNECT` отключает быстрое переподключение при разрыве.
