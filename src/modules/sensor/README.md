## Иерархия классов датчиков

```
MWOSModule
└── MWOSSensorBase<sensorsCount> (базовый класс для всех датчиков)
    ├── MWOSSensorBin<sensorsCount> (бинарные датчики)
    └── MWOSSensorAnalog<sensorsCount> (аналоговые датчики)
        ├── MWOSSensorADC<sensorsCount> (АЦП микроконтроллера)
        ├── MWOSSensorADS1115<sensorsCount> (внешний АЦП ADS1115)
        ├── MWOSSensorDHT (датчики температуры и влажности DHT)
        ├── MWOSSensorDS<sensorsCount> (цифровые термометры DS18x20)
        ├── MWOSSensorFilter<sensorsCount> (фильтрация и коррекция показаний)
        └── MWOSSensorSerial<sensorsCount> (потоковые датчики через UART)
            ├── MWOSSensorLuminOx<> (потоковый датчик O2 LuminOx)
            ├── MWOSSensorCozIR<> (потоковый датчик СO2 CozIR)
            └── MWOSSensorSprintIR<> (потоковый датчик CO2 SprintIR)
```

## Расширенная фильтрация через MWOSSensorFilter

Все модули аналоговых датчиков поддерживают шаблонный параметр для указания типа базового класса. По умолчанию используется `MWOSSensorAnalog`, но можно указать `MWOSSensorFilter` для получения расширенных возможностей фильтрации и коррекции показаний.

### Пример объявления с фильтрацией

```cpp
// Обычный АЦП без расширенной фильтрации
MWOSSensorADC<5> sensorADC;

// АЦП с расширенной фильтрацией Калмана
MWOSSensorADC<5, MWOSSensorFilter<5>> sensorADC;

// Датчик DS18x20 с фильтрацией
MWOSSensorDS<8, MWOSSensorFilter<8>> sensorDS;

// Внешний АЦП ADS1115 с фильтрацией
MWOSSensorADS1115<4, MWOSSensorFilter<4>> sensorADS;
```

### Преимущества использования MWOSSensorFilter

При использовании `MWOSSensorFilter` вместо `MWOSSensorAnalog` модуль получает дополнительные параметры и возможности:

- Коррекция показаний коэффициентами умножения и вычитания
- Сглаживание шумов измерений фильтром Калмана
- Предсказание выхода на плато расширенным фильтром Калмана (EKF)
- Дополнительные настройки фильтров для каждого канала
- Дробные значения результатов измерений

### Ограничения

Использование фильтрации увеличивает потребление памяти и вычислительные затраты. Не рекомендуется для систем с ограниченной памятью или для датчиков, не требующих высокой точности измерений.

## Модуль фильтрации MWOSSensorFilter

Класс `MWOSSensorFilter` наследуется от `MWOSSensorAnalog` и добавляет поддержку коррекции показаний, фильтрации Калмана и расширенного фильтра Калмана с предсказанием плато.

### Назначение модуля

Модуль предназначен для расширения функциональности аналоговых датчиков с возможностью коррекции показаний коэффициентами, сглаживания шумов измерений и предсказания выхода на плато. Поддерживает несколько алгоритмов фильтрации с настраиваемыми параметрами.

### Параметры модуля

| Параметр | Тип | Группа | Хранилище | Описание |
|----------|-----|--------|-----------|----------|
| result | double64 | OPTIONS | NO | Результат с учётом коэффициентов и фильтров |
| coef | float32 | OPTIONS | EEPROM | Коэффициент умножения |
| subCoef | float32 | OPTIONS | EEPROM | Коэффициент вычитания |
| predictedEKF | double64 | OPTIONS | NO | Предсказание EKF |
| filter | bits3 | OPTIONS | EEPROM | Режим фильтра |
| kalmanQ | double64 | OPTIONS | EEPROM | Шум процесса |
| kalmanR | double64 | OPTIONS | EEPROM | Шум измерений |
| kalmanTau | float32 | OPTIONS | EEPROM | Время выхода на плато [сек] |
| filterFinish | bits1 | OPTIONS+READONLY | NO | Окончание фильтрации |
| filterOptions | float32 | OPTIONS | NVS | Дополнительные настройки фильтров |

### Режимы фильтрации

| Режим | Значение | Описание |
|-------|----------|----------|
| EFM_None | 0 | Фильтр отключен |
| EFM_Filter1 | 1 | Фильтр 1 |
| EFM_Filter2 | 2 | Фильтр 2 |
| EFM_Filter3 | 3 | Фильтр 3 |
| EFM_Filter4 | 4 | Фильтр 4 |

### Внутренние переменные

| Переменная | Тип | Описание |
|------------|-----|----------|
| _coef | float[] | Коэффициенты умножения для каждого датчика |
| _subCoef | float[] | Коэффициенты вычитания для каждого датчика |
| _filter | ExtendedFilterMode[] | Режимы фильтрации для каждого датчика |
| sensorFilter | SensorFilter*[] | Указатели на объекты фильтров для каждого датчика |
| _filterOptions | float[] | Дополнительные настройки фильтров |

### Методы модуля

| Метод | Описание |
|-------|----------|
| getFloatValue(index) | Получить значение с учётом коэффициентов |
| getFilteredValue(index) | Получить значение с применённым фильтром |
| readBoolValue(index) | Чтение бинарного значения с обновлением фильтра |

### Настройка параметров фильтра Калмана

**kalmanQ** — шум процесса. Чем выше значение, тем больше доверия новым данным измерений. Рекомендуется устанавливать в диапазоне 0.001 до 0.1.

**kalmanR** — шум измерений. Чем выше значение, тем больше сглаживание показаний. Рекомендуется устанавливать в диапазоне 0.1 до 10.

**kalmanTau** — предполагаемое время выхода на плато в секундах. Используется расширенным фильтром Калмана для предсказания стабилизации показаний.

### Пример настройки фильтрации

```cpp
// Объявление датчика с фильтрацией
MWOSSensorADC<4, MWOSSensorFilter<4>> sensorADC;

// Настройка коэффициентов коррекции
sensorADC._coef[0] = 1.5;
sensorADC._subCoef[0] = 10.0;

// Включение фильтра Калмана
sensorADC._filter[0] = EFM_Filter2;

// Настройка параметров фильтра
sensorADC.sensorFilter[0]->Q = 0.01;
sensorADC.sensorFilter[0]->R = 0.1;
sensorADC.sensorFilter[0]->tau = 5.0;
```

## Базовый класс MWOSSensorBase

Класс `MWOSSensorBase` является фундаментальным базовым классом для всех модулей датчиков в системе. Он реализует общую логику работы с бинарными состояниями датчиков, фильтрацию переключений, управление подтяжкой портов и базовую инфраструктуру для наследования.

### Назначение класса

Этот класс предназначен только для наследования и не используется самостоятельно. Он предоставляет универсальный механизм работы с множественными датчиками одного типа, где количество датчиков задаётся шаблоном при объявлении класса.

### Параметры модуля

| Параметр | Тип | Группа | Хранилище | Описание |
|----------|-----|--------|-----------|----------|
| valueBin | bits1 | READONLY | NO | Бинарные показания датчиков |
| valueCount | uint32 | OPTIONS | NO | Количество измерений с момента включения |
| binFilter | uint8 | OPTIONS | EEPROM | Фильтр переключения [сек/10] |
| getTimeout | uint16 | OPTIONS | EEPROM | Минимальное время между опросами [сек/10] |
| invert | bits1 | OPTIONS | EEPROM | Инвертировать значение |
| alwaysOff | bits1 | OPTIONS | EEPROM | Всегда отключено |
| pull | bits3 | OPTIONS | EEPROM | Подтяжка портов |
| pullOff | bits1 | OPTIONS | EEPROM | Выключать подтяжку между опросами |

### Внутренняя структура состояния датчика

Для каждого датчика модуль хранит структуру битовых флагов в едином байте:

| Бит | Описание |
|-----|----------|
| 0 | Текущее бинарное значение датчика |
| 1 | Флаг изменения бинарного значения |
| 2 | Активное состояние фильтра переключений |
| 3 | Включённый запрос к датчику |
| 4 | Изменение аналогового значения |
| 5-7 | Резерв |

### Методы управления состоянием

| Метод | Описание |
|-------|----------|
| pause() | Переводит все датчики модуля в режим паузы |
| resume() | Снимает датчики с паузы и восстанавливает нормальный режим |
| isPaused() | Возвращает текущее состояние паузы модуля |
| getValueBool(index) | Запрашивает актуальное бинарное значение указанного датчика |
| initSensor(index, pullOn) | Инициализация порта конкретного датчика |
| readBoolValue(index) | Чтение значения датчика |
| updateSensor() | Управление циклом опроса всех датчиков модуля |
| readSensor(index) | Чтение конкретного датчика по индексу |

### Алгоритм опроса датчиков

Метод `updateSensor()` вызывается в событии EVENT_UPDATE и управляет циклом опроса всех датчиков модуля. Если модуль находится в режиме паузы, опрос пропускается. Если включён режим отключения подтяжки между опросами, метод сначала включает подтяжку на всех портах, ожидает время стабилизации, выполняет опрос, затем отключает подтяжку.

## Класс MWOSSensorAnalog

Класс `MWOSSensorAnalog` наследуется от `MWOSSensorBase` и добавляет поддержку аналоговых датчиков с пороговыми значениями, фильтрацией и вычислением процентов.

### Назначение класса

Модуль предназначен только для наследования и не используется самостоятельно. Он предоставляет инфраструктуру для работы с аналоговыми датчиками, включая фильтрацию изменений, пороговые значения для бинарной интерпретации и вычисление процентов от диапазона.

### Параметры модуля

| Параметр | Тип | Группа | Хранилище | Описание |
|----------|-----|--------|-----------|----------|
| value | int32 | READONLY | NO | Последние аналоговые показания |
| fromValue | int32 | OPTIONS | EEPROM | Нижний порог для valueBin |
| toValue | int32 | OPTIONS | EEPROM | Верхний порог для valueBin |
| analogSend | bits1 | OPTIONS | EEPROM | Отправлять аналоговые значения |
| sensorDigital | bits1 | OPTIONS | EEPROM | Режим цифрового датчика |
| analogFilterMin | uint16 | OPTIONS | EEPROM | Минимальное изменение (фильтр) |
| analogFilterMax | uint16 | OPTIONS | EEPROM | Максимальное изменение (сбой) |
| error | int8 | READONLY | NO | Код ошибки |
| average | bits1 | OPTIONS | EEPROM | Среднее арифметическое |

### Методы работы с аналоговыми значениями

| Метод | Описание |
|-------|----------|
| getValueAnalog(index) | Получить аналоговое значение |
| getValuePercent(index) | Процент между fromValue и toValue |
| readAnalogValue(index) | Чтение АЦП |
| readDigitalValue(index) | Чтение цифрового значения |
| setError(code) | Установить код ошибки |

## Системные события модулей

Каждый модуль датчика обрабатывает системные события через метод `onEvent()`. События вызываются системой автоматически в соответствующие моменты работы контроллера.

| Событие | Описание | Когда вызывается |
|---------|----------|------------------|
| EVENT_UPDATE | Тиковая обработка | Каждый такт операционной системы |
| EVENT_INIT | Первая инициализация | При старте контроллера |
| EVENT_CHANGE | Изменение параметров | После сохранения каждого изменённого параметра |
| EVENT_POWER_OFF | Перед отключением питания | Перед выключением контроллера |
| EVENT_TIME_CHANGE | Изменение времени | После синхронизации времени |
| EVENT_CONNECT | Подключение к серверу | При установке соединения |
| EVENT_DISCONNECT | Отключение от сервера | При разрыве соединения |
| EVENT_GET_VALUE | Запрос значения | При запросе значения параметра |
| EVENT_SET_VALUE | Установка значения | При получении команды на изменение |

## Макросы объявления параметров

Система MWOS предоставляет макросы для объявления параметров модулей.

### MWOS_PARAM

Объявляет параметр с идентификатором, именем, типом значения, группой, хранилищем и длиной массива.

```cpp
MWOS_PARAM(id, name, valueType, paramGroup, storage, arrayLength)
```

### MWOS_PARAM_FF

Объявляет параметр с форматом перечисления. Принимает дополнительный параметр формата со значениями через точку с запятой.

```cpp
MWOS_PARAM_FF(id, name, valueType, paramGroup, storage, arrayLength, "format")
```

## Шаблоны модулей датчиков

Все модули датчиков реализованы как шаблонные классы с параметром `sensorsCount`. Этот параметр определяет количество датчиков, поддерживаемых экземпляром модуля.

```cpp
template<MWOS_PARAM_INDEX_UINT sensorsCount>
class MWOSSensorBin : public MWOSSensorBase<sensorsCount> {
    // sensorsCount определяет размер внутренних массивов
};
```

## Наследование и переопределение методов

Модули датчиков используют механизм наследования для расширения функциональности. Базовый класс `MWOSSensorBase` реализует общую логику работы с датчиками. Класс `MWOSSensorAnalog` добавляет поддержку аналоговых значений. Конкретные модули датчиков наследуются от `MWOSSensorAnalog` и переопределяют виртуальные методы для реализации специфичной логики.

### Виртуальные методы для переопределения

| Метод | Класс | Описание |
|-------|-------|----------|
| initSensor(index, pullOn) | MWOSSensorBase | Инициализация порта датчика |
| readBoolValue(index) | MWOSSensorBase | Чтение бинарного значения |
| readAnalogValue(index) | MWOSSensorAnalog | Чтение аналогового значения |
| readDigitalValue(index) | MWOSSensorAnalog | Чтение цифрового значения |
| onEvent(modeEvent) | MWOSModule | Обработка системных событий |
| getValue(param, index) | MWOSModule | Запрос значения параметра |
