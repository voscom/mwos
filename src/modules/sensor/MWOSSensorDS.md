# MWOSSensorDS

**Имя модуля:** `ds`

**MWOSSensorDS** — модуль для подключения цифровых термометров DS18x20 на шине One-Wire. Поддерживает множественные датчики на одном порту с автоматическим поиском и сохранением адресов.

## Описание

Класс предназначен для:
*   **Подключение датчиков DS18x20:** Работа с цифровыми термометрами DS18B20, DS18S20, DS1822 на шине One-Wire
*   **Множественные датчики:** Поддержка до `sensorsCount` датчиков на одном порту микроконтроллера
*   **Автоматический поиск:** Обнаружение всех датчиков на шине с сохранением их 64-битных адресов
*   **Управление питанием:** Опциональный порт для включения/выключения питания шины датчиков

**Наследование:** `MWOSSensorDS<sensorsCount, MWOSSensorAnalogType>` → `MWOSSensorAnalogType` → `MWOSSensorBase<sensorsCount>` → `MWOSModule`

**Тип модуля:** `MODULE_SENSOR`

**Зависимости:**
*   Библиотека `OneWireNg` (https://github.com/crankyoldgit/OneWireNg)
*   Библиотека `MWOSSensorAnalog.h` (базовый класс)
*   Платформа с поддержкой One-Wire (ESP32/ESP8266/STM32/Arduino)

## Основные возможности

*   **Множественные датчики на одной шине:** Поддержка до `sensorsCount` независимых датчиков температуры на одном порту
*   **Автоматическое обнаружение:** Поиск всех датчиков на шине One-Wire при инициализации
*   **Сохранение адресов:** 64-битные адреса датчиков сохраняются в EEPROM для быстрого доступа
*   **Управление питанием:** Опциональный порт для включения/выключения питания шины датчиков
*   **Гибкое наследование:** Возможность указания `MWOSSensorFilter` вместо `MWOSSensorAnalog` для расширенной фильтрации
*   **Поддержка разных моделей:** DS18B20, DS18S20, DS1822 с автоматическим определением разрешения
*   **Высокая точность:** Температура в сотых градуса Цельсия (×100)

## Параметры конфигурации

Класс использует параметры базовых классов и добавляет свои специфические параметры:

| ID | Название | Тип | Группа | Хранилище | Кол-во | Описание |
|:--:|:---|:---|:---|:---|:---:|:---|
| 0 | `valueBin` | bits1 | readonly | NO | sensorsCount | **Бинарные показания.** Пороговое значение на основе fromValue/toValue |
| 1 | `pin` | int16 | pin | FOR_PIN | 1 | **Порт шины One-Wire.** Номер порта для подключения датчиков DS. -1 = не подключён |
| 2 | `valueCount` | uint32 | option | NO | 1 | **Количество измерений.** Счётчик запросов показаний с момента включения |
| 3 | `binFilter` | uint8 | option | EEPROM | 1 | **Фильтр переключения.** Время фильтрации бинарных переключений [сек/10] |
| 4 | `getTimeout` | uint16 | option | EEPROM | 1 | **Таймаут опроса.** Минимальное время между опросами датчиков [сек/10] |
| 5 | `invert` | bits1 | option | EEPROM | 1 | **Инверсия.** Инвертировать бинарное значение датчика |
| 6 | `alwaysOff` | bits1 | option | EEPROM | 1 | **Всегда отключено.** Принудительное отключение опроса датчиков |
| 7 | `pull` | bits3 | option | EEPROM | 1 | **Подтяжка портов.** Режим подтяжки (0=Вход;1=На землю;2=На плюс;3=АЦП) |
| 8 | `pullOff` | bits1 | option | EEPROM | 1 | **Отключение подтяжки.** Выключать подтяжку между опросами |
| 9 | `value` | int32 | readonly | NO | sensorsCount | **Аналоговые показания.** Температура в сотых градуса (°C × 100) |
| 10 | `fromValue` | int32 | option | EEPROM | 1 | **Нижний порог.** Значение для интерпретации бинарной единицы |
| 11 | `toValue` | int32 | option | EEPROM | 1 | **Верхний порог.** Значение для интерпретации бинарной единицы |
| 12 | `analogSend` | bits1 | option | EEPROM | 1 | **Отправка аналоговых.** 0=только при изменении бинарного, 1=при изменении аналогового |
| 14 | `sensorDigital` | bits1 | option | EEPROM | 1 | **Цифровой режим.** При 1 работает как бинарный датчик |
| 15 | `analogFilterMin` | uint16 | option | EEPROM | 1 | **Минимальное изменение.** Игнорировать изменения меньше этого значения |
| 16 | `analogFilterMax` | uint16 | option | EEPROM | 1 | **Максимальное изменение.** Считать сбоем изменения больше этого значения |
| 17 | `error` | int8 | readonly | NO | 1 | **Код ошибки.** 0=нет ошибки, 1=ошибка reset, 2=ошибка CRC, 3=нет места для адресов |
| 18 | `power` | int16 | pin | FOR_PIN | 1 | **Порт питания датчиков.** Номер порта управления питанием шины. -1 = не используется |
| 19 | `address` | int64 | readonly+option | EEPROM | sensorsCount | **Адреса датчиков.** 64-битные адреса DS для каждого канала. 0 = не назначен |

**Примечание:**
*   **Кол-во:** Параметры с `sensorsCount` имеют индивидуальное значение для каждого датчика. Остальные параметры — по 1 значению.
*   **Параметр 19 (address):** Нулевой адрес указывает на не назначенный канал. При записи 0 — сброс адреса датчика. При записи -1 — сброс всех адресов.
*   **Единицы измерения:** Температура возвращается в сотых градуса Цельсия (°C × 100). Для получения реальных значений разделите на 100.0.

## Конфигурация модуля

```
; модуль ds - цифровые термометры DS18x20 на шине One-Wire
; 0 - Бинарные показания (sensorsCount значений)
ds:valueBin = 0, 0, 0, 0, 0, 0, 0, 0
; 1 - Порт шины One-Wire
ds:pin = 4
; 2 - Количество измерений с момента включения
ds:valueCount = 0
; 3 - Фильтр переключения бинарного значения [сек/10]
ds:binFilter = 0
; 4 - Минимальное время между опросами показаний датчика [сек/10]
ds:getTimeout = 10
; 5 - Инвертировать значение
ds:invert = 0
; 6 - Всегда отключено
ds:alwaysOff = 0
; 7 - Подтяжка портов (0=Вход;1=На землю;2=На плюс;3=АЦП)
ds:pull = 2
; 8 - Выключать подтяжку порта датчика между опросами
ds:pullOff = 0
; 9 - Последние показания температуры (sensorsCount значений, °C × 100)
ds:value = 2500, 2600, 2550, 0, 0, 0, 0, 0
; 10 - Нижний порог для valueBin (°C × 100)
ds:fromValue = 0
; 11 - Верхний порог для valueBin (°C × 100)
ds:toValue = 5000
; 12 - Отправлять аналоговые значения (0=только бинарное, 1=аналоговое)
ds:analogSend = 1
; 14 - Режим цифрового датчика (0=аналоговый, 1=цифровой)
ds:sensorDigital = 0
; 15 - Минимальное изменение (фильтр)
ds:analogFilterMin = 0
; 16 - Максимальное изменение (сбой)
ds:analogFilterMax = 0
; 17 - Код ошибки
ds:error = 0
; 18 - Порт управления питанием датчиков
ds:power = -1
; 19 - Адреса DS для каждого обнаруженного датчика (sensorsCount значений)
ds:address = 0, 0, 0, 0, 0, 0, 0, 0
```

**Примечание:**
*   **Кол-во значений:** Параметры `valueBin`, `value`, `address` имеют `sensorsCount` значений (по одному на каждый датчик). Остальные параметры — по 1 значению.
*   **Единицы измерения:** `getTimeout`, `binFilter` — десятые доли секунды. Температура в сотых градуса (°C × 100).
*   **Адреса датчиков:** 64-битные значения. 0 = датчик не назначен. Заполняются автоматически при поиске.
*   **Таймаут опроса:** Рекомендуется устанавливать не менее 10 (1 секунда) для датчиков DS, так как конвертация температуры занимает до 750мс.

## Конструкторы

Класс является шаблонным: `MWOSSensorDS<sensorsCount, MWOSSensorAnalogType>`.

1.  **По умолчанию**
    ```cpp
    MWOSSensorDS();
    ```
    Создаёт модуль с параметрами по умолчанию. Порт инициализируется значением -1 (не подключён).

2.  **С указанием порта и питания**
    ```cpp
    MWOSSensorDS(MWOS_PIN_INT pin, MWOS_PIN_INT pinPower = -1);
    ```
    Создаёт модуль с датчиками на указанном порту One-Wire. Опционально задаёт порт управления питанием.

3.  **С расширенной фильтрацией**
    ```cpp
    MWOSSensorDS<8, MWOSSensorFilter<8>> sensorDS;
    ```
    Создаёт модуль с поддержкой фильтрации Калмана и коррекции показаний коэффициентами.

## Основные методы

### Чтение значений

*   `virtual int32_t readAnalogValue(int16_t arrayIndex)` *(переопределён)*
    *   Выполняет чтение температуры с указанного датчика DS.
    *   Запускает конвертацию при необходимости, читает результат после завершения.
    *   **Возвращает:** `int32_t` — температура в сотых градуса Цельсия (°C × 100).

*   `int64_t getValue(MWOSParam * param, int16_t arrayIndex)` *(переопределён)*
    *   Возвращает значения параметров pin, power, address.
    *   **Возвращает:** `int64_t` — значение запрошенного параметра.

### Инициализация

*   `virtual void onEvent(MWOSModeEvent modeEvent)` *(переопределён)*
    *   **`EVENT_INIT` / `EVENT_CHANGE`:** Загрузка настроек из хранилища, инициализация шины One-Wire, создание объекта ds.
    *   **`EVENT_UPDATE`:** Автоматический поиск датчиков при countDS=0 (пошагово с задержкой 3 сек).

*   `virtual void onReceiveValue(MWOSNetReceiverFields * receiverDat)` *(переопределён)*
    *   Обработка команды записи в параметр address.
    *   При значении 0 — сброс адреса конкретного датчика.
    *   При значении -1 — сброс всех адресов и countDS=0.

### Поиск датчиков

*   `void updateAllDS()` *(приватный)*
    *   Выполняет поиск всех датчиков на шине One-Wire.
    *   Сохраняет новые адреса в EEPROM и устанавливает флаг изменения параметра.
    *   При заполнении всех каналов генерирует ошибку 3.

*   `bool selectDS(int16_t numDS)` *(приватный)*
    *   Выбирает конкретный датчик на шине по индексу и запускает конвертацию температуры.
    *   **Возвращает:** `bool` — успех выбора датчика.

*   `int16_t readDSValue(int16_t numDS)` *(приватный)*
    *   Читает результат конвертации температуры с выбранного датчика.
    *   Проверяет CRC данных, преобразует сырые данные в температуру.
    *   **Возвращает:** `int16_t` — температура в сотых градуса (°C × 100).

## Пример использования

### Базовое подключение

```cpp
#include "MWOSSensorDS.h"

// Создаём тип для 8 датчиков на одной шине
MWOSSensorDS<8> sensorDS;

// Экземпляр модуля с портом 4
sensorDS(4);

void setup() {
    // Настройки по умолчанию (загружаются только если хранилище не инициализировано)
    sensorDS._getTimeout = 10;  // Опрос датчиков каждые 1 секунду
    sensorDS._fromValue = 0;    // Нижний порог для бинарного значения (0°C)
    sensorDS._toValue = 5000;   // Верхний порог для бинарного значения (50°C)
    
    // Запуск системы - загружает настройки из хранилища
    mwos.start();
}

void loop() {
    // Все действия выполняются внутри модулей через обработчики событий
    mwos.update();
}
```

### Подключение с управлением питанием

```cpp
#include "MWOSSensorDS.h"

// Создаём модуль с 4 датчиками
MWOSSensorDS<4> sensorDS;

void setup() {
    // Порт 4 - шина One-Wire, порт 5 - управление питанием
    sensorDS(4, 5);
    
    // Настройка таймаута опроса (минимум 1 секунда для DS18B20)
    sensorDS._getTimeout = 10;
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
    
    // Чтение температуры с первого датчика (°C × 100)
    int32_t temp = sensorDS.getValueAnalog(0);
    
    // Преобразование в реальные значения
    float temperature = temp / 100.0;
}
```

### Подключение с фильтрацией Калмана

```cpp
#include "MWOSSensorDS.h"

// Создаём модуль с фильтрацией
MWOSSensorDS<8, MWOSSensorFilter<8>> sensorDS;

void setup() {
    // Настройка коэффициентов коррекции (загружаются только если хранилище не инициализировано)
    sensorDS._coef[0] = 1.0;
    sensorDS._subCoef[0] = 0.0;
    
    // Включение фильтра Калмана
    sensorDS._filter[0] = EFM_Filter2;
    
    // Настройка параметров фильтра
    sensorDS.sensorFilter[0]->Q = 0.01;
    sensorDS.sensorFilter[0]->R = 0.1;
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
    
    // Чтение фильтрованного значения температуры
    double temp = sensorDS.getFilteredValue(0);
}
```

## Способы настройки параметров

Значения параметров модулей можно задавать несколькими способами:

1.  **Значения по умолчанию в коде** (до `mwos.start()`)
    ```cpp
    void setup() {
        sensorDS._getTimeout = 10;  // Опрос каждые 1 секунду
        sensorDS._fromValue = 0;    // Нижний порог (0°C)
        sensorDS._toValue = 5000;   // Верхний порог (50°C)
        mwos.start();
    }
    ```
    **Важно:** Настройки задаются **до** вызова `mwos.start()`. При первом запуске контроллера значения из хранилища не инициализированы, поэтому будут использованы заданные по умолчанию. При последующих запусках значения загружаются из хранилища, что позволяет изменять их удалённо.

2.  **Файл конфигурации `defaults.cfg`** (для контроллеров с файловой системой)
    ```
    ; 1 - Порт шины One-Wire
    ds:pin = 4
    ; 4 - Минимальное время между опросами [сек/10]
    ds:getTimeout = 10
    ; 10 - Нижний порог для температуры (°C × 100)
    ds:fromValue = 0
    ; 11 - Верхний порог для температуры (°C × 100)
    ds:toValue = 5000
    ; 19 - Адреса датчиков (8 значений, 0 = не назначен)
    ds:address = 0, 0, 0, 0, 0, 0, 0, 0
    ```
    Файл размещается в корневой директории файловой системы контроллера.

3.  **Удалённая настройка через web-интерфейс или mwosApp**
    Параметры можно изменять в реальном времени через:
    *   Веб-интерфейс контроллера
    *   Кроссплатформенное приложение `mwosApp`

4.  **Команды через Serial-терминал**
    ```
    {ds:pin = 4}
    {ds:getTimeout = 10}
    {ds:address:0 = 283749201837465}
    {ds:address = -1}  ; сброс всех адресов
    ```
    Команды отправляются в текстовом виде через последовательный порт.

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `pin` | FOR_PIN | EEPROM или NO в зависимости от конфигурации |
| `power` | FOR_PIN | EEPROM или NO в зависимости от конфигурации |
| `address` | EEPROM | 64-битные адреса датчиков, сохраняются для быстрого доступа |
| `valueBin` | NO | Вычисляемое значение, не хранится |
| `value` | NO | Последние показания температуры, не хранится |
| `fromValue`, `toValue` | EEPROM | Пороговые значения для бинарной интерпретации |
| `analogFilterMin`, `analogFilterMax` | EEPROM | Параметры фильтрации аналоговых значений |
| `binFilter`, `getTimeout` | EEPROM | Тайминги опроса датчиков |
| `invert`, `alwaysOff`, `pullOff` | EEPROM | Флаги режима работы |
| `pull` | EEPROM | Режим подтяжки портов |
| `analogSend`, `sensorDigital`, `average` | EEPROM | Режимы отправки и обработки данных |
| `error` | NO | Код ошибки, не хранится |
| `valueCount` | NO | Счётчик измерений, не хранится |

## Важные замечания

1.  **Порядок инициализации:** Все настройки по умолчанию должны задаваться **до** вызова `mwos.start()`. После запуска системы значения загружаются из хранилища, и повторная установка значений в коде будет перезаписывать сохранённые настройки при каждой перезагрузке.

2.  **Таймаут опроса:** Датчики DS18B20 требуют до 750мс на конвертацию температуры при максимальном разрешении. Устанавливайте `getTimeout` не менее 10 (1 секунда) для избежания конфликтов при опросе.

3.  **Единицы измерения:** Температура возвращается в сотых градуса Цельсия (°C × 100). Для получения реальных значений разделите на 100.0. Например, значение 2500 = 25.00°C.

4.  **Использование с MWOSSensorFilter:** При указании `MWOSSensorFilter` вместо `MWOSSensorAnalog` модуль получает дополнительные параметры для фильтрации Калмана (ID 30-40). Это увеличивает потребление памяти, но улучшает стабильность показаний.

5.  **Подтяжка портов:** Для шины One-Wire рекомендуется использовать режим подтяжки 2 (На плюс) или внешний резистор 4.7кОм между шиной и питанием.

6.  **Коды ошибок:** 0 = нет ошибки, 1 = ошибка reset шины, 2 = ошибка CRC данных, 3 = все адреса заполнены (нет места для новых датчиков).

7.  **Архитектура MWOS:** Все действия выполняются внутри модулей через обработчики событий. Пользовательский код в `loop()` должен содержать только вызов `mwos.update()`. Для расширения функциональности создавайте собственные модули, наследуясь от `MWOSModule`.

8.  **Поиск датчиков:** Автоматический поиск выполняется пошагово при `countDS=0` с задержкой 3 секунды между шагами. Это позволяет избежать перегрузки шины при поиске множества датчиков.

9.  **Сброс адресов:** Для сброса адреса конкретного датчика запишите 0 в параметр `address` с нужным индексом. Для сброса всех адресов запишите -1 в параметр `address`.

10. **Поддерживаемые модели:** DS18B20 (12 бит), DS18S20 (9-12 бит), DS1822. Модуль автоматически определяет разрешение и корректирует время конвертации.

11. **Питание шины:** При использовании порта питания (`power`) модуль автоматически управляет включением/выключением питания шины перед опросом датчиков для экономии энергии.

*   **MWOSSensorAnalog**: Базовый класс для аналоговых датчиков. Добавляет поддержку пороговых значений и фильтрации аналоговых сигналов.
*   **MWOSSensorFilter**: Расширение для фильтрации Калмана и коррекции показаний коэффициентами.
*   **Библиотека OneWireNg**: https://github.com/crankyoldgit/OneWireNg — библиотека для работы с шиной One-Wire.
*   **MWOSSensorDHT**: Модуль для датчиков температуры и влажности DHT серии.
