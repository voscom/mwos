# MWOSSensorFilter

**Имя модуля:** `sensor`

**MWOSSensorFilter** — расширение аналоговых датчиков с коррекцией показаний коэффициентами и фильтрацией Калмана. Предназначен только для наследования.

## Описание

Класс предназначен для:
*   **Коррекция показаний:** Применение коэффициентов умножения и вычитания к аналоговым значениям датчиков
*   **Фильтрация Калмана:** Сглаживание шумов измерений с помощью алгоритма Калмана
*   **Расширенный фильтр Калмана (EKF):** Предсказание выхода на плато для стабильных показаний
*   **Гибкая настройка:** Настраиваемые параметры шума процесса и шума измерений для каждого датчика

**Наследование:** `MWOSSensorFilter<sensorsCount>` → `MWOSSensorAnalog<sensorsCount>` → `MWOSSensorBase<sensorsCount>` → `MWOSModule`

**Тип модуля:** `MODULE_SENSOR`

**Зависимости:**
*   Библиотека `MWOSSensorAnalog.h` (базовый класс)
*   Библиотека `SensorFilter.h` (алгоритмы фильтрации)
*   Библиотека `SensorFilter1.h`, `SensorFilter2.h`, `SensorFilter3.h`, `SensorFilter4.h` (варианты фильтров)

## Основные возможности

*   **Коррекция коэффициентами:** Умножение и вычитание значений для калибровки датчиков
*   **4 режима фильтрации:** Выбор из 4 вариантов алгоритмов фильтра Калмана
*   **Предсказание плато:** EKF предсказывает стабилизацию показаний для медленных датчиков
*   **Дробные значения:** Поддержка float/double для точных вычислений
*   **Гибкое наследование:** Может использоваться вместо MWOSSensorAnalog при объявлении модулей (MWOSSensorADC, MWOSSensorDS и др.)
*   **Индивидуальные настройки:** Отдельные параметры фильтра для каждого датчика в модуле

## Параметры конфигурации

Класс использует параметры базовых классов и добавляет свои специфические параметры:

| ID | Название | Тип | Группа | Хранилище | Кол-во | Описание |
|:--:|:---|:---|:---|:---|:---:|:---|
| 0 | `valueBin` | bits1 | readonly | NO | sensorsCount | **Бинарные показания.** Пороговое значение на основе fromValue/toValue |
| 1 | `pin` | int16 | pin | FOR_PIN | sensorsCount | **Порты датчиков.** Номера портов для каждого датчика (в потомках) |
| 2 | `valueCount` | uint32 | option | NO | 1 | **Количество измерений.** Счётчик запросов показаний с момента включения |
| 3 | `binFilter` | uint8 | option | EEPROM | 1 | **Фильтр переключения.** Время фильтрации бинарных переключений [сек/10] |
| 4 | `getTimeout` | uint16 | option | EEPROM | 1 | **Таймаут опроса.** Минимальное время между опросами датчиков [сек/10] |
| 5 | `invert` | bits1 | option | EEPROM | 1 | **Инверсия.** Инвертировать бинарное значение датчика |
| 6 | `alwaysOff` | bits1 | option | EEPROM | 1 | **Всегда отключено.** Принудительное отключение опроса датчиков |
| 7 | `pull` | bits3 | option | EEPROM | 1 | **Подтяжка портов.** Режим подтяжки (0=Вход;1=На землю;2=На плюс;3=АЦП) |
| 8 | `pullOff` | bits1 | option | EEPROM | 1 | **Отключение подтяжки.** Выключать подтяжку между опросами |
| 9 | `value` | int32 | readonly | NO | sensorsCount | **Аналоговые показания.** Последние значения датчиков |
| 10 | `fromValue` | int32 | option | EEPROM | 1 | **Нижний порог.** Значение для интерпретации бинарной единицы |
| 11 | `toValue` | int32 | option | EEPROM | 1 | **Верхний порог.** Значение для интерпретации бинарной единицы |
| 12 | `analogSend` | bits1 | option | EEPROM | 1 | **Отправка аналоговых.** 0=только при изменении бинарного, 1=при изменении аналогового |
| 14 | `sensorDigital` | bits1 | option | EEPROM | 1 | **Цифровой режим.** При 1 работает как бинарный датчик |
| 15 | `analogFilterMin` | uint16 | option | EEPROM | 1 | **Минимальное изменение.** Игнорировать изменения меньше этого значения |
| 16 | `analogFilterMax` | uint16 | option | EEPROM | 1 | **Максимальное изменение.** Считать сбоем изменения больше этого значения |
| 17 | `error` | int8 | readonly | NO | 1 | **Код ошибки.** Код последней ошибки чтения датчика |
| 18 | `average` | bits1 | option | EEPROM | 1 | **Усреднение.** Вычислять среднее арифметическое с предыдущим значением |
| 30 | `result` | double64 | option | NO | sensorsCount | **Результат с фильтрами.** Значение с учётом коэффициентов и фильтров |
| 31 | `coef` | float32 | option | EEPROM | sensorsCount | **Коэффициент умножения.** Применяется к аналоговому значению |
| 32 | `subCoef` | float32 | option | EEPROM | sensorsCount | **Коэффициент вычитания.** Вычитается перед умножением на coef |
| 34 | `predictedEKF` | double64 | option | NO | sensorsCount | **Предсказание EKF.** Предсказанное значение выхода на плато |
| 35 | `filter` | bits3 | option | EEPROM | sensorsCount | **Режим фильтра.** 0=Выкл;1=Фильтр1;2=Фильтр2;3=Фильтр3;4=Фильтр4 |
| 36 | `kalmanQ` | double64 | option | EEPROM | sensorsCount | **Шум процесса.** Чем выше, тем больше доверия новым данным |
| 37 | `kalmanR` | double64 | option | EEPROM | sensorsCount | **Шум измерений.** Чем выше, тем больше сглаживание |
| 38 | `kalmanTau` | float32 | option | EEPROM | sensorsCount | **Время выхода на плато.** Предполагаемое время стабилизации [сек] |
| 39 | `filterFinish` | bits1 | option+readonly | NO | sensorsCount | **Окончание фильтрации.** Признак завершения замеров с фильтрацией |
| 40 | `filterOptions` | float32 | option | NVS | FilterOptionsCount | **Доп. настройки фильтров.** Общие для всех фильтров модуля |

**Примечание:**
*   **Кол-во:** Параметры с `sensorsCount` имеют индивидуальное значение для каждого датчика. Остальные параметры — по 1 значению.
*   **Параметр 40 (filterOptions):** Имеет `FilterOptionsCount` значений (по умолчанию 5). Общие настройки для всех фильтров модуля.
*   **Параметр 35 (filter):** Формат значений: "Выкл;Фильтр1;Фильтр2;Фильтр3;Фильтр4"

## Конструкторы

Класс является шаблонным: `MWOSSensorFilter<sensorsCount>`.

1.  **По умолчанию**
    ```cpp
    MWOSSensorFilter();
    ```
    Создаёт модуль с параметрами по умолчанию. Коэффициенты умножения инициализируются единицей, коэффициенты вычитания — нулём.

2.  **С расширенной фильтрацией для потомков**
    ```cpp
    MWOSSensorADC<4, MWOSSensorFilter<4>> sensorADC;
    ```
    Создаёт модуль потомка (например, MWOSSensorADC) с поддержкой фильтрации Калмана и коррекции показаний коэффициентами.

## Основные методы

### Чтение значений

*   `virtual int64_t getValue(MWOSParam * param, int16_t arrayIndex)` *(переопределён)*
    *   Возвращает значение запрошенного параметра.
    *   Для параметров 30-40 возвращает значения фильтрации и коэффициентов.
    *   **Возвращает:** `int64_t` — значение параметра.

*   `double getFloatValue(int16_t arrayIndex)`
    *   Получить значение с учётом коэффициентов коррекции.
    *   Формула: `value = (rawValue - subCoef) * coef`
    *   **Возвращает:** `double` — скорректированное значение.

*   `double getFilteredValue(int16_t arrayIndex)`
    *   Получить значение с применённым фильтром Калмана.
    *   При отключённом фильтре возвращает `getFloatValue()`.
    *   **Возвращает:** `double` — фильтрованное значение.

### Инициализация

*   `virtual void onEvent(MWOSModeEvent modeEvent)` *(переопределён)*
    *   **`EVENT_INIT` / `EVENT_CHANGE`:** Загрузка настроек из хранилища, инициализация объектов фильтров, настройка параметров Калмана.
    *   **`EVENT_UPDATE`:** Вызывает обработку базового класса для опроса датчиков.

*   `virtual bool readBoolValue(uint16_t arrayIndex)` *(переопределён)*
    *   Вызывает стандартный обработчик для чтения нового значения.
    *   При включённом фильтре обновляет фильтр Калмана новым значением.
    *   **Возвращает:** `bool` — бинарное состояние датчика.

## Пример использования

### Подключение с фильтрацией Калмана

```cpp
#include "MWOSSensorFilter.h"
#include "MWOSSensorADC.h"

// Создаём модуль АЦП с фильтрацией
MWOSSensorADC<4, MWOSSensorFilter<4>> sensorADC;

// Массив пинов в PROGMEM
const uint8_t PROGMEM pins[] = {34, 35, 36, 39};

// Экземпляр модуля
sensorADC(pins);

void setup() {
    // Настройки по умолчанию (загружаются только если хранилище не инициализировано)
    sensorADC._coef[0] = 1.0;      // Коэффициент умножения
    sensorADC._subCoef[0] = 0.0;   // Коэффициент вычитания
    sensorADC._filter[0] = 2;      // Фильтр 2 (Калман)
    sensorADC._kalmanTau[0] = 5.0; // Время выхода на плато 5 сек
    
    // Запуск системы - загружает настройки из хранилища
    mwos.start();
}

void loop() {
    // Все действия выполняются внутри модулей через обработчики событий
    mwos.update();
}
```

### Настройка параметров фильтра Калмана

```cpp
#include "MWOSSensorFilter.h"
#include "MWOSSensorDS.h"

// Создаём модуль DS18x20 с фильтрацией
MWOSSensorDS<8, MWOSSensorFilter<8>> sensorDS;

void setup() {
    // Настройка коэффициентов коррекции (загружаются только если хранилище не инициализировано)
    sensorDS._coef[0] = 1.0;
    sensorDS._subCoef[0] = 0.0;
    
    // Включение фильтра Калмана
    sensorDS._filter[0] = 2;  // Фильтр 2
    
    // Настройка параметров фильтра
    // Q - шум процесса (чем выше, тем больше доверия новым данным)
    sensorDS.sensorFilter[0]->Q = 0.01;
    // R - Шум измерений (чем выше, тем больше сглаживание)
    sensorDS.sensorFilter[0]->R = 0.1;
    // tau - предполагаемое время выхода на плато (сек)
    sensorDS.sensorFilter[0]->tau = 5.0;
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
    
    // Чтение фильтрованного значения
    double temp = sensorDS.getFilteredValue(0);
}
```

### Чтение значений с фильтрацией

```cpp
#include "MWOSSensorFilter.h"
#include "MWOSSensorADC.h"

// Создаём модуль с фильтрацией
MWOSSensorADC<4, MWOSSensorFilter<4>> sensorADC;

void setup() {
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
    
    // Чтение сырого значения
    int32_t raw = sensorADC.getValueAnalog(0);
    
    // Чтение значения с коэффициентами
    double corrected = sensorADC.getFloatValue(0);
    
    // Чтение фильтрованного значения
    double filtered = sensorADC.getFilteredValue(0);
    
    // Чтение предсказания EKF
    double predicted = sensorADC.getValueByParamId(0.0, 34, 0);
}
```

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `coef`, `subCoef` | EEPROM | Коэффициенты коррекции, сохраняются |
| `filter` | EEPROM | Режим фильтрации, сохраняется |
| `kalmanQ`, `kalmanR`, `kalmanTau` | EEPROM | Параметры фильтра Калмана, сохраняются |
| `filterOptions` | NVS | Дополнительные настройки фильтров (ESP32) |
| `result`, `predictedEKF` | NO | Вычисляемые значения, не хранятся |
| `filterFinish` | NO | Флаг состояния, не хранится |
| `value` | NO | Последние показания датчиков, не хранятся |
| `fromValue`, `toValue` | EEPROM | Пороговые значения для бинарной интерпретации |
| `analogFilterMin`, `analogFilterMax` | EEPROM | Параметры фильтрации аналоговых значений |
| `binFilter`, `getTimeout` | EEPROM | Тайминги опроса датчиков |
| `invert`, `alwaysOff`, `pullOff` | EEPROM | Флаги режима работы |
| `pull` | EEPROM | Режим подтяжки портов |
| `analogSend`, `sensorDigital`, `average` | EEPROM | Режимы отправки и обработки данных |
| `error` | NO | Код ошибки, не хранится |
| `valueCount` | NO | Счётчик измерений, не хранится |

## Важные замечания

1.  **Только для наследования:** Класс `MWOSSensorFilter` предназначен только для наследования и не используется самостоятельно. Он используется как шаблонный параметр при объявлении модулей-потомков (MWOSSensorADC, MWOSSensorDS, MWOSSensorDHT и др.).

2.  **Порядок инициализации:** Все настройки по умолчанию должны задаваться **до** вызова `mwos.start()`. После запуска системы значения загружаются из хранилища, и повторная установка значений в коде будет перезаписывать сохранённые настройки при каждой перезагрузке.

3.  **Параметры фильтра Калмана:**
    *   **Q (шум процесса):** Чем выше значение, тем больше доверия новым данным измерений. Рекомендуемый диапазон: 0.001 - 0.1.
    *   **R (шум измерений):** Чем выше значение, тем больше сглаживание показаний. Рекомендуемый диапазон: 0.1 - 10.
    *   **tau (время выхода на плато):** Предполагаемое время стабилизации показаний в секундах. Используется EKF для предсказания.

4.  **Режимы фильтрации:** Доступны 4 режима фильтра (EFM_Filter1 - EFM_Filter4). Каждый режим использует свой алгоритм фильтрации с разными характеристиками сглаживания и скорости реакции.

5.  **Потребление памяти:** Использование `MWOSSensorFilter` увеличивает потребление памяти из-за хранения объектов фильтров и дополнительных параметров. Не рекомендуется для систем с ограниченной памятью.

6.  **Хранилище NVS:** Параметр `filterOptions` использует хранилище NVS (только ESP32). На других платформах может использоваться EEPROM.

7.  **Архитектура MWOS:** Все действия выполняются внутри модулей через обработчики событий. Пользовательский код в `loop()` должен содержать только вызов `mwos.update()`. Для расширения функциональности создавайте собственные модули, наследуясь от `MWOSModule`.

8.  **Инициализация фильтров:** При изменении режима фильтрации (`filter`) старый объект фильтра удаляется и создаётся новый с загруженными параметрами из хранилища.

9.  **Режим паузы:** При установке режима паузы (`alwaysOff=true`) фильтр возвращает последнее сохранённое значение через метод `getFinishResult()`.

10. **Обновление фильтра:** Фильтр Калмана обновляется при каждом чтении значения датчика через метод `readBoolValue()`. Метод `update()` вызывается автоматически с новым измерением и текущим временем.

## См. также

*   **MWOSSensorAnalog**: Базовый класс для аналоговых датчиков. Предоставляет инфраструктуру для работы с аналоговыми значениями и пороговой интерпретацией.
*   **MWOSSensorBase**: Базовый класс для всех датчиков. Реализует общую логику опроса, фильтрации бинарных значений и управления состоянием датчиков.
*   **MWOSSensorADC**: Модуль для подключения аналоговых датчиков к портам АЦП микроконтроллера. Может использовать `MWOSSensorFilter` для расширенной фильтрации.
*   **MWOSSensorDS**: Модуль для цифровых термометров DS18x20 на шине One-Wire. Может использовать `MWOSSensorFilter` для сглаживания показаний температуры.
*   **MWOSSensorDHT**: Модуль для датчиков температуры и влажности DHT серии. Может использовать `MWOSSensorFilter` для стабилизации показаний.
*   **Библиотека SensorFilter**: Алгоритмы фильтрации Калмана для встраиваемых систем.
