# MWOSSensorBase

**Имя модуля:** `sensor`

**MWOSSensorBase** — базовый шаблонный класс для всех модулей датчиков в системе MWOS. Реализует общую логику работы с бинарными состояниями датчиков, фильтрацию переключений и управление подтяжкой портов.

## Описание

Класс предназначен для:
*   **Базовая инфраструктура датчиков:** Предоставляет универсальный механизм работы с множественными датчиками одного типа
*   **Фильтрация переключений:** Игнорирование кратковременных переключений для устранения дребезга контактов
*   **Управление подтяжкой портов:** Настройка режима подтяжки и отключение подтяжки между опросами для экономии энергии
*   **Режим паузы:** Возможность временно отключить опрос датчиков без изменения настроек

**Наследование:** `MWOSSensorBase<sensorsCount>` → `MWOSModule`

**Тип модуля:** `MODULE_SENSOR`

**Зависимости:**
*   Библиотека `MWOSModule.h` (базовый класс)
*   Библиотека `MWTimeout.h` (таймеры)
*   Платформа с поддержкой портов (ESP32/ESP8266/STM32/Arduino)

## Основные возможности

*   **Множественные датчики:** Поддержка до `sensorsCount` независимых датчиков в одном модуле через шаблон
*   **Битовая структура состояния:** Эффективное хранение состояния каждого датчика в едином байте (бинарное значение, флаг изменения, фильтр, запрос)
*   **Фильтрация переключений:** Настройка времени фильтрации для игнорирования дребезга контактов механических датчиков
*   **Управление подтяжкой:** Автоматическое включение/выключение подтяжки портов перед опросом для экономии энергии
*   **Режим паузы:** Временная остановка опроса датчиков с сохранением настроек
*   **Счётчик измерений:** Подсчёт количества опросов датчиков с момента включения контроллера
*   **Инвазия значений:** Программная инверсия логического состояния датчиков

## Параметры конфигурации

Класс использует параметры базовых классов и добавляет свои специфические параметры:

| ID | Название | Тип | Группа | Хранилище | Кол-во | Описание |
|:--:|:---|:---|:---|:---|:---:|:---|
| 0 | `valueBin` | bits1 | readonly | NO | sensorsCount | **Бинарные показания.** Текущее состояние каждого датчика (0/1). Инвертируется при необходимости |
| 2 | `valueCount` | uint32 | option | NO | 1 | **Количество измерений.** Счётчик запросов показаний с момента включения контроллера |
| 3 | `binFilter` | uint8 | option | EEPROM | 1 | **Фильтр переключения.** Время фильтрации бинарных переключений [сек/10]. Кратковременные переключения игнорируются |
| 4 | `getTimeout` | uint16 | option | EEPROM | 1 | **Таймаут опроса.** Минимальное время между опросами датчиков [сек/10] |
| 5 | `invert` | bits1 | option | EEPROM | 1 | **Инверсия.** Инвертировать бинарное значение датчика (0→1, 1→0) |
| 6 | `alwaysOff` | bits1 | option | EEPROM | 1 | **Всегда отключено.** Принудительное отключение опроса датчиков. Одновременно с invert — всегда включено |
| 7 | `pull` | bits3 | option | EEPROM | 1 | **Подтяжка портов.** Режим подтяжки (0=Вход;1=На землю;2=На плюс;3=АЦП) |
| 8 | `pullOff` | bits1 | option | EEPROM | 1 | **Отключение подтяжки.** Выключать подтяжку между опросами для экономии энергии |

**Примечание:**
*   **Кол-во:** Параметр `valueBin` имеет `sensorsCount` значений (по одному на каждый датчик). Остальные параметры — по 1 значению для всего модуля.
*   **Единицы измерения:** `getTimeout`, `binFilter` — десятые доли секунды. Значение 10 = 1 секунда.
*   **Параметр 7 (pull):** Формат значений: "Вход;На землю;На плюс;АЦП"

## Конфигурация модуля

```
; модуль sensor - базовый класс для бинарных датчиков
; 0 - Бинарные показания (sensorsCount значений)
sensor:valueBin = 0, 0, 0, 0, 0
; 2 - Количество измерений с момента включения
sensor:valueCount = 0
; 3 - Фильтр переключения бинарного значения [сек/10]
sensor:binFilter = 0
; 4 - Минимальное время между опросами показаний датчика [сек/10]
sensor:getTimeout = 1
; 5 - Инвертировать значение
sensor:invert = 0
; 6 - Всегда отключено
sensor:alwaysOff = 0
; 7 - Подтяжка портов (0=Вход;1=На землю;2=На плюс;3=АЦП)
sensor:pull = 0
; 8 - Выключать подтяжку порта датчика между опросами
sensor:pullOff = 0
```

**Примечание:**
*   **Кол-во значений:** Параметр `valueBin` имеет `sensorsCount` значений (по одному на каждый датчик). Остальные параметры — по 1 значению.
*   **Единицы измерения:** `getTimeout`, `binFilter` — десятые доли секунды.
*   **Значения подтяжки:** 0=Вход без подтяжки, 1=Подтяжка к земле, 2=Подтяжка к питанию, 3=Режим АЦП

## Конструкторы

Класс является шаблонным: `MWOSSensorBase<sensorsCount>`.

1.  **По умолчанию**
    ```cpp
    MWOSSensorBase();
    ```
    Создаёт модуль с параметрами по умолчанию. Имя модуля устанавливается в "sensor". Тип модуля устанавливается в `MODULE_SENSOR`.

## Основные методы

### Чтение значений

*   `bool getValueBool(int16_t arrayIndex = 0)`
    *   Запрашивает актуальное бинарное значение указанного датчика.
    *   Генерирует событие `EVENT_GET_VALUE` и возвращает полученное значение.
    *   **Возвращает:** `bool` — бинарное состояние датчика (с учётом инверсии).

*   `virtual bool readBoolValue(uint16_t arrayIndex)` *(виртуальный)*
    *   Вызывается через таймаут для всех индексов датчиков.
    *   Переопределяется в классах-потомках для реализации конкретного механизма чтения.
    *   **Возвращает:** `bool` — бинарное состояние датчика (по умолчанию `true`).

### Инициализация

*   `virtual void onEvent(MWOSModeEvent modeEvent)` *(переопределён)*
    *   **`EVENT_INIT` / `EVENT_CHANGE`:** Загрузка настроек из хранилища, инициализация всех датчиков, запуск таймера опроса.
    *   **`EVENT_UPDATE`:** Вызывает метод `updateSensor()` для опроса датчиков.

*   `virtual void initSensor(int16_t index, bool pullOn)` *(виртуальный)*
    *   Инициализация порта конкретного датчика.
    *   Переопределяется в классах-потомках для настройки конкретного типа порта.
    *   **Параметры:** `index` — индекс датчика, `pullOn` — включить подтяжку.

### Управление состоянием

*   `void pause()`
    *   Переводит все датчики модуля в режим паузы.
    *   Устанавливает флаг `alwaysOff` в истину.
    *   Устанавливает флаг изменения параметра `p_alwaysOff`.

*   `void resume()`
    *   Снимает датчики с паузы и восстанавливает нормальный режим опроса.
    *   Сбрасывает флаг `alwaysOff` в ноль.
    *   Сохраняет значение в хранилище и вызывает событие `EVENT_CHANGE` для переинициализации.

*   `bool isPaused()`
    *   Проверяет текущее состояние паузы модуля.
    *   **Возвращает:** `bool` — истину, если датчики находятся в режиме паузы.

### Внутренние методы

*   `void updateSensor()`
    *   Вызывается каждый тик операционной системы через событие `EVENT_UPDATE`.
    *   Управляет циклом опроса всех датчиков модуля.
    *   При включённом режиме `pullOff` сначала включает подтяжку, ожидает стабилизации, выполняет опрос, затем отключает подтяжку.

*   `void readSensor(uint16_t index)`
    *   Выполняет чтение конкретного датчика по индексу.
    *   Сравнивает новое значение с предыдущим, применяет фильтр переключений при необходимости.
    *   Устанавливает флаг изменения при обнаружении переключения.

## Пример использования

### Базовое подключение

```cpp
#include "MWOSSensorBase.h"

// Создаём тип для 5 бинарных датчиков
MWOSSensorBase<5> sensorBase;

void setup() {
    // Настройки по умолчанию (загружаются только если хранилище не инициализировано)
    sensorBase._binFilter = 2;   // Фильтрация переключений 0.2 сек
    sensorBase._getTimeout = 5;  // Опрос датчиков каждые 0.5 сек
    sensorBase._pull = 2;        // Подтяжка к питанию
    
    // Запуск системы - загружает настройки из хранилища
    mwos.start();
}

void loop() {
    // Все действия выполняются внутри модулей через обработчики событий
    mwos.update();
}
```

### Управление паузой датчиков

```cpp
#include "MWOSSensorBase.h"

// Создаём модуль с 3 датчиками
MWOSSensorBase<3> sensorBase;

void setup() {
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
    
    // Проверка состояния паузы
    if (sensorBase.isPaused()) {
        // Датчики на паузе
    }
    
    // Постановка на паузу
    sensorBase.pause();
    
    // Снятие с паузы
    sensorBase.resume();
}
```

### Чтение значений датчиков

```cpp
#include "MWOSSensorBase.h"

// Создаём модуль с 4 датчиками
MWOSSensorBase<4> sensorBase;

void setup() {
    // Настройка инверсии для всех датчиков
    sensorBase._invert = 1;  // Инвертировать логическое состояние
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
    
    // Чтение бинарного значения датчика
    bool state = sensorBase.getValueBool(0);
    
    // Чтение счётчика измерений
    uint32_t count = sensorBase._valueCount;
}
```

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `valueBin` | NO | Вычисляемое значение, не хранится |
| `valueCount` | NO | Счётчик измерений, не хранится |
| `binFilter`, `getTimeout` | EEPROM | Тайминги опроса датчиков |
| `invert`, `alwaysOff`, `pullOff` | EEPROM | Флаги режима работы |
| `pull` | EEPROM | Режим подтяжки портов |

## Важные замечания

1.  **Только для наследования:** Класс `MWOSSensorBase` предназначен только для наследования и не используется самостоятельно. Конкретные модули датчиков наследуются от этого класса и переопределяют виртуальные методы.

2.  **Порядок инициализации:** Все настройки по умолчанию должны задаваться **до** вызова `mwos.start()`. После запуска системы значения загружаются из хранилища, и повторная установка значений в коде будет перезаписывать сохранённые настройки при каждой перезагрузке.

3.  **Фильтрация переключений:** Параметр `binFilter` задаёт время в десятых долях секунды. Кратковременные переключения, длящиеся меньше заданного времени, игнорируются. Это позволяет устранить дребезг контактов механических датчиков (кнопки, герконы, концевые выключатели).

4.  **Подтяжка портов:** Для бинарных датчиков рекомендуется использовать режим подтяжки 1 (На землю) или 2 (На плюс) в зависимости от схемы подключения. Режим 3 (АЦП) используется только при пороговом срабатывании через аналоговый вход.

5.  **Режим pullOff:** При включённом параметре `pullOff` модуль автоматически отключает подтяжку портов между опросами для экономии энергии. Перед каждым опросом подтяжка включается на время `MWOS_SENSOR_PULL_TIME_MS` (по умолчанию 50 мс) для стабилизации сигнала.

6.  **Архитектура MWOS:** Все действия выполняются внутри модулей через обработчики событий. Пользовательский код в `loop()` должен содержать только вызов `mwos.update()`. Для расширения функциональности создавайте собственные модули, наследуясь от `MWOSModule`.

7.  **Битовая структура:** Для каждого датчика модуль хранит структуру битовых флагов в едином байте, что позволяет эффективно использовать память микроконтроллера. Биты: 0=бинарное значение, 1=флаг изменения, 2=активный фильтр, 3=запрос, 4=аналоговое изменение, 5-7=резерв.

8.  **Таймаут опроса:** Параметр `getTimeout` задаёт минимальное время между последовательными опросами всех датчиков модуля. Увеличьте это значение для медленных датчиков или для снижения нагрузки на систему.

9.  **Счётчик измерений:** Параметр `valueCount` является 32-битным и может переполниться после ~4 миллиардов измерений. Предусмотрите сброс при длительной работе системы.

10. **Режим паузы:** При установке режима паузы (`alwaysOff=true`) опрос датчиков полностью прекращается. Для восстановления нормального режима используйте метод `resume()`, который выполняет переинициализацию датчиков через событие `EVENT_CHANGE`.

## См. также

*   **MWOSModule**: Базовый класс для всех модулей контроллера. Реализует общую инфраструктуру модулей, параметры и события.
*   **MWOSSensorAnalog**: Базовый класс для аналоговых датчиков. Наследуется от `MWOSSensorBase`, добавляет поддержку аналоговых значений и пороговой интерпретации.
*   **MWOSSensorBin**: Модуль для подключения бинарных датчиков к портам микроконтроллера. Наследуется от `MWOSSensorBase`.
*   **MWOSSensorADC**: Модуль для подключения аналоговых датчиков к портам АЦП микроконтроллера. Наследуется от `MWOSSensorAnalog`.
*   **Библиотека MWTimeout**: Система таймеров для отложенных операций и периодических событий.
