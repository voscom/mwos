# MWOSSensorAnalog

**Имя модуля:** `sensor`

**MWOSSensorAnalog** — базовый модуль для аналоговых датчиков с поддержкой пороговых значений, фильтрации и вычисления процентов. Предназначен только для наследования.

## Описание

Класс предназначен для:
*   **Чтение аналоговых сигналов:** Базовая инфраструктура для работы с аналоговыми датчиками
*   **Пороговая интерпретация:** Преобразование аналоговых значений в бинарные на основе диапазона fromValue/toValue
*   **Фильтрация изменений:** Игнорирование мелких и сбойных изменений аналоговых значений
*   **Вычисление процентов:** Расчёт процентного значения в заданном диапазоне

**Наследование:** `MWOSSensorAnalog<sensorsCount>` → `MWOSSensorBase<sensorsCount>` → `MWOSModule`

**Тип модуля:** `MODULE_SENSOR`

**Зависимости:**
*   Библиотека `MWOSSensorBase.h` (базовый класс)
*   Платформа с поддержкой АЦП (для конкретных реализаций)

## Основные возможности

*   **Аналоговые значения:** Хранение последних показаний датчиков в массиве `_value`
*   **Пороговая интерпретация:** Автоматическое преобразование аналоговых значений в бинарные на основе диапазона
*   **Фильтрация минимальных изменений:** Игнорирование изменений меньше `_analog_filter_min` для устранения шума
*   **Фильтрация максимальных изменений:** Отсечение сбойных значений больше `_analog_filter_max`
*   **Усреднение:** Вычисление среднего арифметического с предыдущим значением для сглаживания
*   **Режим цифрового датчика:** Переключение в режим бинарного чтения через параметр `sensorDigital`
*   **Коды ошибок:** Механизм установки и отслеживания ошибок чтения датчиков
*   **Гибкое наследование:** Возможность указания `MWOSSensorFilter` вместо `MWOSSensorAnalog` для расширенной фильтрации

## Параметры конфигурации

Класс использует параметры базовых классов и добавляет свои специфические параметры:

| ID | Название | Тип | Группа | Хранилище | Кол-во | Описание |
|:--:|:---|:---|:---|:---|:---:|:---|
| 0 | `valueBin` | bits1 | readonly | NO | sensorsCount | **Бинарные показания.** Пороговое значение на основе fromValue/toValue |
| 1 | `pin` | int16 | pin | FOR_PIN | sensorsCount | **Порты датчиков.** Номера портов для каждого датчика (в потомках) |
| 2 | `valueCount` | uint32 | option | NO | 1 | **Количество измерений.** Счётчик запросов показаний с момента включения |
| 3 | `binFilter` | uint8 | option | EEPROM | 1 | **Фильтр переключения.** Время фильтрации бинарных переключений [сек/10] |
| 4 | `getTimeout` | uint16 | option | EEPROM | 1 | **Таймаут опроса.** Минимальное время между опросами датчиков [сек/10] |
| 5 | `invert` | bits1 | option | EEPROM | 1 | **Инверсия.** Инвертировать бинарное значение датчика |
| 6 | `alwaysOff` | bits1 | option | EEPROM | 1 | **Всегда отключено.** Принудительное отключение опроса датчиков |
| 7 | `pull` | bits3 | option | EEPROM | 1 | **Подтяжка портов.** Режим подтяжки (0=Вход;1=На землю;2=На плюс;3=АЦП) |
| 8 | `pullOff` | bits1 | option | EEPROM | 1 | **Отключение подтяжки.** Выключать подтяжку между опросами |
| 9 | `value` | int32 | readonly | NO | sensorsCount | **Аналоговые показания.** Последние значения датчиков |
| 10 | `fromValue` | int32 | option | EEPROM | 1 | **Нижний порог.** Значение для интерпретации бинарной единицы |
| 11 | `toValue` | int32 | option | EEPROM | 1 | **Верхний порог.** Значение для интерпретации бинарной единицы |
| 12 | `analogSend` | bits1 | option | EEPROM | 1 | **Отправка аналоговых.** 0=только при изменении бинарного, 1=при изменении аналогового |
| 14 | `sensorDigital` | bits1 | option | EEPROM | 1 | **Цифровой режим.** При 1 работает как бинарный датчик |
| 15 | `analogFilterMin` | uint16 | option | EEPROM | 1 | **Минимальное изменение.** Игнорировать изменения меньше этого значения |
| 16 | `analogFilterMax` | uint16 | option | EEPROM | 1 | **Максимальное изменение.** Считать сбоем изменения больше этого значения |
| 17 | `error` | int8 | readonly | NO | 1 | **Код ошибки.** Код последней ошибки чтения датчика |
| 18 | `average` | bits1 | option | EEPROM | 1 | **Усреднение.** Вычислять среднее арифметическое с предыдущим значением |

**Примечание:**
*   **Кол-во:** Параметры с `sensorsCount` имеют индивидуальное значение для каждого датчика. Остальные параметры — по 1 значению.
*   **Единицы измерения:** `getTimeout`, `binFilter` — десятые доли секунды.
*   **Параметр 1 (pin):** Определяется в классах-потомках (MWOSSensorBin, MWOSSensorADC и др.).

**Примечание:**

*   **Единицы измерения:** `getTimeout`, `binFilter` — десятые доли секунды.
*   **Диапазон АЦП:** Зависит от платформы. ESP32: 0-4095 (12 бит), ESP8266: 0-1024 (10 бит).

## Конструкторы

Класс является шаблонным: `MWOSSensorAnalog<sensorsCount>`.

1.  **По умолчанию**
    ```cpp
    MWOSSensorAnalog();
    ```
    Создаёт модуль с параметрами по умолчанию. Инициализирует внутренние переменные нулевыми значениями.

2.  **С расширенной фильтрацией**
    ```cpp
    MWOSSensorAnalog<sensorsCount, MWOSSensorFilter<sensorsCount>> sensorAnalog;
    ```
    Создаёт модуль с поддержкой фильтрации Калмана и коррекции показаний коэффициентами.

## Основные методы

### Чтение значений

*   `virtual int64_t getValue(MWOSParam * param, int16_t arrayIndex)` *(переопределён)*
    *   Возвращает значение запрошенного параметра.
    *   Для параметра `value` возвращает последние аналоговые показания.
    *   **Возвращает:** `int64_t` — значение параметра.

*   `int64_t getValueAnalog(int16_t arrayIndex = 0)`
    *   Запрашивает аналоговое значение указанного датчика.
    *   **Возвращает:** `int64_t` — аналоговое значение датчика.

*   `float getValuePercent(uint16_t arrayIndex)`
    *   Вычисляет процент текущего значения в диапазоне от `fromValue` до `toValue`.
    *   **Возвращает:** `float` — процентное значение (0-100).

*   `virtual bool readBoolValue(uint16_t arrayIndex)` *(переопределён)*
    *   Выполняет чтение датчика с фильтрацией и пороговой интерпретацией.
    *   При `_sensorDigital=1` вызывает `readDigitalValue()`.
    *   Иначе вызывает `readAnalogValue()` и применяет фильтры.
    *   **Возвращает:** `bool` — бинарное состояние датчика.

*   `virtual int32_t readAnalogValue(int16_t arrayIndex)` *(виртуальный)*
    *   Опрос аналогового датчика для получения новых показаний.
    *   Переопределяется в классах-потомках для конкретной реализации.
    *   **Возвращает:** `int32_t` — аналоговое значение (по умолчанию 0).

*   `virtual int32_t readDigitalValue(int16_t arrayIndex)` *(виртуальный)*
    *   Опрос цифрового датчика для получения новых показаний.
    *   Переопределяется в классах-потомках для конкретной реализации.
    *   **Возвращает:** `int32_t` — цифровое значение (по умолчанию 0).

### Инициализация

*   `virtual void onEvent(MWOSModeEvent modeEvent)` *(переопределён)*
    *   **`EVENT_INIT` / `EVENT_CHANGE`:** Загрузка настроек из хранилища (`fromValue`, `toValue`, `analogSend`, `sensorDigital`, `analogFilterMin`, `analogFilterMax`, `average`).
    *   **`EVENT_UPDATE`:** Вызывает обработку базового класса для опроса датчиков.

### Управление ошибками

*   `void setError(uint8_t errorCode)`
    *   Устанавливает код ошибки для модуля.
    *   При изменении кода ошибки устанавливает флаг изменения параметра `error`.
    *   При ненулевом коде ошибки выводит сообщение в лог.

## Пример использования

### Базовое подключение (наследник)

```cpp
#include "MWOSSensorAnalog.h"

// Создаём тип для 5 аналоговых датчиков
MWOSSensorADC<5> sensorAnalog;

// Массив пинов в PROGMEM
const uint8_t PROGMEM pins[] = {34, 35, 36, 39, 40};

// Экземпляр модуля
sensorAnalog(pins);

void setup() {
    // Настройки по умолчанию (загружаются только если хранилище не инициализировано)
    sensorAnalog._fromValue = 500;   // Нижний порог
    sensorAnalog._toValue = 3500;    // Верхний порог
    sensorAnalog._analogFilterMin = 10;  // Минимальное изменение
    
    // Запуск системы - загружает настройки из хранилища
    mwos.start();
}

void loop() {
    // Все действия выполняются внутри модулей через обработчики событий
    mwos.update();
}
```

### Подключение с фильтрацией Калмана

```cpp
#include "MWOSSensorAnalog.h"

// Создаём модуль с фильтрацией
MWOSSensorADC<4, MWOSSensorFilter<4>> sensorAnalog;

void setup() {
    // Настройка коэффициентов коррекции (загружаются только если хранилище не инициализировано)
    sensorAnalog._coef[0] = 1.0;
    sensorAnalog._subCoef[0] = 0.0;
    
    // Включение фильтра Калмана
    sensorAnalog._filter[0] = EFM_Filter2;
    
    // Настройка параметров фильтра
    sensorAnalog.sensorFilter[0]->Q = 0.01;
    sensorAnalog.sensorFilter[0]->R = 0.1;
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
    
    // Чтение фильтрованного значения
    double value = sensorAnalog.getFilteredValue(0);
}
```

### Чтение процентного значения

```cpp
#include "MWOSSensorAnalog.h"

// Создаём модуль с 3 датчиками
MWOSSensorADC<3> sensorAnalog;

void setup() {
    // Настройка диапазона для процентного вычисления
    sensorAnalog._fromValue = 0;     // 0%
    sensorAnalog._toValue = 4095;    // 100%
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
    
    // Чтение процентного значения
    float percent = sensorAnalog.getValuePercent(0);
}
```

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `value` | NO | Последние показания датчиков, не хранится |
| `fromValue`, `toValue` | EEPROM | Пороговые значения для бинарной интерпретации |
| `analogFilterMin`, `analogFilterMax` | EEPROM | Параметры фильтрации аналоговых значений |
| `analogSend`, `sensorDigital`, `average` | EEPROM | Режимы отправки и обработки данных |
| `valueBin` | NO | Вычисляемое значение, не хранится |
| `valueCount` | NO | Счётчик измерений, не хранится |
| `binFilter`, `getTimeout` | EEPROM | Тайминги опроса датчиков |
| `invert`, `alwaysOff`, `pullOff` | EEPROM | Флаги режима работы |
| `pull` | EEPROM | Режим подтяжки портов |
| `error` | NO | Код ошибки, не хранится |

## Важные замечания

1.  **Только для наследования:** Класс `MWOSSensorAnalog` предназначен только для наследования и не используется самостоятельно. Конкретные модули датчиков (MWOSSensorADC, MWOSSensorDS, MWOSSensorDHT и др.) наследуются от этого класса и переопределяют виртуальные методы `readAnalogValue()` и `readDigitalValue()`.

2.  **Порядок инициализации:** Все настройки по умолчанию должны задаваться **до** вызова `mwos.start()`. После запуска системы значения загружаются из хранилища, и повторная установка значений в коде будет перезаписывать сохранённые настройки при каждой перезагрузке.

3.  **Пороговая интерпретация:** Бинарное значение определяется как единица, если аналоговое значение находится в диапазоне от `fromValue` до `toValue` включительно. За пределами этого диапазона — ноль.

4.  **Фильтрация минимальных изменений:** Параметр `analogFilterMin` задаёт минимальное изменение аналогового значения, которое игнорируется системой. Изменения меньше этого порога считаются шумом и отфильтровываются.

5.  **Фильтрация максимальных изменений:** Параметр `analogFilterMax` задаёт максимальное допустимое изменение аналогового значения. Изменения больше этого порога считаются сбойными и пропускаются при двукратном повторении.

6.  **Усреднение:** При включённом параметре `average` вычисляется среднее арифметическое между предыдущим и текущим значением. Это сглаживает резкие скачки показаний.

7.  **Режим цифрового датчика:** При `sensorDigital=1` модуль работает как бинарный датчик, вызывая метод `readDigitalValue()` вместо `readAnalogValue()`.

8.  **Отправка значений:** Параметр `analogSend` управляет режимом отправки данных. При 0 отправляются только изменения бинарного состояния. При 1 отправляются изменения аналогового значения с учётом фильтров.

9.  **Архитектура MWOS:** Все действия выполняются внутри модулей через обработчики событий. Пользовательский код в `loop()` должен содержать только вызов `mwos.update()`. Для расширения функциональности создавайте собственные модули, наследуясь от `MWOSModule`.

10. **Использование с MWOSSensorFilter:** При указании `MWOSSensorFilter` вместо `MWOSSensorAnalog` модуль получает дополнительные параметры для фильтрации Калмана (ID 30-40). Это увеличивает потребление памяти, но улучшает стабильность показаний.

11. **Коды ошибок:** Код ошибки сохраняется в параметре `error` (ID 17). Конкретные значения кодов определяются в классах-потомках (например, 98=ошибка чтения DHT, 99=датчик не инициализирован).

## См. также

*   **MWOSSensorBase**: Базовый класс для всех датчиков. Реализует общую логику опроса, фильтрации бинарных значений и управления состоянием датчиков.
*   **MWOSSensorBin**: Модуль для подключения бинарных датчиков к портам микроконтроллера. Наследуется от `MWOSSensorBase`.
*   **MWOSSensorADC**: Модуль для подключения аналоговых датчиков к портам АЦП микроконтроллера. Наследуется от `MWOSSensorAnalog`.
*   **MWOSSensorFilter**: Расширение для фильтрации Калмана и коррекции показаний коэффициентами. Может использоваться вместо `MWOSSensorAnalog` при объявлении модуля.
*   **MWOSSensorDS**: Модуль для цифровых термометров DS18x20 на шине One-Wire. Наследуется от `MWOSSensorAnalog`.
*   **MWOSSensorDHT**: Модуль для датчиков температуры и влажности DHT серии. Наследуется от `MWOSSensorAnalog`.
