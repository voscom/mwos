# MWOSSensorBin

**Имя модуля:** `sensor`

**MWOSSensorBin** — модуль для подключения бинарных датчиков к портам микроконтроллера. Поддерживает множественные датчики, пороговое срабатывание через АЦП и фильтрацию переключений.

## Описание

Класс предназначен для:
*   **Подключение бинарных датчиков:** Кнопки, герконы, концевые выключатели, оптопары и другие датчики с двумя состояниями
*   **Пороговое срабатывание через АЦП:** Возможность настройки порога аналогового значения для интерпретации бинарного состояния
*   **Фильтрация переключений:** Игнорирование кратковременных переключений для устранения дребезга контактов
*   **Множественные датчики:** Поддержка нескольких независимых бинарных каналов в одном модуле

**Наследование:** `MWOSSensorBin<sensorsCount>` → `MWOSSensorBase<sensorsCount>` → `MWOSModule`

**Тип модуля:** `MODULE_SENSOR`

**Зависимости:**
*   Библиотека `MWOSSensorBase.h` (базовый класс)
*   Библиотека `MWOSPins.h` (работа с портами)
*   Макрос `MWOS_SENSOR_BIN_ADC` (опционально, для поддержки АЦП)

## Основные возможности

*   **Множественные бинарные каналы:** Поддержка до `sensorsCount` независимых датчиков в одном модуле
*   **Пороговое срабатывание через АЦП:** При заданном пороге `adc_from` бинарная единица определяется при превышении аналогового значения
*   **Фильтрация переключений:** Настройка времени фильтрации для игнорирования дребезга контактов
*   **Инверсия значений:** Возможность инвертировать логическое состояние датчиков программно
*   **Управление подтяжкой портов:** Настройка режима подтяжки и отключение подтяжки между опросами для экономии энергии
*   **Режим паузы:** Возможность временно отключить опрос датчиков без изменения настроек

## Параметры конфигурации

Класс использует параметры базовых классов и добавляет свои специфические параметры:

| ID | Название | Тип | Группа | Хранилище | Кол-во | Описание |
|:--:|:---|:---|:---|:---|:---:|:---|
| 0 | `valueBin` | bits1 | readonly | NO | sensorsCount | **Бинарные показания.** Текущее состояние каждого датчика (0/1) |
| 1 | `pin` | int16 | pin | FOR_PIN | sensorsCount | **Порты датчиков.** Номера портов для каждого датчика. -1 = не подключён, -3 = ошибка |
| 2 | `valueCount` | uint32 | option | NO | 1 | **Количество измерений.** Счётчик запросов показаний с момента включения |
| 3 | `binFilter` | uint8 | option | EEPROM | 1 | **Фильтр переключения.** Время фильтрации бинарных переключений [сек/10] |
| 4 | `getTimeout` | uint16 | option | EEPROM | 1 | **Таймаут опроса.** Минимальное время между опросами датчиков [сек/10] |
| 5 | `invert` | bits1 | option | EEPROM | 1 | **Инверсия.** Инвертировать бинарное значение датчика |
| 6 | `alwaysOff` | bits1 | option | EEPROM | 1 | **Всегда отключено.** Принудительное отключение опроса датчиков |
| 7 | `pull` | bits3 | option | EEPROM | 1 | **Подтяжка портов.** Режим подтяжки (0=Вход;1=На землю;2=На плюс;3=АЦП) |
| 8 | `pullOff` | bits1 | option | EEPROM | 1 | **Отключение подтяжки.** Выключать подтяжку между опросами |
| 9 | `adc_from` | uint16 | option | EEPROM | sensorsCount | **Порог АЦП.** Только если включена поддержка АЦП. При значении > 0 бинарная единица определяется при превышении аналогового значения |

**Примечание:**
*   **Кол-во:** Параметры с `sensorsCount` имеют индивидуальное значение для каждого датчика. Остальные параметры — по 1 значению.
*   **Параметр 9 (adc_from):** Доступен только если определён макрос `MWOS_SENSOR_BIN_ADC=1`. На платформах с ограниченной памятью может быть отключён.
*   **Значение -3 в параметре pin:** Указывает на ошибку инициализации порта (порт не поддерживает заданный режим).

## Конфигурация модуля

```
; модуль sensor - бинарные датчики микроконтроллера
; 0 - Бинарные показания (sensorsCount значений)
sensor:valueBin = 0, 0, 0, 0, 0
; 1 - Порты датчиков (sensorsCount значений)
sensor:pin = 4, 5, 6, 7, -1
; 2 - Количество измерений с момента включения
sensor:valueCount = 0
; 3 - Фильтр переключения бинарного значения [сек/10]
sensor:binFilter = 0
; 4 - Минимальное время между опросами показаний датчика [сек/10]
sensor:getTimeout = 1
; 5 - Инвертировать значение
sensor:invert = 0
; 6 - Всегда отключено
sensor:alwaysOff = 0
; 7 - Подтяжка портов (0=Вход;1=На землю;2=На плюс;3=АЦП)
sensor:pull = 0
; 8 - Выключать подтяжку порта датчика между опросами
sensor:pullOff = 0
; 9 - Порог АЦП для бинарного срабатывания (sensorsCount значений)
sensor:adc_from = 0, 0, 0, 0, 0
```

**Примечание:**
*   **Кол-во значений:** Параметры `valueBin`, `pin`, `adc_from` имеют `sensorsCount` значений (по одному на каждый датчик). Остальные параметры — по 1 значению.
*   **Единицы измерения:** `getTimeout`, `binFilter` — десятые доли секунды.
*   **Значения портов:** `-1` = порт не подключён, `-3` = ошибка инициализации порта.

## Конструкторы

Класс является шаблонным: `MWOSSensorBin<sensorsCount>`.

1.  **По умолчанию**
    ```cpp
    MWOSSensorBin();
    ```
    Создаёт модуль с параметрами по умолчанию. Все порты инициализируются значением -1 (не подключены).

2.  **Из массива в PROGMEM**
    ```cpp
    MWOSSensorBin(const uint8_t * pins);
    ```
    Создаёт модуль с портами из массива в памяти PROGMEM. Позволяет задать конфигурацию портов на этапе компиляции.

3.  **Одиночный пин**
    ```cpp
    MWOSSensorBin(MWOS_PIN_INT pin);
    ```
    Создаёт модуль с одним датчиком на указанном порту. Удобно для простых конфигураций с одним датчиком.

## Основные методы

### Чтение значений

*   `virtual bool readBoolValue(uint16_t arrayIndex)` *(переопределён)*
    *   Выполняет чтение состояния датчика для указанного канала.
    *   При включённой поддержке АЦП и заданном пороге `adc_from` сравнивает аналоговое значение с порогом.
    *   Иначе читает цифровое состояние порта (LOW/HIGH).
    *   **Возвращает:** `bool` — бинарное состояние датчика.

*   `bool getValueBool(int16_t arrayIndex = 0)`
    *   Запрашивает актуальное бинарное значение указанного датчика.
    *   Генерирует событие `EVENT_GET_VALUE` и возвращает полученное значение.
    *   **Возвращает:** `bool` — бинарное состояние датчика.

### Инициализация

*   `virtual void onEvent(MWOSModeEvent modeEvent)` *(переопределён)*
    *   **`EVENT_INIT` / `EVENT_CHANGE`:** Загрузка настроек из хранилища, инициализация портов.
    *   **`EVENT_UPDATE`:** Вызывает обработку базового класса для опроса датчиков.

*   `virtual void initSensor(int16_t index, bool pullOn)` *(переопределён)*
    *   Настраивает порт датчика в заданном режиме.
    *   При ошибке настройки записывает значение -3 в параметр `pin` и устанавливает флаг изменения.

### Управление состоянием

*   `void pause()` — Переводит все датчики модуля в режим паузы.
*   `void resume()` — Снимает датчики с паузы и восстанавливает нормальный режим опроса.
*   `bool isPaused()` — Возвращает текущее состояние паузы модуля.

## Пример использования

### Базовое подключение

```cpp
#include "MWOSSensorBin.h"

// Создаём тип для 5 бинарных датчиков
MWOSSensorBin<5> sensorBin;

// Массив пинов в PROGMEM
const uint8_t PROGMEM pins[] = {4, 5, 6, 7, 8};

// Экземпляр модуля с портами из PROGMEM
sensorBin(pins);

void setup() {
    // Настройки по умолчанию (загружаются только если хранилище не инициализировано)
    sensorBin._binFilter = 2;  // Фильтрация переключений 0.2 сек
    sensorBin._getTimeout = 5; // Опрос датчиков каждые 0.5 сек
    sensorBin._pull = 2;       // Подтяжка к питанию
    
    // Запуск системы - загружает настройки из хранилища
    mwos.start();
}

void loop() {
    // Все действия выполняются внутри модулей через обработчики событий
    mwos.update();
}
```

### Подключение с пороговым срабатыванием через АЦП

```cpp
#include "MWOSSensorBin.h"

// Создаём модуль с 3 датчиками
MWOSSensorBin<3> sensorBin;

void setup() {
    // Настройка порогов АЦП для каждого датчика
    // При значении АЦП >= порога датчик считается включённым (1)
    sensorBin._adc_from[0] = 500;  // Порог для первого датчика
    sensorBin._adc_from[1] = 800;  // Порог для второго датчика
    sensorBin._adc_from[2] = 0;    // Третий датчик работает в цифровом режиме
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
    
    // Чтение бинарного значения датчика
    bool state = sensorBin.getValueBool(0);
}
```

### Подключение одиночного датчика

```cpp
#include "MWOSSensorBin.h"

// Создаём модуль с одним датчиком на порту 4
MWOSSensorBin<1> sensorBin(4);

void setup() {
    // Настройка инверсии значения
    sensorBin._invert = 1;  // Инвертировать логическое состояние
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
}
```

## Способы настройки параметров

Значения параметров модулей можно задавать несколькими способами:

1.  **Значения по умолчанию в коде** (до `mwos.start()`)
    ```cpp
    void setup() {
        sensorBin._binFilter = 2;  // Фильтрация 0.2 сек
        sensorBin._pull = 2;       // Подтяжка к питанию
        mwos.start();
    }
    ```
    **Важно:** Настройки задаются **до** вызова `mwos.start()`. При первом запуске контроллера значения из хранилища не инициализированы, поэтому будут использованы заданные по умолчанию. При последующих запусках значения загружаются из хранилища, что позволяет изменять их удалённо.

2.  **Файл конфигурации `defaults.cfg`** (для контроллеров с файловой системой)
    ```
    ; 1 - Порты датчиков (5 значений)
    sensor:pin = 4, 5, 6, 7, 8
    ; 3 - Фильтр переключения бинарного значения [сек/10]
    sensor:binFilter = 2
    ; 7 - Подтяжка портов (0=Вход;1=На землю;2=На плюс;3=АЦП)
    sensor:pull = 2
    ```
    Файл размещается в корневой директории файловой системы контроллера.

3.  **Удалённая настройка через web-интерфейс или mwosApp**
    Параметры можно изменять в реальном времени через:
    *   Веб-интерфейс контроллера
    *   Кроссплатформенное приложение `mwosApp`

4.  **Команды через Serial-терминал**
    ```
    {sensor:pin = 4, 5, 6, 7, 8}
    {sensor:binFilter = 2}
    {sensor:pull = 2}
    ```
    Команды отправляются в текстовом виде через последовательный порт.

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `pin` | FOR_PIN | EEPROM или NO в зависимости от конфигурации |
| `adc_from` | EEPROM | Только если включена поддержка АЦП |
| `valueBin` | NO | Вычисляемое значение, не хранится |
| `binFilter`, `getTimeout` | EEPROM | Тайминги опроса датчиков |
| `invert`, `alwaysOff`, `pullOff` | EEPROM | Флаги режима работы |
| `pull` | EEPROM | Режим подтяжки портов |
| `valueCount` | NO | Счётчик измерений, не хранится |

## Важные замечания

1.  **Порядок инициализации:** Все настройки по умолчанию должны задаваться **до** вызова `mwos.start()`. После запуска системы значения загружаются из хранилища, и повторная установка значений в коде будет перезаписывать сохранённые настройки при каждой перезагрузке.

2.  **Поддержка АЦП:** Параметр `adc_from` доступен только если определён макрос `MWOS_SENSOR_BIN_ADC=1`. На платформах с ограниченной памятью (режим `MWOS_MINIMUM_RAM`) поддержка АЦП может быть отключена.

3.  **Фильтрация переключений:** Параметр `binFilter` задаёт время в десятых долях секунды. Кратковременные переключения, длящиеся меньше заданного времени, игнорируются. Это позволяет устранить дребезг контактов механических датчиков.

4.  **Подтяжка портов:** Для бинарных датчиков рекомендуется использовать режим подтяжки 1 (На землю) или 2 (На плюс) в зависимости от схемы подключения. Режим 3 (АЦП) используется только при пороговом срабатывании через аналоговый вход.

5.  **Неиспользуемые каналы:** Устанавливайте значение `-1` в параметре `pin` для неиспользуемых каналов. Это отключает опрос датчика на этом канале и экономит ресурсы системы.

6.  **Обработка ошибок портов:** При ошибке инициализации порта (значение -3 в параметре `pin`) модуль устанавливает флаг изменения параметра для уведомления системы. Проверьте подключение порта и его возможности.

7.  **Архитектура MWOS:** Все действия выполняются внутри модулей через обработчики событий. Пользовательский код в `loop()` должен содержать только вызов `mwos.update()`. Для расширения функциональности создавайте собственные модули, наследуясь от `MWOSModule`.

## См. также

*   **MWOSSensorBase**: Базовый класс для всех датчиков. Реализует общую логику опроса, фильтрации бинарных значений и управления состоянием датчиков.
*   **MWOSSensorAnalog**: Базовый класс для аналоговых датчиков. Добавляет поддержку пороговых значений и фильтрации аналоговых сигналов.
*   **MWOSSensorADC**: Модуль для подключения аналоговых датчиков к портам АЦП микроконтроллера.
*   **Библиотека MWOSPins**: Система управления портами микроконтроллера с поддержкой расширителей.
