# MWOSSensorDHT

**Имя модуля:** `dht`

**MWOSSensorDHT** — модуль для подключения датчиков температуры и влажности серии DHT. Поддерживает множественные датчики и автоматическое чтение показаний.

## Описание

Класс предназначен для:
*   **Чтение температуры и влажности:** Работа с датчиками DHT11, DHT22, DHT33, DHT44, AM2301, AM2302, AM2303
*   **Два параметра value:** Параметр 0 — температура (°C × 10), параметр 1 — влажность (% × 10)
*   **Автоматическая инициализация:** Автоматическое создание объекта библиотеки DHTNew при инициализации порта
*   **Обработка ошибок:** Коды ошибок при неудачном чтении датчика

**Наследование:** `MWOSSensorDHT<MWOSSensorAnalogType>` → `MWOSSensorAnalogType` → `MWOSSensorBase<sensorsCount>` → `MWOSModule`

**Тип модуля:** `MODULE_SENSOR`

**Зависимости:**
*   Библиотека `DHTNew` (https://github.com/RobTillaart/DHTNew.git)
*   Библиотека `MWOSSensorAnalog.h` (базовый класс)

## Основные возможности

*   **Поддержка множественных датчиков DHT:** До 2 датчиков (температура и влажность) в одном модуле
*   **Автоматическое чтение показаний:** Чтение температуры и влажности при каждом опросе датчика
*   **Обработка ошибок чтения:** Коды ошибок 98 (ошибка чтения DHT) и 99 (датчик не инициализирован)
*   **Гибкое наследование:** Возможность указания MWOSSensorFilter вместо MWOSSensorAnalog для расширенной фильтрации
*   **Интеграция с системой MWOS:** Полная интеграция с параметрами, хранилищами и событиями MWOS

## Параметры конфигурации

Класс использует параметры базовых классов и добавляет свои специфические параметры:

| ID | Название | Тип | Группа | Хранилище | Кол-во | Описание |
|:--:|:---|:---|:---|:---|:---:|:---|
| 0 | `valueBin` | bits1 | readonly | NO | 2 | **Бинарные показания.** Пороговое значение на основе fromValue/toValue |
| 1 | `pin` | int16 | pin | FOR_PIN | 1 | **Порт датчика.** Номер порта для подключения датчика DHT. -1 = не подключён |
| 2 | `valueCount` | uint32 | option | NO | 1 | **Количество измерений.** Счётчик запросов показаний с момента включения |
| 3 | `binFilter` | uint8 | option | EEPROM | 1 | **Фильтр переключения.** Время фильтрации бинарных переключений [сек/10] |
| 4 | `getTimeout` | uint16 | option | EEPROM | 1 | **Таймаут опроса.** Минимальное время между опросами датчиков [сек/10] |
| 5 | `invert` | bits1 | option | EEPROM | 1 | **Инверсия.** Инвертировать бинарное значение датчика |
| 6 | `alwaysOff` | bits1 | option | EEPROM | 1 | **Всегда отключено.** Принудительное отключение опроса датчиков |
| 7 | `pull` | bits3 | option | EEPROM | 1 | **Подтяжка портов.** Режим подтяжки (0=Вход;1=На землю;2=На плюс;3=АЦП) |
| 8 | `pullOff` | bits1 | option | EEPROM | 1 | **Отключение подтяжки.** Выключать подтяжку между опросами |
| 9 | `value` | int32 | readonly | NO | 2 | **Аналоговые показания.** Температура (индекс 0) и влажность (индекс 1) |
| 10 | `fromValue` | int32 | option | EEPROM | 1 | **Нижний порог.** Значение для интерпретации бинарной единицы |
| 11 | `toValue` | int32 | option | EEPROM | 1 | **Верхний порог.** Значение для интерпретации бинарной единицы |
| 12 | `analogSend` | bits1 | option | EEPROM | 1 | **Отправка аналоговых.** 0=только при изменении бинарного, 1=при изменении аналогового |
| 14 | `sensorDigital` | bits1 | option | EEPROM | 1 | **Цифровой режим.** При 1 работает как бинарный датчик |
| 15 | `analogFilterMin` | uint16 | option | EEPROM | 1 | **Минимальное изменение.** Игнорировать изменения меньше этого значения |
| 16 | `analogFilterMax` | uint16 | option | EEPROM | 1 | **Максимальное изменение.** Считать сбоем изменения больше этого значения |
| 17 | `error` | int8 | readonly | NO | 1 | **Код ошибки.** 98=ошибка чтения DHT, 99=датчик не инициализирован |
| 18 | `average` | bits1 | option | EEPROM | 1 | **Усреднение.** Вычислять среднее арифметическое с предыдущим значением |

**Примечание:**
*   **Кол-во:** Параметр `value` имеет 2 значения (температура и влажность). Остальные параметры — по 1 значению.
*   **Единицы измерения:** Температура возвращается в градусах Цельсия × 10, влажность в процентах × 10.
*   **Параметр 1 (pin):** Единственный порт для датчика DHT. Оба значения (температура и влажность) читаются с одного датчика.

## Конфигурация модуля

```
; модуль dht - датчики температуры и влажности DHT
; 0 - Бинарные показания (2 значения: температура, влажность)
dht:valueBin = 0, 0
; 1 - Порт датчика
dht:pin = 4
; 2 - Количество измерений с момента включения
dht:valueCount = 0
; 3 - Фильтр переключения бинарного значения [сек/10]
dht:binFilter = 0
; 4 - Минимальное время между опросами показаний датчика [сек/10]
dht:getTimeout = 10
; 5 - Инвертировать значение
dht:invert = 0
; 6 - Всегда отключено
dht:alwaysOff = 0
; 7 - Подтяжка портов (0=Вход;1=На землю;2=На плюс;3=АЦП)
dht:pull = 2
; 8 - Выключать подтяжку порта датчика между опросами
dht:pullOff = 0
; 9 - Последние показания (2 значения: температура ×10, влажность ×10)
dht:value = 250, 600
; 10 - Нижний порог для valueBin (температура)
dht:fromValue = 0
; 11 - Верхний порог для valueBin (температура)
dht:toValue = 500
; 12 - Отправлять аналоговые значения (0=только бинарное, 1=аналоговое)
dht:analogSend = 1
; 14 - Режим цифрового датчика (0=аналоговый, 1=цифровой)
dht:sensorDigital = 0
; 15 - Минимальное изменение (фильтр)
dht:analogFilterMin = 0
; 16 - Максимальное изменение (сбой)
dht:analogFilterMax = 0
; 17 - Код ошибки
dht:error = 0
; 18 - Среднее арифметическое с предыдущим значением
dht:average = 0
```

**Примечание:**
*   **Кол-во значений:** Параметр `valueBin` и `value` имеют 2 значения (температура и влажность). Остальные параметры — по 1 значению.
*   **Единицы измерения:** `getTimeout`, `binFilter` — десятые доли секунды. Температура и влажность умножаются на 10 для хранения в целочисленном формате.
*   **Таймаут опроса:** Рекомендуется устанавливать не менее 10 (1 секунда) для датчиков DHT, так как они имеют ограниченную частоту опроса.

## Конструкторы

Класс является шаблонным: `MWOSSensorDHT<MWOSSensorAnalogType>`.

1.  **По умолчанию**
    ```cpp
    MWOSSensorDHT();
    ```
    Создаёт модуль с параметрами по умолчанию. Порт инициализируется значением -1 (не подключён).

2.  **С указанием порта**
    ```cpp
    MWOSSensorDHT(MWOS_PIN_INT pin);
    ```
    Создаёт модуль с датчиком на указанном порту. Удобно для простых конфигураций с одним датчиком.

3.  **С расширенной фильтрацией**
    ```cpp
    MWOSSensorDHT<MWOSSensorFilter<2>> sensorDHT;
    ```
    Создаёт модуль с поддержкой фильтрации Калмана и коррекции показаний коэффициентами.

## Основные методы

### Чтение значений

*   `virtual int32_t readAnalogValue(int16_t arrayIndex)` *(переопределён)*
    *   Выполняет чтение показаний датчика DHT.
    *   Для индекса 0 читается температура, для индекса 1 читается влажность.
    *   При ошибке чтения устанавливает код ошибки 98.
    *   **Возвращает:** `int32_t` — значение температуры (°C × 10) или влажности (% × 10).

*   `int64_t getValueAnalog(int16_t arrayIndex = 0)` *(унаследован)*
    *   Запрашивает аналоговое значение указанного датчика.
    *   **Возвращает:** `int64_t` — значение температуры (индекс 0) или влажности (индекс 1).

*   `float getValuePercent(uint16_t arrayIndex)` *(унаследован)*
    *   Вычисляет процент текущего значения в диапазоне от fromValue до toValue.
    *   **Возвращает:** `float` — процентное значение.

### Инициализация

*   `virtual void onEvent(MWOSModeEvent modeEvent)` *(переопределён)*
    *   **`EVENT_INIT` / `EVENT_CHANGE`:** Загрузка настроек из хранилища, инициализация объекта DHTNew при изменении порта.
    *   **`EVENT_UPDATE`:** Вызывает обработку базового класса для опроса датчиков.

*   `void setError(uint8_t errorCode)` *(унаследован)*
    *   Устанавливает код ошибки для модуля.
    *   Коды: 98 = ошибка чтения DHT, 99 = датчик не инициализирован.

## Пример использования

### Базовое подключение

```cpp
#include "MWOSSensorDHT.h"

// Создаём тип для 2 датчиков (температура и влажность)
MWOSSensorDHT<> sensorDHT;

// Экземпляр модуля с портом 4
sensorDHT(4);

void setup() {
    // Настройки по умолчанию (загружаются только если хранилище не инициализировано)
    sensorDHT._getTimeout = 10;  // Опрос датчика каждые 1 секунду
    sensorDHT._fromValue = 0;    // Нижний порог для температуры
    sensorDHT._toValue = 500;    // Верхний порог для температуры (50°C)
    
    // Запуск системы - загружает настройки из хранилища
    mwos.start();
}

void loop() {
    // Все действия выполняются внутри модулей через обработчики событий
    mwos.update();
}
```

### Подключение с фильтрацией Калмана

```cpp
#include "MWOSSensorDHT.h"

// Создаём модуль с фильтрацией
MWOSSensorDHT<MWOSSensorFilter<2>> sensorDHT;

void setup() {
    // Настройка коэффициентов коррекции (загружаются только если хранилище не инициализировано)
    sensorDHT._coef[0] = 1.0;    // Температура
    sensorDHT._coef[1] = 1.0;    // Влажность
    sensorDHT._subCoef[0] = 0.0;
    sensorDHT._subCoef[1] = 0.0;
    
    // Включение фильтра Калмана
    sensorDHT._filter[0] = EFM_Filter2;
    sensorDHT._filter[1] = EFM_Filter2;
    
    // Настройка параметров фильтра
    sensorDHT.sensorFilter[0]->Q = 0.01;
    sensorDHT.sensorFilter[0]->R = 0.1;
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
    
    // Чтение фильтрованного значения температуры
    double temp = sensorDHT.getFilteredValue(0);
    double hum = sensorDHT.getFilteredValue(1);
}
```

### Чтение показаний датчика

```cpp
#include "MWOSSensorDHT.h"

// Создаём модуль с одним датчиком на порту 4
MWOSSensorDHT<> sensorDHT(4);

void setup() {
    // Настройка таймаута опроса (минимум 1 секунда для DHT)
    sensorDHT._getTimeout = 10;
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
    
    // Чтение температуры (°C × 10)
    int32_t temp = sensorDHT.getValueAnalog(0);
    
    // Чтение влажности (% × 10)
    int32_t hum = sensorDHT.getValueAnalog(1);
    
    // Преобразование в реальные значения
    float temperature = temp / 10.0;
    float humidity = hum / 10.0;
}
```

## Способы настройки параметров

Значения параметров модулей можно задавать несколькими способами:

1.  **Значения по умолчанию в коде** (до `mwos.start()`)
    ```cpp
    void setup() {
        sensorDHT._getTimeout = 10;  // Опрос каждые 1 секунду
        sensorDHT._fromValue = 0;    // Нижний порог
        sensorDHT._toValue = 500;    // Верхний порог (50°C)
        mwos.start();
    }
    ```
    **Важно:** Настройки задаются **до** вызова `mwos.start()`. При первом запуске контроллера значения из хранилища не инициализированы, поэтому будут использованы заданные по умолчанию. При последующих запусках значения загружаются из хранилища, что позволяет изменять их удалённо.

2.  **Файл конфигурации `defaults.cfg`** (для контроллеров с файловой системой)
    ```
    ; 1 - Порт датчика
    dht:pin = 4
    ; 4 - Минимальное время между опросами [сек/10]
    dht:getTimeout = 10
    ; 10 - Нижний порог для температуры
    dht:fromValue = 0
    ; 11 - Верхний порог для температуры
    dht:toValue = 500
    ```
    Файл размещается в корневой директории файловой системы контроллера.

3.  **Удалённая настройка через web-интерфейс или mwosApp**
    Параметры можно изменять в реальном времени через:
    *   Веб-интерфейс контроллера
    *   Кроссплатформенное приложение `mwosApp`

4.  **Команды через Serial-терминал**
    ```
    {dht:pin = 4}
    {dht:getTimeout = 10}
    {dht:fromValue = 0}
    {dht:toValue = 500}
    ```
    Команды отправляются в текстовом виде через последовательный порт.

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `pin` | FOR_PIN | EEPROM или NO в зависимости от конфигурации |
| `valueBin` | NO | Вычисляемое значение, не хранится |
| `value` | NO | Последние показания датчика, не хранится |
| `fromValue`, `toValue` | EEPROM | Пороговые значения для бинарной интерпретации |
| `analogFilterMin`, `analogFilterMax` | EEPROM | Параметры фильтрации аналоговых значений |
| `binFilter`, `getTimeout` | EEPROM | Тайминги опроса датчиков |
| `invert`, `alwaysOff`, `pullOff` | EEPROM | Флаги режима работы |
| `pull` | EEPROM | Режим подтяжки портов |
| `analogSend`, `sensorDigital`, `average` | EEPROM | Режимы отправки и обработки данных |
| `error` | NO | Код ошибки, не хранится |
| `valueCount` | NO | Счётчик измерений, не хранится |

## Важные замечания

1.  **Порядок инициализации:** Все настройки по умолчанию должны задаваться **до** вызова `mwos.start()`. После запуска системы значения загружаются из хранилища, и повторная установка значений в коде будет перезаписывать сохранённые настройки при каждой перезагрузке.

2.  **Таймаут опроса:** Датчики DHT имеют ограниченную частоту опроса (не чаще 1 раза в секунду). Устанавливайте `getTimeout` не менее 10 (1 секунда) для избежания ошибок чтения.

3.  **Единицы измерения:** Температура возвращается в градусах Цельсия × 10, влажность в процентах × 10. Для получения реальных значений разделите на 10.0.

4.  **Использование с MWOSSensorFilter:** При указании `MWOSSensorFilter` вместо `MWOSSensorAnalog` модуль получает дополнительные параметры для фильтрации Калмана (ID 30-40). Это увеличивает потребление памяти, но улучшает стабильность показаний.

5.  **Подтяжка портов:** Для датчиков DHT рекомендуется использовать режим подтяжки 2 (На плюс) или 0 (Вход без подтяжки), так как датчики требуют подтяжки к питанию для корректной работы.

6.  **Коды ошибок:** 98 = ошибка чтения датчика (неудачная попытка чтения), 99 = датчик не инициализирован (порт не задан или объект DHTNew не создан).

7.  **Архитектура MWOS:** Все действия выполняются внутри модулей через обработчики событий. Пользовательский код в `loop()` должен содержать только вызов `mwos.update()`. Для расширения функциональности создавайте собственные модули, наследуясь от `MWOSModule`.

8.  **Поддерживаемые датчики:** DHT11, DHT22, DHT33, DHT44, AM2301, AM2302, AM2303. Автоматическое определение типа датчика выполняется библиотекой DHTNew.

9.  **Два значения value:** Модуль всегда создаётся с 2 значениями (индекс 0 = температура, индекс 1 = влажность). Оба значения читаются с одного датчика при каждом опросе.

## См. также

*   **MWOSSensorBase**: Базовый класс для всех датчиков. Реализует общую логику опроса и фильтрации бинарных значений.
*   **MWOSSensorAnalog**: Базовый класс для аналоговых датчиков. Добавляет поддержку пороговых значений и фильтрации аналоговых сигналов.
*   **MWOSSensorFilter**: Расширение для фильтрации Калмана и коррекции показаний коэффициентами.
*   **Библиотека DHTNew**: https://github.com/RobTillaart/DHTNew.git — библиотека для работы с датчиками DHT.
