# MWOSSensorADC

**Имя модуля:** `adc`

**MWOSSensorADC** — модуль для подключения аналоговых датчиков к портам АЦП микроконтроллера. Поддерживает множественные датчики, усреднение замеров и настройку затухания для ESP32.

## Описание

Класс предназначен для:
*   **Чтение аналоговых сигналов:** Подключение потенциометров, фоторезисторов, терморезисторов и других аналоговых сенсоров
*   **Усреднение замеров:** Выполнение серии замеров с усреднением результата для снижения шума измерений
*   **Настройка АЦП (ESP32):** Выбор диапазона измеряемых напряжений через настройку затухания сигнала
*   **Множественные датчики:** Поддержка нескольких независимых аналоговых каналов в одном модуле

**Наследование:** `MWOSSensorADC<sensorsCount, MWOSSensorAnalogType>` → `MWOSSensorAnalogType` → `MWOSSensorBase<sensorsCount>` → `MWOSModule`

**Тип модуля:** `MODULE_SENSOR`

**Зависимости:**
*   Библиотека `MWOSSensorAnalog.h` (базовый класс)
*   Библиотека `MWOSPins.h` (работа с портами)
*   Платформа ESP32 (опционально, для параметра `attenuation`)

## Основные возможности

*   **Множественные каналы АЦП:** Поддержка до `sensorsCount` независимых аналоговых каналов
*   **Усреднение замеров:** Выполнение серии замеров с усреднением результата для повышения точности измерений
*   **Настройка затухания (ESP32):** Выбор диапазона измеряемых напряжений через настройку затухания АЦП
*   **Гибкое наследование:** Возможность указания `MWOSSensorFilter` вместо `MWOSSensorAnalog` для расширенной фильтрации
*   **Автоматическая инициализация портов:** Настройка портов в режим АЦП при инициализации модуля
*   **Поддержка расширителей портов:** Работа через систему портов MWOS с поддержкой расширителей

## Параметры конфигурации

Класс использует параметры базовых классов и добавляет свои специфические параметры:

| ID | Название | Тип | Группа | Хранилище | Кол-во | Описание |
|:--:|:---|:---|:---|:---|:---:|:---|
| 0 | `valueBin` | bits1 | readonly | NO | sensorsCount | **Бинарные показания.** Пороговое значение на основе `fromValue`/`toValue` |
| 1 | `pin` | int16 | pin | FOR_PIN | sensorsCount | **Порты АЦП.** Номера портов для каждого датчика. -1 = не подключён |
| 2 | `valueCount` | uint32 | option | NO | 1 | **Количество измерений.** Счётчик запросов показаний с момента включения |
| 3 | `binFilter` | uint8 | option | EEPROM | 1 | **Фильтр переключения.** Время фильтрации бинарных переключений [сек/10] |
| 4 | `getTimeout` | uint16 | option | EEPROM | 1 | **Таймаут опроса.** Минимальное время между опросами датчиков [сек/10] |
| 5 | `invert` | bits1 | option | EEPROM | 1 | **Инверсия.** Инвертировать бинарное значение датчика |
| 6 | `alwaysOff` | bits1 | option | EEPROM | 1 | **Всегда отключено.** Принудительное отключение опроса датчиков |
| 7 | `pull` | bits3 | option | EEPROM | 1 | **Подтяжка портов.** Режим подтяжки (0=Вход;1=На землю;2=На плюс;3=АЦП) |
| 8 | `pullOff` | bits1 | option | EEPROM | 1 | **Отключение подтяжки.** Выключать подтяжку между опросами |
| 9 | `value` | int32 | readonly | NO | sensorsCount | **Аналоговые показания.** Последние значения АЦП |
| 10 | `fromValue` | int32 | option | EEPROM | 1 | **Нижний порог.** Значение для интерпретации бинарной единицы |
| 11 | `toValue` | int32 | option | EEPROM | 1 | **Верхний порог.** Значение для интерпретации бинарной единицы |
| 12 | `analogSend` | bits1 | option | EEPROM | 1 | **Отправка аналоговых.** 0=только при изменении бинарного, 1=при изменении аналогового |
| 14 | `sensorDigital` | bits1 | option | EEPROM | 1 | **Цифровой режим.** При 1 работает как бинарный датчик |
| 15 | `analogFilterMin` | uint16 | option | EEPROM | 1 | **Минимальное изменение.** Игнорировать изменения меньше этого значения |
| 16 | `analogFilterMax` | uint16 | option | EEPROM | 1 | **Максимальное изменение.** Считать сбоем изменения больше этого значения |
| 17 | `error` | int8 | readonly | NO | 1 | **Код ошибки.** Код последней ошибки чтения датчика |
| 18 | `average` | bits1 | option | EEPROM | 1 | **Усреднение.** Вычислять среднее арифметическое с предыдущим значением |
| 20 | `attenuation` | bits2 | option | EEPROM | sensorsCount | **Затухание АЦП.** Только ESP32. Диапазон напряжений (ADC_DB_0;ADC_DB_2_5;ADC_DB_6;ADC_DB_12) |
| 21 | `mean` | uint8 | option | EEPROM | 1 | **Количество замеров.** Для усреднения до среднего арифметического |

**Примечание:**
*   **Кол-во:** Параметры с `sensorsCount` имеют индивидуальное значение для каждого датчика. Остальные параметры — по 1 значению.
*   **Параметр 20 (attenuation):** Доступен только на платформе ESP32. На других платформах игнорируется.
*   **Параметр 21 (mean):** При значении 0 или 1 выполняется одиночный замер. При значении 2 и более — серия замеров с усреднением.

## Конфигурация модуля

```
; модуль adc - аналоговые датчики АЦП микроконтроллера
; 0 - Бинарные показания (sensorsCount значений)
adc:valueBin = 0, 0, 0, 0, 0
; 1 - Порты АЦП (sensorsCount значений)
adc:pin = 34, 35, 36, 39, -1
; 2 - Количество измерений с момента включения
adc:valueCount = 0
; 3 - Фильтр переключения бинарного значения [сек/10]
adc:binFilter = 0
; 4 - Минимальное время между опросами показаний датчика [сек/10]
adc:getTimeout = 1
; 5 - Инвертировать значение
adc:invert = 0
; 6 - Всегда отключено
adc:alwaysOff = 0
; 7 - Подтяжка портов (0=Вход;1=На землю;2=На плюс;3=АЦП)
adc:pull = 0
; 8 - Выключать подтяжку порта датчика между опросами
adc:pullOff = 0
; 9 - Последние аналоговые показания (sensorsCount значений)
adc:value = 0, 0, 0, 0, 0
; 10 - Нижний порог для valueBin
adc:fromValue = 0
; 11 - Верхний порог для valueBin
adc:toValue = 4095
; 12 - Отправлять аналоговые значения (0=только бинарное, 1=аналоговое)
adc:analogSend = 0
; 14 - Режим цифрового датчика (0=аналоговый, 1=цифровой)
adc:sensorDigital = 0
; 15 - Минимальное изменение (фильтр)
adc:analogFilterMin = 0
; 16 - Максимальное изменение (сбой)
adc:analogFilterMax = 0
; 17 - Код ошибки
adc:error = 0
; 18 - Среднее арифметическое с предыдущим значением
adc:average = 0
; 20 - Затухание АЦП (ESP32, sensorsCount значений)
adc:attenuation = 2, 2, 2, 2, 2
; 21 - Количество замеров для усреднения
adc:mean = 1
```

**Примечание:**
*   **Кол-во значений:** Параметры `valueBin`, `pin`, `value`, `attenuation` имеют `sensorsCount` значений (по одному на каждый датчик). Остальные параметры — по 1 значению.
*   **Единицы измерения:** `getTimeout`, `binFilter` — десятые доли секунды. `attenuation` — индекс из списка (0=ADC_DB_0, 1=ADC_DB_2_5, 2=ADC_DB_6, 3=ADC_DB_12).
*   **Диапазон АЦП:** Зависит от платформы. ESP32: 0-4095 (12 бит), ESP8266: 0-1024 (10 бит).

## Конструкторы

Класс является шаблонным: `MWOSSensorADC<sensorsCount, MWOSSensorAnalogType>`.

1.  **По умолчанию**
    ```cpp
    MWOSSensorADC();
    ```
    Создаёт модуль с параметрами по умолчанию. Все порты инициализируются значением -1 (не подключены).

2.  **Из массива в PROGMEM**
    ```cpp
    MWOSSensorADC(const uint8_t * pins);
    ```
    Создаёт модуль с портами из массива в памяти PROGMEM. Позволяет задать конфигурацию портов на этапе компиляции.

3.  **Одиночный пин**
    ```cpp
    MWOSSensorADC(MWOS_PIN_INT pin);
    ```
    Создаёт модуль с одним датчиком на указанном порту АЦП. Удобно для простых конфигураций с одним датчиком.

4.  **С расширенной фильтрацией**
    ```cpp
    MWOSSensorADC<sensorsCount, MWOSSensorFilter<sensorsCount>> sensorADC;
    ```
    Создаёт модуль с поддержкой фильтрации Калмана и коррекции показаний коэффициентами.

## Основные методы

### Чтение значений

*   `virtual int32_t readAnalogValue(int16_t arrayIndex)` *(переопределён)*
    *   Выполняет чтение значения АЦП для указанного канала.
    *   При `mean >= 2` выполняет серию замеров с усреднением результата.
    *   **Возвращает:** `int32_t` — значение АЦП (0-4095 для ESP32, 0-1024 для ESP8266).

*   `virtual int32_t readDigitalValue(int16_t arrayIndex)` *(переопределён)*
    *   Выполняет чтение цифрового состояния порта АЦП.
    *   **Возвращает:** `int32_t` — 0 или 1.

### Инициализация

*   `virtual void onEvent(MWOSModeEvent modeEvent)` *(переопределён)*
    *   **`EVENT_INIT` / `EVENT_CHANGE`:** Загрузка настроек из хранилища, инициализация портов, настройка затухания (ESP32).
    *   **`EVENT_UPDATE`:** Вызывает обработку базового класса для опроса датчиков.

*   `virtual void initSensor(int16_t index, bool pullOn)` *(переопределён)*
    *   Настраивает порт АЦП в заданном режиме.
    *   Для ESP32 устанавливает затухание АЦП из параметра `attenuation`.
    *   При ошибке настройки записывает значение -3 в параметр `pin`.

## Пример использования

### Базовое подключение

```cpp
#include "MWOSSensorADC.h"

// Создаём тип для 5 датчиков
MWOSSensorADC<5> sensorADC;

// Массив пинов в PROGMEM
const uint8_t PROGMEM pins[] = {34, 35, 36, 39, 40};

// Экземпляр модуля с портами из PROGMEM
sensorADC(pins);

void setup() {
    // Настройки по умолчанию (загружаются только если хранилище не инициализировано)
    sensorADC._mean = 5;  // Усреднение 5 замеров
    sensorADC._fromValue = 500;  // Нижний порог для бинарного значения
    sensorADC._toValue = 3500;   // Верхний порог для бинарного значения
    
    // Запуск системы - загружает настройки из хранилища
    mwos.start();
}

void loop() {
    // Все действия выполняются внутри модулей через обработчики событий
    mwos.update();
}
```

### Настройка затухания для ESP32

```cpp
#include "MWOSSensorADC.h"

#ifdef ESP32
MWOSSensorADC<2> sensorADC;

void setup() {
    // Настройка затухания для разных диапазонов
    // 0 = ADC_DB_0 (±6.144V)
    // 1 = ADC_DB_2_5 (±4.096V)
    // 2 = ADC_DB_6 (±2.048V)
    // 3 = ADC_DB_12 (±1.024V)
    sensorADC._attenuation[0] = 2; // ±2.048V для первого канала
    sensorADC._attenuation[1] = 3; // ±1.024V для второго канала
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
}
#endif
```

### Подключение с фильтрацией Калмана

```cpp
#include "MWOSSensorADC.h"

// Создаём модуль с фильтрацией
MWOSSensorADC<4, MWOSSensorFilter<4>> sensorADC;

void setup() {
    // Настройка коэффициентов коррекции (загружаются только если хранилище не инициализировано)
    sensorADC._coef[0] = 1.5;
    sensorADC._subCoef[0] = 10.0;
    
    // Включение фильтра Калмана
    sensorADC._filter[0] = EFM_Filter2;
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
}
```

## Способы настройки параметров

Значения параметров модулей можно задавать несколькими способами:

1.  **Значения по умолчанию в коде** (до `mwos.start()`)
    ```cpp
    void setup() {
        sensorADC._attenuation[0] = 2; // ±2.048V
        sensorADC._attenuation[1] = 3; // ±1.024V
        mwos.start();
    }
    ```
    **Важно:** Настройки задаются **до** вызова `mwos.start()`. При первом запуске контроллера значения из хранилища не инициализированы, поэтому будут использованы заданные по умолчанию. При последующих запусках значения загружаются из хранилища, что позволяет изменять их удалённо.

2.  **Файл конфигурации `defaults.cfg`** (для контроллеров с файловой системой)
    ```
    ; 20 - Затухание АЦП (ESP32, sensorsCount значений)
    adc:attenuation = 2, 2, 2, 2, 2
    ; 21 - Количество замеров для усреднения
    adc:mean = 5
    ```
    Файл размещается в корневой директории файловой системы контроллера.

3.  **Удалённая настройка через web-интерфейс или mwosApp**
    Параметры можно изменять в реальном времени через:
    *   Веб-интерфейс контроллера
    *   Кроссплатформенное приложение `mwosApp`

4.  **Команды через Serial-терминал**
    ```
    {adc:attenuation = 2, 2, 2, 2, 2}
    {adc:mean = 5}
    ```
    Команды отправляются в текстовом виде через последовательный порт.

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `pin` | FOR_PIN | EEPROM или NO в зависимости от конфигурации |
| `attenuation` | EEPROM | Только ESP32, настройка затухания АЦП |
| `mean` | EEPROM | Количество замеров для усреднения |
| `valueBin` | NO | Вычисляемое значение, не хранится |
| `value` | NO | Последние показания АЦП, не хранится |
| `fromValue`, `toValue` | EEPROM | Пороговые значения для бинарной интерпретации |
| `analogFilterMin`, `analogFilterMax` | EEPROM | Параметры фильтрации аналоговых значений |
| `binFilter`, `getTimeout` | EEPROM | Тайминги опроса датчиков |
| `invert`, `alwaysOff`, `pullOff` | EEPROM | Флаги режима работы |
| `pull` | EEPROM | Режим подтяжки портов |
| `analogSend`, `sensorDigital`, `average` | EEPROM | Режимы отправки и обработки данных |
| `error` | NO | Код ошибки, не хранится |
| `valueCount` | NO | Счётчик измерений, не хранится |

## Важные замечания

1.  **Порядок инициализации:** Все настройки по умолчанию должны задаваться **до** вызова `mwos.start()`. После запуска системы значения загружаются из хранилища, и повторная установка значений в коде будет перезаписывать сохранённые настройки при каждой перезагрузке.

2.  **Диапазон АЦП зависит от платформы:** ESP32 использует 12-битное АЦП (0-4095), ESP8266 — 10-битное (0-1024). Учитывайте это при настройке пороговых значений `fromValue` и `toValue`.

3.  **Параметр attenuation только для ESP32:** На других платформах этот параметр игнорируется. Для ESP32 выбирайте затухание в зависимости от ожидаемого диапазона напряжений.

4.  **Усреднение увеличивает время опроса:** При `mean >= 2` каждый замер выполняется в цикле с задержкой 1 мс между замерами. Общее время опроса = `mean × 1 мс`.

5.  **Использование с MWOSSensorFilter:** При указании `MWOSSensorFilter` вместо `MWOSSensorAnalog` модуль получает дополнительные параметры для фильтрации Калмана (ID 30-40). Это увеличивает потребление памяти.

6.  **Подтяжка портов:** Для аналоговых датчиков рекомендуется использовать режим подтяжки 3 (АЦП) или 0 (Вход без подтяжки). Режимы 1 и 2 могут исказить показания.

7.  **Неиспользуемые каналы:** Устанавливайте значение `-1` в параметре `pin` для неиспользуемых каналов. Это отключает опрос датчика на этом канале.

8.  **Архитектура MWOS:** Все действия выполняются внутри модулей через обработчики событий. Пользовательский код в `loop()` должен содержать только вызов `mwos.update()`. Для расширения функциональности создавайте собственные модули, наследуясь от `MWOSModule`.

## См. также

*   **MWOSSensorBase**: Базовый класс для всех датчиков. Реализует общую логику опроса и фильтрации бинарных значений.
*   **MWOSSensorAnalog**: Базовый класс для аналоговых датчиков. Добавляет поддержку пороговых значений и фильтрации аналоговых сигналов.
*   **MWOSSensorFilter**: Расширение для фильтрации Калмана и коррекции показаний коэффициентами.
*   **MWOSSensorADS1115**: Внешний 16-битный АЦП ADS1115 через I2C для повышенной точности измерений.
*   **Библиотека MWOSPins**: Система управления портами микроконтроллера с поддержкой расширителей.
