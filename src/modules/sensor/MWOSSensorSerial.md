# MWOSSensorSerial

**Имя модуля:** `sensorSerial`

**MWOSSensorSerial** — универсальный модуль для получения потоковых данных от датчиков через последовательный порт UART. Поддерживает чтение текстовых данных с различных потоковых датчиков.

## Описание

Класс предназначен для:
*   **Чтение потоковых данных:** Получение показаний от датчиков с текстовым протоколом обмена через последовательный порт
*   **Парсинг текстовых данных:** Автоматическое распознавание буквенных идентификаторов параметров и извлечение числовых значений
*   **Гибкая настройка скорости:** Поддержка различных скоростей последовательного порта (от 1200 до 115200 бод)
*   **Множественные параметры:** Одновременное чтение нескольких параметров от одного датчика

**Наследование:** `MWOSSensorSerial<sensorsCount, MWOSSensorAnalogType>` → `MWOSSensorAnalogType` → `MWOSSensorBase<sensorsCount>` → `MWOSModule`

**Тип модуля:** `MODULE_SENSOR`

**Зависимости:**
*   Библиотека `MWOSSensorAnalog.h` (базовый класс)
*   Библиотека `Stream` (последовательный порт)
*   Платформа с поддержкой UART (ESP32/ESP8266/STM32/Arduino)

## Основные возможности

*   **Универсальный парсинг:** Поддержка любых датчиков с текстовым протоколом (буква + число)
*   **Настраиваемые идентификаторы:** Возможность задания буквенных идентификаторов для каждого параметра
*   **Автоматическая настройка скорости:** Конфигурация скорости последовательного порта через параметры
*   **Режим запросов:** Отправка команд перевода датчика в режим запросов при инициализации
*   **Гибкое наследование:** Возможность указания `MWOSSensorFilter` вместо `MWOSSensorAnalog` для расширенной фильтрации
*   **Обработка ошибок:** Автоматическое сохранение кодов ошибок при чтении данных

## Параметры конфигурации

Класс использует параметры базовых классов и добавляет свои специфические параметры:

| ID | Название | Тип | Группа | Хранилище | Кол-во | Описание |
|:--:|:---|:---|:---|:---|:---:|:---|
| 0 | `valueBin` | bits1 | readonly | NO | sensorsCount | **Бинарные показания.** Пороговое значение на основе fromValue/toValue |
| 1 | `pin` | int16 | pin+readonly | NO | MWOSSensorStreamPinsCount | **Порты RX/TX.** Номера портов для подключения датчика. -1 = не подключён |
| 2 | `valueCount` | uint32 | option | NO | 1 | **Количество измерений.** Счётчик запросов показаний с момента включения |
| 3 | `binFilter` | uint8 | option | EEPROM | 1 | **Фильтр переключения.** Время фильтрации бинарных переключений [сек/10] |
| 4 | `getTimeout` | uint16 | option | EEPROM | 1 | **Таймаут опроса.** Минимальное время между опросами датчиков [сек/10] |
| 5 | `invert` | bits1 | option | EEPROM | 1 | **Инверсия.** Инвертировать бинарное значение датчика |
| 6 | `alwaysOff` | bits1 | option | EEPROM | 1 | **Всегда отключено.** Принудительное отключение опроса датчиков |
| 7 | `pull` | bits3 | option | EEPROM | 1 | **Подтяжка портов.** Режим подтяжки (0=Вход;1=На землю;2=На плюс;3=АЦП) |
| 8 | `pullOff` | bits1 | option | EEPROM | 1 | **Отключение подтяжки.** Выключать подтяжку между опросами |
| 9 | `value` | int32 | readonly | NO | sensorsCount | **Аналоговые показания.** Последние распарсенные значения параметров |
| 10 | `fromValue` | int32 | option | EEPROM | 1 | **Нижний порог.** Значение для интерпретации бинарной единицы |
| 11 | `toValue` | int32 | option | EEPROM | 1 | **Верхний порог.** Значение для интерпретации бинарной единицы |
| 12 | `analogSend` | bits1 | option | EEPROM | 1 | **Отправка аналоговых.** 0=только при изменении бинарного, 1=при изменении аналогового |
| 14 | `sensorDigital` | bits1 | option | EEPROM | 1 | **Цифровой режим.** При 1 работает как бинарный датчик |
| 15 | `analogFilterMin` | uint16 | option | EEPROM | 1 | **Минимальное изменение.** Игнорировать изменения меньше этого значения |
| 16 | `analogFilterMax` | uint16 | option | EEPROM | 1 | **Максимальное изменение.** Считать сбоем изменения больше этого значения |
| 17 | `error` | int8 | readonly | NO | 1 | **Код ошибки.** Код последней ошибки чтения датчика |
| 18 | `average` | bits1 | option | EEPROM | 1 | **Усреднение.** Вычислять среднее арифметическое с предыдущим значением |
| 20 | `streamChar` | uint8 | option | EEPROM | sensorsCount+1 | **Буквы параметров.** Буквенные идентификаторы для каждого параметра датчика |
| 21 | `baudrate` | uint8 | option | EEPROM | 1 | **Скорость порта.** Скорость последовательного порта (0=не менять, 1-10=1200-115200 бод) |

**Примечание:**
*   **Кол-во:** Параметры с `sensorsCount` имеют индивидуальное значение для каждого параметра датчика. Остальные параметры — по 1 значению.
*   **Параметр 1 (pin):** Имеет `MWOSSensorStreamPinsCount` значений (по умолчанию 2: RX и TX).
*   **Параметр 20 (streamChar):** Имеет `sensorsCount+1` значений (дополнительный индекс для кода ошибки).
*   **Параметр 21 (baudrate):** 0=не менять скорость, 1=1200, 2=2400, 3=4800, 4=9600, 5=14400, 6=19200, 7=28800, 8=38400, 9=57600, 10=115200 бод.

## Конфигурация модуля

```
; модуль sensorSerial - потоковые датчики через UART
; 0 - Бинарные показания (sensorsCount значений)
sensorSerial:valueBin = 0, 0, 0, 0, 0
; 1 - Порты RX/TX (MWOSSensorStreamPinsCount значений)
sensorSerial:pin = 16, 17, -1, -1, -1
; 2 - Количество измерений с момента включения
sensorSerial:valueCount = 0
; 3 - Фильтр переключения бинарного значения [сек/10]
sensorSerial:binFilter = 0
; 4 - Минимальное время между опросами показаний датчика [сек/10]
sensorSerial:getTimeout = 10
; 5 - Инвертировать значение
sensorSerial:invert = 0
; 6 - Всегда отключено
sensorSerial:alwaysOff = 0
; 7 - Подтяжка портов (0=Вход;1=На землю;2=На плюс;3=АЦП)
sensorSerial:pull = 0
; 8 - Выключать подтяжку порта датчика между опросами
sensorSerial:pullOff = 0
; 9 - Последние показания (sensorsCount значений)
sensorSerial:value = 3000, 257, 959, 2500, 4
; 10 - Нижний порог для valueBin
sensorSerial:fromValue = 0
; 11 - Верхний порог для valueBin
sensorSerial:toValue = 4095
; 12 - Отправлять аналоговые значения (0=только бинарное, 1=аналоговое)
sensorSerial:analogSend = 1
; 14 - Режим цифрового датчика (0=аналоговый, 1=цифровой)
sensorSerial:sensorDigital = 0
; 15 - Минимальное изменение (фильтр)
sensorSerial:analogFilterMin = 0
; 16 - Максимальное изменение (сбой)
sensorSerial:analogFilterMax = 0
; 17 - Код ошибки
sensorSerial:error = 0
; 18 - Среднее арифметическое с предыдущим значением
sensorSerial:average = 0
; 20 - Буквы параметров (sensorsCount+1 значений)
sensorSerial:streamChar = 79, 84, 80, 37, 101, 0
; 21 - Скорость порта (0=не менять, 1-10=1200-115200 бод)
sensorSerial:baudrate = 4
```

**Примечание:**
*   **Кол-во значений:** Параметры `valueBin`, `pin`, `value`, `streamChar` имеют множественные значения. Остальные параметры — по 1 значению.
*   **Единицы измерения:** `getTimeout`, `binFilter` — десятые доли секунды.
*   **Буквы параметров:** Значения ASCII кодов букв (O=79, T=84, P=80, %=37, e=101).
*   **Скорость порта:** 4 = 9600 бод (по умолчанию для большинства датчиков).

## Конструкторы

Класс является шаблонным: `MWOSSensorSerial<sensorsCount, MWOSSensorAnalogType>`.

1.  **С профилем и портами**
    ```cpp
    MWOSSensorSerial(Stream *serial, const uint8_t *profile, const uint8_t *pins);
    ```
    Создаёт модуль с указанием последовательного порта, профиля датчика и массива пинов. Профиль содержит скорость, буквы параметров и команду режима запросов.

2.  **С расширенной фильтрацией**
    ```cpp
    MWOSSensorSerial<4, MWOSSensorFilter<4>> sensorSerial;
    ```
    Создаёт модуль с поддержкой фильтрации Калмана и коррекции показаний коэффициентами.

3.  **С альтернативной шиной UART**
    ```cpp
    MWOSSensorSerial<4>(&Serial1, profile, pins);
    ```
    Создаёт модуль с указанием альтернативного последовательного порта (Serial1, Serial2).

## Основные методы

### Чтение значений

*   `virtual int32_t readAnalogValue(int16_t arrayIndex)` *(переопределён)*
    *   Отправляет запрос датчику для получения нового значения параметра.
    *   Отправляет буквенный идентификатор параметра через последовательный порт.
    *   **Возвращает:** `int32_t` — ранее полученное усреднённое значение параметра.

*   `size_t SendChar(char ch, bool ln)`
    *   Отправляет символ в последовательный порт.
    *   При `ln=true` добавляет перевод строки `\r\n`.
    *   **Возвращает:** `size_t` — количество отправленных байт.

*   `size_t SendStr(const String &s, bool ln, bool needAnswerString)`
    *   Отправляет строку в последовательный порт.
    *   При `needAnswerString=true` ожидает ответ от датчика.
    *   **Возвращает:** `size_t` — количество отправленных байт.

### Инициализация

*   `virtual void onEvent(MWOSModeEvent modeEvent)` *(переопределён)*
    *   **`EVENT_INIT` / `EVENT_CHANGE`:** Загрузка настроек из хранилища, настройка скорости порта, отправка команды режима запросов.
    *   **`EVENT_UPDATE`:** Чтение входящего потока данных, парсинг буквенных идентификаторов и числовых значений.

*   `void setProfile(const uint8_t *profile, bool fromProgMem)`
    *   Устанавливает профиль датчика из PROGMEM или ОЗУ.
    *   Загружает скорость порта, буквенные идентификаторы, команду режима запросов.
    *   Сохраняет изменения в хранилище при инициализированном хранилище.

### Парсинг данных

*   `int8_t IndexOfCmd(char ch)` *(приватный)*
    *   Находит индекс параметра по буквенному идентификатору.
    *   Инициализирует переменные для парсинга числа.
    *   **Возвращает:** `int8_t` — индекс параметра или -1 если не найдено.

## Пример использования

### Подключение датчика LuminOx (кислород)

```cpp
#include "MWOSSensorSerial.h"

// Создаём тип для 4 параметров датчика LuminOx
MWOSSensorSerial<4> sensorSerialO2;

// Пины датчика O2 в PROGMEM
const uint8_t PROGMEM sensPinsO2[] = {16, 17}; // RX, TX

// Профиль датчика LuminOx в PROGMEM
// Первый байт - скорость (4=9600), потом буквы параметров (O,T,P,%,e)
const uint8_t PROGMEM profileLuminOx[] = {4, 'O','T','P','%','e'};

// Экземпляр модуля
sensorSerialO2(&Serial1, profileLuminOx, sensPinsO2);

void setup() {
    // Настройки по умолчанию (загружаются только если хранилище не инициализировано)
    sensorSerialO2._getTimeout = 10;  // Опрос каждые 1 секунду
    sensorSerialO2._fromValue = 0;    // Нижний порог
    sensorSerialO2._toValue = 4000;   // Верхний порог
    
    // Запуск системы - загружает настройки из хранилища
    mwos.start();
}

void loop() {
    // Все действия выполняются внутри модулей через обработчики событий
    mwos.update();
}
```

### Подключение с фильтрацией Калмана

```cpp
#include "MWOSSensorSerial.h"

// Создаём модуль с фильтрацией
MWOSSensorSerial<4, MWOSSensorFilter<4>> sensorSerial;

// Пины и профиль
const uint8_t PROGMEM pins[] = {16, 17};
const uint8_t PROGMEM profile[] = {4, 'A','B','C','D','E'};

void setup() {
    // Экземпляр модуля
    sensorSerial(&Serial1, profile, pins);
    
    // Настройка коэффициентов коррекции (загружаются только если хранилище не инициализировано)
    sensorSerial._coef[0] = 1.0;
    sensorSerial._subCoef[0] = 0.0;
    
    // Включение фильтра Калмана
    sensorSerial._filter[0] = EFM_Filter2;
    
    // Настройка параметров фильтра
    sensorSerial.sensorFilter[0]->Q = 0.01;
    sensorSerial.sensorFilter[0]->R = 0.1;
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
    
    // Чтение фильтрованного значения
    double value = sensorSerial.getFilteredValue(0);
}
```

### Настройка скорости порта

```cpp
#include "MWOSSensorSerial.h"

// Создаём модуль с 5 параметрами
MWOSSensorSerial<5> sensorSerial;

const uint8_t PROGMEM pins[] = {16, 17};
const uint8_t PROGMEM profile[] = {4, 'A','B','C','D','E'};

void setup() {
    // Экземпляр модуля
    sensorSerial(&Serial1, profile, pins);
    
    // Настройка скорости порта (загружается только если хранилище не инициализировано)
    // 0=не менять, 1=1200, 2=2400, 3=4800, 4=9600, 5=14400
    // 6=19200, 7=28800, 8=38400, 9=57600, 10=115200
    sensorSerial._baudrate = 4;  // 9600 бод
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
}
```

## Способы настройки параметров

Значения параметров модулей можно задавать несколькими способами:

1.  **Значения по умолчанию в коде** (до `mwos.start()`)
    ```cpp
    void setup() {
        sensorSerial._getTimeout = 10;  // Опрос каждые 1 секунду
        sensorSerial._baudrate = 4;     // 9600 бод
        mwos.start();
    }
    ```
    **Важно:** Настройки задаются **до** вызова `mwos.start()`. При первом запуске контроллера значения из хранилища не инициализированы, поэтому будут использованы заданные по умолчанию. При последующих запусках значения загружаются из хранилища, что позволяет изменять их удалённо.

2.  **Файл конфигурации `defaults.cfg`** (для контроллеров с файловой системой)
    ```
    ; 1 - Порты RX/TX (2 значения)
    sensorSerial:pin = 16, 17
    ; 4 - Минимальное время между опросами [сек/10]
    sensorSerial:getTimeout = 10
    ; 20 - Буквы параметров (ASCII коды)
    sensorSerial:streamChar = 79, 84, 80, 37, 101
    ; 21 - Скорость порта (4=9600 бод)
    sensorSerial:baudrate = 4
    ```
    Файл размещается в корневой директории файловой системы контроллера.

3.  **Удалённая настройка через web-интерфейс или mwosApp**
    Параметры можно изменять в реальном времени через:
    *   Веб-интерфейс контроллера
    *   Кроссплатформенное приложение `mwosApp`

4.  **Команды через Serial-терминал**
    ```
    {sensorSerial:pin = 16, 17}
    {sensorSerial:getTimeout = 10}
    {sensorSerial:streamChar = 79, 84, 80, 37, 101}
    {sensorSerial:baudrate = 4}
    ```
    Команды отправляются в текстовом виде через последовательный порт.

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `pin` | NO | Порты не сохраняются (readonly) |
| `streamChar` | EEPROM | Буквенные идентификаторы параметров |
| `baudrate` | EEPROM | Скорость последовательного порта |
| `valueBin` | NO | Вычисляемое значение, не хранится |
| `value` | NO | Последние распарсенные значения, не хранятся |
| `fromValue`, `toValue` | EEPROM | Пороговые значения для бинарной интерпретации |
| `analogFilterMin`, `analogFilterMax` | EEPROM | Параметры фильтрации аналоговых значений |
| `binFilter`, `getTimeout` | EEPROM | Тайминги опроса датчиков |
| `invert`, `alwaysOff`, `pullOff` | EEPROM | Флаги режима работы |
| `pull` | EEPROM | Режим подтяжки портов |
| `analogSend`, `sensorDigital`, `average` | EEPROM | Режимы отправки и обработки данных |
| `error` | NO | Код ошибки, не хранится |
| `valueCount` | NO | Счётчик измерений, не хранится |

## Важные замечания

1.  **Порядок инициализации:** Все настройки по умолчанию должны задаваться **до** вызова `mwos.start()`. После запуска системы значения загружаются из хранилища, и повторная установка значений в коде будет перезаписывать сохранённые настройки при каждой перезагрузке.

2.  **Формат данных датчика:** Датчик должен передавать данные в формате "буква + число". Десятичные точки и запятые игнорируются при парсинге. Значение параметра сохраняется как целое число со знаком.

3.  **Пример потока данных:** Для датчика LuminOx: `O 0300.0 T +25.7 P 0959 % 025.00 e 0004`. В этом примере: O=3000, T=257, P=959, %=2500, e=4.

4.  **Таймаут опроса:** Параметр `getTimeout` задаёт минимальное время между опросами датчика. Для медленных датчиков увеличьте это значение для избежания конфликтов.

5.  **Использование с MWOSSensorFilter:** При указании `MWOSSensorFilter` вместо `MWOSSensorAnalog` модуль получает дополнительные параметры для фильтрации Калмана (ID 30-40). Это увеличивает потребление памяти, но улучшает стабильность показаний.

6.  **Скорость порта:** Параметр `baudrate` со значением 0 не меняет скорость порта. Используйте это при работе с портами, скорость которых задана elsewhere.

7.  **Архитектура MWOS:** Все действия выполняются внутри модулей через обработчики событий. Пользовательский код в `loop()` должен содержать только вызов `mwos.update()`. Для расширения функциональности создавайте собственные модули, наследуясь от `MWOSModule`.

8.  **Парсинг данных:** Модуль автоматически распознаёт буквенные идентификаторы и извлекает числовые значения. Отрицательные числа поддерживаются (знак минус перед числом).

9.  **Режим запросов:** При наличии команды `_commandModeASK` модуль отправляет её при инициализации для перевода датчика в режим запросов.

10. **Обработка ошибок:** Код ошибки сохраняется в параметре `error` (ID 17) при неудачном чтении или парсинге данных от датчика.

11. **Количество параметров:** Параметр `streamChar` имеет `sensorsCount+1` значений. Дополнительный индекс используется для хранения кода ошибки датчика.

## См. также

*   **MWOSSensorBase**: Базовый класс для всех датчиков. Реализует общую логику опроса и фильтрации бинарных значений.
*   **MWOSSensorAnalog**: Базовый класс для аналоговых датчиков. Добавляет поддержку пороговых значений и фильтрации аналоговых сигналов.
*   **MWOSSensorFilter**: Расширение для фильтрации Калмана и коррекции показаний коэффициентами.
*   **MWOSSensorDHT**: Модуль для датчиков температуры и влажности DHT серии.
*   **Библиотека Stream**: Стандартная библиотека для работы с последовательными портами.
