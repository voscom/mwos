# MWOSEncoder

**Имя модуля:** `encoder`

**MWOSEncoder** — модуль для подключения энкодеров с измерением частоты вращения и направления. Расширяет функциональность MWOSCounter поддержкой аппаратных и программных энкодеров.

## Описание

Класс предназначен для:
*   **Подсчёт импульсов энкодера:** Работа с инкрементальными энкодерами (A/B фаза)
*   **Измерение частоты вращения:** Расчёт скорости вращения в реальном времени
*   **Определение направления:** Автоматическое определение направления вращения (вперёд/назад)
*   **Аппаратная поддержка:** Использование аппаратных счётчиков ESP32 (PCNT) или программный подсчёт

**Наследование:** `MWOSEncoder` → `MWOSCounter` → `MWOSModule`

**Тип модуля:** `MODULE_ENCODER`

**Зависимости:**
*   Библиотека `MWOSCounter.h` (базовый класс счётчика)
*   Платформа ESP32 (опционально, для аппаратного режима PCNT)

## Основные возможности

*   **Поддержка A/B энкодеров:** Подключение квадратурных энкодеров с определением направления
*   **Измерение скорости:** Расчёт частоты вращения в Гц или оборотах в минуту
*   **Режим референса:** Поддержка нулевой метки (Z-сигнал) для абсолютного позиционирования
*   **Фильтрация помех:** Настройка фильтра для устранения дребезга контактов энкодера
*   **Контроль диапазона:** Ограничение минимального и максимального значения позиции
*   **Целевые позиции:** Установка вызова функции по достижении заданной позиции
*   **Пауза/продолжение:** Временная остановка счёта без потери текущего значения

## Параметры конфигурации

Класс использует параметры базовых классов и добавляет свои специфические параметры:

| ID | Название | Тип | Группа | Хранилище | Кол-во | Описание |
|:--:|:---|:---|:---|:---|:---:|:---|
| 0 | `counter` | int32 | control | RTC | 1 | **Текущая позиция.** Счёт без учёта шага. Восстанавливается после перезагрузки |
| 1 | `position` | double64 | control | NO | 1 | **Позиция с шагом.** Счёт с учётом коэффициента шага |
| 2 | `status` | int8 | control | NO | 1 | **Статус счётчика.** 0=назад;1=вперёд;2=остановлен;3=на паузе |
| 3 | `step` | float32 | option | EEPROM | 1 | **Шаг энкодера.** Коэффициент преобразования счёта в физические единицы |
| 4 | `minimum` | int32 | option | EEPROM | 1 | **Минимум счётчика.** Нижняя граница диапазона |
| 5 | `maximum` | int32 | option | EEPROM | 1 | **Максимум счётчика.** Верхняя граница диапазона |
| 6 | `sendTimeDSec` | uint16 | option | EEPROM | 1 | **Таймаут отправки.** Период отправки значений на сервер [сек/10] |
| 7 | `type` | bits3 | option | EEPROM | 1 | **Тип счётчика.** 0=Software;1=Interrupt;2=Hardware;3=HardwareAlter;4=Timeout |
| 10 | `needSpeed` | bits1 | option | EEPROM | 1 | **Считать скорость.** Включить измерение частоты вращения |
| 11 | `invertDirection` | bits1 | option | EEPROM | 1 | **Инверсия направления.** Инвертировать направление вращения |
| 12 | `pull` | bits3 | option | EEPROM | 1 | **Подтяжка портов.** Режим (Вход;На землю;На плюс;АЦП) |
| 13 | `invertSignal` | bits1 | option | EEPROM | 1 | **Инверсия сигналов.** Инвертировать сигналы A/B энкодера |
| 14 | `pinA` | int16 | pin | FOR_PIN | 1 | **Порт A.** Номер порта для фазы A энкодера |
| 15 | `pinB` | int16 | pin | FOR_PIN | 1 | **Порт B.** Номер порта для фазы B энкодера |
| 16 | `pinRef` | int16 | pin | FOR_PIN | 1 | **Порт референса.** Номер порта для Z-сигнала (нулевая метка) |
| 17 | `filter` | uint16 | option | EEPROM | 1 | **Фильтр помех.** Время фильтрации сигналов энкодера [мкс] |
| 18 | `sendTime` | uint16 | option | EEPROM | 1 | **Время отправки.** Период отправки значений [сек/1000] |
| 19 | `stopTime` | uint16 | option | EEPROM | 1 | **Таймаут остановки.** Время до признания энкодера остановленным [мс] |
| 20 | `speed` | float32 | readonly | RTC | 1 | **Частота вращения.** Текущая скорость в Гц |

**Примечание:**
*   **Кол-во:** Все параметры имеют 1 значение (одиночный энкодер).
*   **Параметр 7 (type):** Формат значений: "Software;Interrupt;Hardware;HardwareAlter;Timeout"
*   **Параметр 2 (status):** Формат значений: "Назад;Вперёд;Остановлен;На паузе"
*   **Параметр 20 (speed):** Только для чтения, вычисляется в реальном времени.

## Конфигурация модуля

```
; модуль encoder - энкодер с измерением частоты вращения
; 0 - Текущая позиция счёта (без учёта шага)
encoder:counter = 0

; 2 - Статус счётчика (0=назад;1=вперёд;2=остановлен;3=на паузе)
encoder:status = 2
; 3 - Шаг энкодера (коэффициент преобразования)
encoder:step = 1.0
; 4 - Минимум счётчика (нижняя граница)
encoder:minimum = -2147483648
; 5 - Максимум счётчика (верхняя граница)
encoder:maximum = 2147483648
; 6 - Таймаут отправки значений на сервер [сек/10]
encoder:sendTimeDSec = 10
; 7 - Тип счётчика (0=Software;1=Interrupt;2=Hardware;3=HardwareAlter;4=Timeout)
encoder:type = 1
; 10 - Считать скорость (0=нет, 1=да)
encoder:needSpeed = 1
; 11 - Инверсия направления (0=нет, 1=да)
encoder:invertDirection = 0
; 12 - Подтяжка портов (0=Вход;1=На землю;2=На плюс;3=АЦП)
encoder:pull = 2
; 13 - Инверсия сигналов A/B (0=нет, 1=да)
encoder:invertSignal = 0
; 14 - Порт A энкодера
encoder:pinA = 25
; 15 - Порт B энкодера
encoder:pinB = 26
; 16 - Порт референса (Z-сигнал)
encoder:pinRef = -1
; 17 - Фильтр помех [мкс]
encoder:filter = 100
; 18 - Время отправки значений [сек/1000]
encoder:sendTime = 100
; 19 - Таймаут остановки [мс]
encoder:stopTime = 500
; 20 - Частота вращения (Гц, только чтение)
encoder:speed = 0.0
```

**Примечание:**
*   **Кол-во значений:** Все параметры имеют по 1 значению.
*   **Единицы измерения:** `sendTimeDSec` — десятые доли секунды. `filter` — микросекунды. `stopTime` — миллисекунды. `speed` — Герцы.
*   **Порты энкодера:** `pinA` и `pinB` обязательны для работы. `pinRef` опционален (нулевая метка).

## Конструкторы

Класс не является шаблонным (одиночный энкодер).

1.  **По умолчанию**
    ```cpp
    MWOSEncoder();
    ```
    Создаёт модуль с параметрами по умолчанию. Тип счётчика устанавливается в режим прерываний.

2.  **С указанием портов**
    ```cpp
    MWOSEncoder(MWOS_PIN_INT pinA, MWOS_PIN_INT pinB, MWOS_PIN_INT pinRef = -1);
    ```
    Создаёт модуль с указанными портами для фаз A, B и опционально референса.

3.  **С указанием типа**
    ```cpp
    MWOSEncoder(MWOSCounterType type);
    ```
    Создаёт модуль с указанным типом счётчика (Software, Interrupt, Hardware, HardwareAlter).

## Основные методы

### Чтение значений

*   `int32_t getCounter()` *(унаследован)*
    *   Возвращает текущее значение счётчика без учёта шага.
    *   **Возвращает:** `int32_t` — сырое значение счёта.

*   `double getPosition()` *(унаследован)*
    *   Возвращает позицию с учётом шага.
    *   **Возвращает:** `double` — позиция в физических единицах.

*   `float getSpeed()`
    *   Возвращает текущую частоту вращения энкодера.
    *   **Возвращает:** `float` — скорость в Гц.

*   `int8_t getStatus()` *(унаследован)*
    *   Возвращает текущий статус счётчика.
    *   **Возвращает:** `int8_t` — статус (0=назад;1=вперёд;2=остановлен;3=на паузе).

### Управление счётом

*   `void StartCount(int32_t position)` *(унаследован)*
    *   Сбросить позицию и начать отсчёт.
    *   **Параметры:** `position` — начальная позиция.

*   `void StartPos(double newPosition)` *(унаследован)*
    *   Установить позицию с учётом шага.
    *   **Параметры:** `newPosition` — новая позиция в физических единицах.

*   `void Pause()` *(унаследован)*
    *   Приостановить отсчёт. Статус устанавливается в "на паузе".

*   `void Resume()` *(унаследован)*
    *   Снять с паузы и восстановить нормальный режим счёта.

*   `void setPosEvent(int32_t pos, void (*callback)())` *(унаследован)*
    *   Установить вызов функции по достижении позиции.
    *   **Параметры:** `pos` — целевая позиция, `callback` — функция обратного вызова.

*   `void setRefEvent(void (*callback)())`
    *   Установить вызов функции при прохождении референса (Z-сигнал).
    *   **Параметры:** `callback` — функция обратного вызова.

### Инициализация

*   `virtual void onEvent(MWOSModeEvent modeEvent)` *(переопределён)*
    *   **`EVENT_INIT` / `EVENT_CHANGE`:** Загрузка настроек из хранилища, инициализация портов энкодера.
    *   **`EVENT_UPDATE`:** Вызов обработки счёта (чтение импульсов, обновление скорости и статуса).

## Пример использования

### Базовое подключение

```cpp
#include "modules/encoder/MWOSEncoder.h"

// Создаём энкодер
MWOSEncoder encoder;

void setup() {
    // Настройки по умолчанию (загружаются только если хранилище не инициализировано)
    // encoder._step = 0.1;  // Шаг 0.1 мм на импульс
    // encoder._pinA = 25;   // Порт A
    // encoder._pinB = 26;   // Порт B
    
    // Запуск системы - загружает настройки из хранилища
    mwos.start();
}

void loop() {
    // Все действия выполняются внутри модулей через обработчики событий
    mwos.update();
}
```

### Подключение с указанием портов

```cpp
#include "modules/encoder/MWOSEncoder.h"

// Создаём энкодер с портами A=25, B=26
MWOSEncoder encoder(25, 26);

void setup() {
    // Настройка шага энкодера
    encoder._step = 0.05;  // 0.05 мм на импульс
    
    // Настройка диапазона
    encoder._minimum = 0;
    encoder._maximum = 10000;
    
    // Включение измерения скорости
    encoder._needSpeed = 1;
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
    
    // Чтение позиции
    double pos = encoder.getPosition();
    
    // Чтение скорости
    float speed = encoder.getSpeed();
}
```

### Подключение с референсом (Z-сигнал)

```cpp
#include "modules/encoder/MWOSEncoder.h"

// Создаём энкодер с референсом
MWOSEncoder encoder(25, 26, 27);  // A=25, B=26, Ref=27

// Функция обратного вызова при прохождении референса
void onRefReached() {
    // Энкодер прошёл нулевую метку
    encoder.StartCount(0);  // Сбросить позицию в ноль
}

void setup() {
    // Установка вызова по прохождению референса
    encoder.setRefEvent(onRefReached);
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
}
```

### Подключение с аппаратным счётчиком (ESP32)

```cpp
#include "modules/encoder/MWOSEncoder.h"

#ifdef ESP32
// Создаём энкодер с аппаратной поддержкой PCNT
MWOSEncoder encoder(CT_HARDWARE_ALTER);

void setup() {
    // Настройка портов
    encoder._pinA = 25;
    encoder._pinB = 26;
    
    // Настройка шага энкодера
    encoder._step = 0.05;  // 0.05 мм на импульс
    
    // Настройка фильтра помех
    encoder._filter = 100;  // 100 мкс
    
    // Запуск системы
    mwos.start();
}

void loop() {
    mwos.update();
    
    // Чтение позиции
    double pos = encoder.getPosition();
}
#endif
```

## Хранение данных

| Параметр | Хранилище | Комментарий |
|:---|:---|:---|
| `counter` | RTC | Текущее значение счёта, восстанавливается после перезагрузки |
| `step` | EEPROM | Шаг энкодера, сохраняется |
| `minimum`, `maximum` | EEPROM | Границы диапазона, сохраняются |
| `sendTimeDSec`, `sendTime` | EEPROM | Таймауты отправки, сохраняются |
| `type` | EEPROM | Тип счётчика, сохраняется |
| `needSpeed`, `invertDirection` | EEPROM | Флаги режима работы, сохраняются |
| `pull`, `invertSignal` | EEPROM | Настройки портов, сохраняются |
| `filter`, `stopTime` | EEPROM | Параметры фильтрации, сохраняются |
| `pinA`, `pinB`, `pinRef` | FOR_PIN | Порты энкодера, сохраняются |
| `position` | NO | Вычисляемое значение, не хранится |
| `status` | NO | Текущий статус, не хранится |
| `speed` | RTC | Текущая скорость, восстанавливается после перезагрузки |

## Важные замечания

1.  **Типы счётчиков:** Доступны 5 режимов работы:
    *   0 = Software (программный подсчёт)
    *   1 = Interrupt (на прерываниях)
    *   2 = Hardware (аппаратный счётчик)
    *   3 = HardwareAlter (альтернативный режим, PCNT на ESP32)
    *   4 = Timeout (режим таймера для измерения времени)

2.  **Порядок инициализации:** Все настройки по умолчанию должны задаваться **до** вызова `mwos.start()`. После запуска системы значения загружаются из хранилища, и повторная установка значений в коде будет перезаписывать сохранённые настройки при каждой перезагрузке.

3.  **Шаг энкодера:** Параметр `step` задаёт коэффициент преобразования сырого счёта в физические единицы. Например, 0.1 = 0.1 мм на импульс.

4.  **Измерение скорости:** Параметр `needSpeed` включает измерение частоты вращения. Это требует дополнительных вычислений в событии EVENT_UPDATE.

5.  **Фильтр помех:** Параметр `filter` задаёт время фильтрации сигналов энкодера в микросекундах. Помогает устранить дребезг контактов и ложные срабатывания.

6.  **Таймаут остановки:** Параметр `stopTime` определяет время без импульсов, после которого энкодер считается остановленным (статус = 2).

7.  **Режим паузы:** При установке режима паузы (`Pause()`) счётчик останавливается, но текущее значение сохраняется. Для возобновления счёта используйте `Resume()`.

8.  **Архитектура MWOS:** Все действия выполняются внутри модулей через обработчики событий. Пользовательский код в `loop()` должен содержать только вызов `mwos.update()`.

9.  **Сохранение состояния:** Значение счётчика сохраняется в RTC (энергонезависимое ОЗУ) при каждом изменении. Это обеспечивает восстановление состояния после перезагрузки без износа EEPROM.

10. **Аппаратная поддержка:** На ESP32 доступен режим HardwareAlter (PCNT) для высокоточного подсчёта импульсов без нагрузки на процессор.

11. **Референс (Z-сигнал):** Опциональный порт для нулевой метки энкодера. При прохождении референса может вызываться функция обратного вызова через `setRefEvent()`.

12. **Инверсия направления:** Параметр `invertDirection` позволяет программно инвертировать направление вращения без изменения подключения проводов.

## См. также

*   **MWOSCounter**: Базовый класс для счётчиков импульсов. MWOSEncoder наследуется от него и добавляет поддержку энкодеров.
*   **MWOSModule**: Базовый класс для всех модулей контроллера. Реализует общую инфраструктуру модулей, параметры и события.
*   **MWOSKey**: Модуль ключей для управления портами микроконтроллера. Может использоваться вместе с энкодером для управления исполнительными механизмами.
*   **MWOSKeySchedule**: Модуль ключей с таймерами и расписанием. Может использоваться для автоматического управления по позиции энкодера.
*   **MWOSConsts**: Константы и перечисления системы MWOS, включая типы модулей и событий.
